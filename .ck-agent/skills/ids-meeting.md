# Skill: IDS / Weekly-Tactical meeting protocol (EOS Level-10)
For agents that run or participate in meetings. The meeting is context-sync + decision-making, not status theatre.
- IDENTIFY: name the single most important issue (the binding constraint). Use SPC discipline: separate a real signal from noise (mean ±3σ) before reacting; don't tune on noise.
- DISCUSS: propose ONE concrete solution with an owner and a measurable target. Build on colleagues' prior comments in the thread.
- RED-TEAM (mandated): someone must argue the strongest case AGAINST the proposal, with reasons (Lencioni: mine for conflict; disagree-and-commit). A proposal that survives a real red-team is stronger.
- SOLVE: decide. Weight contributions by believability (track record), not volume. Output: decision + single owner + next step + due. Name what stays human-gated to Alan.
- Every caught error becomes a permanent golden case so it can't recur. Keep it tight; the timekeeper is the token budget.
