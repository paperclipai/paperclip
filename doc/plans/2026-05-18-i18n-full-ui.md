# 前端国际化（i18n）完整支持实现计划

> **面向 AI 代理的工作者：** 必需子技能：使用 subagent-driven-development（推荐）或 executing-plans 逐任务实现此计划。步骤使用复选框（`- [ ]`）语法来跟踪进度。

**目标：** 让 Paperclip 前端 UI 全面支持英文和中文切换，覆盖所有页面和组件的界面文本。

**架构：** 项目已有 i18next + react-i18next 基础设施（37 种语言 JSON 文件），但仅 1 个组件使用了翻译（3 个字符串）。计划分 6 个阶段：添加语言切换器和持久化 → 重建翻译键体系并按组件逐批迁移 → 验证。

**技术栈：** i18next, react-i18next, i18next-browser-languagedetector, localStorage

**现有基础设施：**
- `ui/src/i18n/index.ts` — i18next 初始化入口（默认语言 `en`，硬编码）
- `ui/src/i18n/locales.ts` — 自动加载所有 `locales/*.json`
- `ui/src/i18n/locale-validation.ts` — 翻译 JSON 校验器
- `ui/src/i18n/locales/en.json` — 仅有 3 个 key
- `ui/src/i18n/locales/zh-CN.json` — 同上

### 任务 2：添加语言检测、持久化和语言切换器

**文件：**
- 修改：`ui/src/i18n/index.ts`
- 新增：`ui/src/components/LanguageSwitcher.tsx`
- 修改：`ui/src/components/Sidebar.tsx`

- [ ] **步骤 1：重构 i18n/index.ts**

添加 `i18next-browser-languagedetector` 依赖并注入，配置 localStorage 持久化，将默认语言从硬编码 `"en"` 改为从 localStorage 读取。暴露 `changeLanguage` 工具函数。

```typescript
import LanguageDetector from "i18next-browser-languagedetector";

const detector = new LanguageDetector();
// 配置：localStorage 优先（key: "paperclip_locale"），其次浏览器语言

i18n.use(detector).use(initReactI18next).init({
  ...i18nextOptions,
  detection: {
    order: ["localStorage", "navigator"],
    lookupLocalStorage: "paperclip_locale",
    caches: ["localStorage"],
  },
});

export function changeLanguage(lng: string) {
  localStorage.setItem("paperclip_locale", lng);
  void i18n.changeLanguage(lng);
}
```

- [ ] **步骤 2：创建 LanguageSwitcher.tsx**

一个简单的下拉选择器组件，使用 `changeLanguage` 切换语言，显示 "English" 和 "中文"。放在 Sidebar 底部区域。

- [ ] **步骤 3：在 Sidebar 底部插入 LanguageSwitcher**

在 Sidebar 底部（或用户设置区域下方）添加 LanguageSwitcher 组件，不影响现有导航布局。

- [ ] **步骤 4：Sidebar 和 InstanceSidebar 文本迁移**

将 Sidebar.tsx 和 InstanceSidebar.tsx 中的所有硬编码导航标签、分区标题、ARIA 标签替换为 `{t("nav.sidebar.xxx")}` 调用。
