# v2026.416.0

> Released: 2026-04-16

## Highlights

- **Font CDN independence** — UI fonts are now bundled locally instead of loaded from Google Fonts, improving privacy and offline reliability.
- **Backup reliability fix** — `backup-docker.sh` now correctly captures `PIPESTATUS` before it is reset by subsequent commands.
- **Upstream sync** — Merged upstream `paperclipai/paperclip` master (commits through `7f893ac4`), bringing workspace runtime improvements, heartbeat tooling hardening, issue detail UX improvements, and new sidebar preference persistence.
- **Test environment fix** — Fixed `localStorage.clear is not a function` test failures caused by Node.js 25's built-in localStorage conflicting with jsdom tests.

## Fixed

- `fix(ui)`: Bundle fonts locally — removes Google Fonts CDN dependency for privacy and offline support.
- `fix(backup)`: Capture `PIPESTATUS` array immediately after piped command to prevent reset by subsequent shell operations.
- `fix(backup)`: Apply `chmod 600` to temp backup file immediately after `mktemp`, before `pg_dump` writes, closing a world-readable race window on macOS where `umask 077` is not reliable.
- `fix(heartbeat)`: Bound `reconcileStrandedAssignedIssues` query with `.limit(100)` and cap `resolveRunScopedMentionedSkillKeys` comments query with `.limit(500)` to prevent OOM/DB timeout on large installs.
- `fix(heartbeat)`: Move `logActivity` for issue reopen into the transaction in `releaseIssueExecutionAndPromote`, closing a silent audit gap if the process crashes between tx commit and the post-tx call.
- `fix(heartbeat)`: Make `escalateStrandedAssignedIssue` idempotent — wraps the status update in a `SELECT … FOR UPDATE` transaction, preventing duplicate escalation comments from concurrent reconcile ticks.

## Added

- `packages/db/src/migrations/0057_heartbeat_jsonb_indexes.sql` — Expression indexes on `heartbeat_runs (context_snapshot ->> 'issueId')` and `agent_wakeup_requests (payload ->> 'issueId')`, eliminating sequential scans in heartbeat reconciliation on large installs.

## Added

- `ui/src/test-setup.ts` — Vitest setup file that polyfills `localStorage` for Node.js 25 compatibility with jsdom test environments.
- `ui/vitest.config.ts` — Registers the test setup file.
- `ui/package.json` — Adds `jsdom` and `happy-dom` dev dependencies for proper vitest environment support.

## Changed

- Merged upstream workspace runtime refactors: sidebar preferences persistence, workspace runtime controls UI, improved heartbeat comment-wake batching, live updates provider, and enhanced issue chat thread.
