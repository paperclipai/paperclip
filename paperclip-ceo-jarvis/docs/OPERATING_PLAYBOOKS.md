# JARVIS Operating Playbooks

## Daily CEO Operating Loop

Purpose: convert the current system state into one committed daily outcome.

Flow:

1. Generate morning briefing.
2. Rank top three decisions.
3. Protect focus by identifying what to delegate, delay, or delete.
4. CEO commits the daily plan.

## Strategic Decision War-game

Purpose: test important choices before commitment.

Flow:

1. Frame decision and constraints.
2. Scan market and opportunity signals.
3. Review architecture and execution impact.
4. Estimate financial/KPI impact.
5. CEO approval gate.

## Product Launch Room

Purpose: move from idea to launch with safety and rollback planning.

Flow:

1. Launch readiness checklist.
2. Security and privacy check.
3. Generate launch communications.
4. Execute approved release.

## Incident Response Room

Purpose: contain damage, preserve evidence, communicate clearly, and restore service.

Flow:

1. Classify severity.
2. Contain and preserve evidence.
3. Prepare restoration plan.
4. Draft stakeholder communication.

## Authorized Integration Onboarding

Purpose: connect data sources without unsafe credentials or excessive permissions.

Flow:

1. Define exact scopes.
2. Configure environment or vault keys.
3. Run safe read-only sync.
4. Document connector SOP.

## Weekly Strategy Review

Purpose: convert weekly activity into calibrated learning and next-week constraints.

Flow:

1. Summarize week.
2. Score prediction accuracy.
3. Identify strategic drift.
4. Set next-week constraints.
