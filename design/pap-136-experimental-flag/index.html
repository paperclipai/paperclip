PAP-136 · Put all the work on this worktree behind an experimental feature flag

"Conference Room Chat" — experimental flag interaction

One new instance-level experimental flag, enableConferenceRoomChat, gates every user-visible change from the PAP-46-nux-tweaks worktree: the Conference Room chat page, the live activity feed, the capsule onboarding wizard, chat-style task threads, brand status chips, and the team-centric copy. OFF by default — with the flag off the app looks and behaves exactly like master. It follows the same pattern the board approved for the Streamlined Left Navigation flag.

3 screens Desktop + mobile Lo-fi · monochrome Click any wireframe to zoom

Interaction flow

Default OFF → settings toggle → instant unlock → reversible at any time.

Settings > Instance settings > Experimental — flag OFF (default)

The new card sits in the existing experimental list, right after "Streamlined Left Navigation Bar".

Regions

  1. New "Conference Room Chat" card — title + two-line description + toggle, identical chrome to every other experimental card
  2. Existing flags are untouched and compose independently (e.g. Streamlined Left Nav can be on while this is off)

Proposed two-line description

Adds a Conference Room — one chat where you and your whole team work together — plus the live activity feed and the redesigned onboarding. Also restyles task threads as chat bubbles. Turn off anytime to restore the classic UI.

Mobile: same card full-width; description wraps to four lines.

Toggling ON — instant, no reload

Toggle persists via the existing PATCH instance-experimental-settings route; react-query invalidation makes the sidebar pick it up immediately.

Regions

  1. Toggle flips ON — saved per instance, same persistence as all other experimental flags
  2. "Conference Room" appears in the app sidebar instantly (on mobile: visible the next time the nav drawer opens)
  3. Save confirmation toast
Reversible. Toggling OFF removes the nav item and restores classic task threads / onboarding immediately. No data is lost — conference-room history and artifacts stay in the DB, just hidden.

What the flag unlocks

Everything user-visible from this worktree, as one switch.

Regions

  1. "Conference Room" sidebar nav item + /board-chat route
  2. Chat surface — agent bubbles with name/icon header + action row, task chips, typing dots, user bubbles
  3. Live activity feed rail (on mobile: a "Feed" tab)
  4. Shared ChatComposer — Agent/Plan mode chip, translucent surface
Also gated by the same flag: capsule onboarding wizard (FrontDoor + OnboardingChat + AgentCapsule), chat-style task comment threads, brand status chips, team-centric copy, and the /api/board/chat/stream server endpoint (403 when off).
