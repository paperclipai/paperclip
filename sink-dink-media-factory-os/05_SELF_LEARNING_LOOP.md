# Self-Learning Daily Improvement Loop

## Purpose
Make the organisation improve itself daily without uncontrolled automatic changes.

## Daily Review
Every day the system should review:
- Yesterday content output
- Hook strength
- Script quality
- Carousel clarity
- Design readability
- Caption and hashtag relevance
- QA issues
- Workflow delays
- Agent errors
- User feedback

## Scoring
Each output should receive:
- Hook score
- Clarity score
- Brand fit score
- Upload readiness score
- Risk score
- Improvement priority

## Improvement Actions
The System Improvement Agent can suggest:
- Better hook formulas
- Better caption patterns
- Better slide structures
- Better workflow routing
- Skill updates
- Agent instruction updates
- Tool or connector improvements

## Approval Rule
The system may suggest improvements automatically, but major changes require human approval before being applied.

## Memory Rule
Approved improvements should be recorded in durable docs, skills, or templates so the organisation becomes more efficient over time.

## Success Criteria
The system should become faster, cleaner, and more reliable every week while keeping the human in control.
