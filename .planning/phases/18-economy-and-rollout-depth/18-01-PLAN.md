# Phase 18: Economy and Rollout Depth - Plan 01

## 목표

Marketplace/P&L/enterprise rollout 화면을 실제 설정과 검수 evidence가 연결된 운영자-ready 표면으로 만든다.

## 작업

- [x] P&L summary API에 settlement evidence를 추가한다.
- [x] P&L UI에서 settlement source table, ledger type, actor drilldown, approved deliverable, coin ledger를 표시한다.
- [x] Marketplace evidence에 approved base price, earned gold estimate, latest approved deliverables, reputation/collaboration/subscription basis를 추가한다.
- [x] Marketplace UI를 Jarvis marketplace copy와 evidence drilldown으로 보강한다.
- [x] Enterprise rollout overview에 SSO/template/binding/policy별 `ready/partial/missing` evidence를 추가한다.
- [x] Rollout UI가 저장된 실제 설정값을 form에 hydrate하고 검수 evidence를 표시하게 한다.
- [x] 요구사항/로드맵/정합성 문서를 갱신한다.
- [x] shared/server/ui typecheck와 scoped route tests를 실행한다.

## 검증 기준

- `ECON-01`: marketplace/P&L 화면이 금화, 기준가, 품질, 협업 보상의 실제 산정 근거를 연결해 표시한다.
- `ENT-01`: enterprise rollout 화면이 SSO, template, binding mode, policy default를 운영 검수 가능한 설정 흐름으로 제공한다.

## 위험 및 경계

- 실제 SSO handshake와 SCIM sync는 구현하지 않는다.
- Embedded Postgres 테스트는 호스트 제약으로 skip될 수 있다. 이 경우 typecheck와 test collection 결과를 기록한다.
