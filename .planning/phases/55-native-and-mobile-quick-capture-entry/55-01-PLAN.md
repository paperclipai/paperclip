## 1. Add a bounded local quick-capture queue

- Create `ui/src/lib/rt2-quick-capture-queue.ts`.
- Queue item fields:
  - `id` as stable client-generated local id and inbound `eventId`
  - `companyId`, `projectId`, `source`, `channel`, `text`
  - `status`: `draft`, `queued`, `sending`, `failed`, `sent`
  - `createdAt`, `updatedAt`, `lastAttemptedAt`, `lastError`, `sentDraftId`, `sentDraftStatus`
- Store only the reviewed text payload and minimal retry context. Do not store auth tokens, cookies, signing secrets, or session data.
- Use guarded JSON parsing, schema-like normalization, max item count, max text length, stale/corrupt entry recovery, and per-company keys when company id exists.
- Provide pure functions for enqueue/update/remove/list/markSending/markFailed/markSent and a small browser-storage adapter.
- Add `ui/src/lib/rt2-quick-capture-queue.test.ts` covering:
  - valid queue round-trip
  - corrupt storage recovery
  - max count/text bounds
  - state transitions
  - no auth/session fields persisted

## 2. Build the mobile/PWA quick capture page

  - selected company

  - last sync result

- On network/API failure:

  - show `전송 실패` and retry action
  - retry on explicit button, `online` event, and app focus when context is valid

<threat_model>
- **T-55-01 Local data exposure:** Offline queue stores business text on the device. Mitigation: bounded storage, no auth/session/secrets, visible device-local state, clear delete/remove path.
- **T-55-02 Cross-company send:** A queued item could be sent to the wrong company after context changes. Mitigation: store company/project context, show target context before send, and block/reconfirm if context is missing or mismatched.
- **T-55-03 Unauthorized submission:** Missing/expired auth could silently fail or create confusing state. Mitigation: explicit auth state, local save only, and server send blocked until session exists.
- **T-55-04 Duplicate retry:** Repeated retries could create duplicate drafts. Mitigation: stable local `eventId`, sent-state tracking, and existing backend duplicate detection.
- **T-55-05 Product identity regression:** PWA install metadata could expose Paperclip. Mitigation: update manifest and extend identity gate tests.
</threat_model>
