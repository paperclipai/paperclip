<objective>
Close Phase 27 security gap T-27-02 by making concurrent writes to the same actor ledger balance scope serialize before computing `balanceAfter`.

Purpose: Phase 27 already removed application-level read-then-write logic, but the security audit found that concurrent SQL subquery inserts were not protected by serializable isolation or an equivalent lock. This plan adds deterministic per-actor transaction-scoped locking and regression coverage.
</objective>
