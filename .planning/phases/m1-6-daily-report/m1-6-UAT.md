# M1.6 UAT - 일일업무보고서 시스템

## UAT Checks

### UAT-1: 일일업무보고서 생성
- [ ] 사용자가 날짜를 선택하고 새 보고서를 생성할 수 있다
- [ ] 보고서가 draft 상태로 생성된다
- [ ] 보고서 목록에 새 보고서가 표시된다

### UAT-2: Task 카드 관리
- [ ] 사용자가 Task를 드래그하여 보고서에 추가할 수 있다
- [ ] Task 카드가 보고서 내에 표시된다
- [ ] Task 완료 상태가 변경되면 보고서에 반영된다

### UAT-3: To-Do 관리
- [ ] 사용자가 To-Do를 생성할 수 있다
- [ ] To-Do에 산출물 필수 필드가 있다 (deliverable_name, deliverable_type)
- [ ] To-Do 완료 시 deliverable_link 입력이 요구된다

### UAT-4: 산출물 정의
- [ ] 모든 Task와 To-Do에 산출물 정의 필드가 있다
- [ ] 산출물 유형 드롭다운이 있다 (문서/데이터/보고서/코드)
- [ ] 산출물 링크 필드가 있다

### UAT-5: 금화 현황
- [ ] 보고서 화면에 금화 현황이 표시된다
- [ ] 금화 획득 내역이 표시된다

### UAT-6: 상태 변경
- [ ] 사용자가 보고서를 제출(submit)할 수 있다
- [ ] 상태가 'submitted'로 변경된다
- [ ] 상태 배지가 올바른 색상으로 표시된다

### UAT-7: AI 자산화 버튼 (Stub)
- [ ] [AI 자산화] 버튼이 존재한다
- [ ] 버튼 클릭 시 stub 응답이 온다

### UAT-8: 3패널 레이아웃
- [ ] 좌측 패널에 OKR 트리가 표시된다
- [ ] 중앙 패널에 보고서 내용이 표시된다
- [ ] 우측 패널에 상세/자비스 영역이 표시된다

### UAT-9: 금일 보고서 조회
- [ ] /api/daily-reports/today 엔드포인트가 동작한다
- [ ] 금일 날짜의 보고서가 반환된다
- [ ] 금일 보고서가 없으면 빈 객체가 반환된다

### UAT-10: 빌드 검증
- [ ] pnpm -r typecheck 통과
- [ ] UI 빌드 성공
