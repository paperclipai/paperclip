   ```typescript
   export interface CreateWorkInput {
     companyId: string;
     taskIssueId?: string;
     deliverableWorkProductId?: string;
     actorType?: Rt2DomainEventActorType;
     actorId?: string;
   }

   export interface UpdateWorkStateInput {
     workEntityId: string;
     newState: 'draft' | 'active' | 'completed' | 'cancelled';

   }

   export function rt2WorkEntityService(db: Db) {
     // createWork: inserts rt2V33WorkEntities with state='draft', emits rt2.work.created
     async function createWork(input: CreateWorkInput): Promise<Rt2V33WorkEntities>

     // updateWorkState: validates current state, updates to new state, emits rt2.work.state_changed
     async function updateWorkState(input: UpdateWorkStateInput): Promise<Rt2V33WorkEntities>

     // archiveWork: sets archivedAt timestamp, emits rt2.work.archived (for migration)
     async function archiveWork(workEntityId: string, migrationBatchId: string): Promise<void>

     async function getWorkById(workEntityId: string): Promise<Rt2V33WorkEntities | null>

     async function listWorksByCompany(companyId: string, state?: string): Promise<Rt2V33WorkEntities[]>

     // getWorkProjectorReadModel: returns aggregated work stats by state for a company
     async function getWorkProjectorReadModel(companyId: string): Promise<{
       total: number;
       draft: number;
       active: number;
       completed: number;
       cancelled: number;
     }>
   }
   ```

3. **Event emission pattern** (follow rt2-task-execution.ts):
   - `createWork` emits `rt2.work.created` with idempotency key `rt2.work.created:${workEntityId}`

   - `archiveWork` emits `rt2.work.archived` with idempotency key `rt2.work.archived:${workEntityId}:${migrationBatchId}`

4. **State transition validation**:
   - draft → active: allowed
   - active → completed: allowed
   - active → cancelled: allowed
   - completed → * : NOT allowed (terminal state)
   - cancelled → * : NOT allowed (terminal state)
   - Any other transition: throw error

   - Work projector read model is maintained via the projector pattern (same as rt2.activity_live_bridge)

**WORK-03: Legacy migration script — idempotent, preserves data, archives legacy**

1. **Read existing patterns** from `server/src/services/rt2-domain-events.ts` (append pattern) for idempotency

2. **Create `server/src/services/rt2-work-migration.ts`** with:

   ```typescript
   export interface MigrationBatch {
     batchId: string;
     startedAt: Date;
     completedAt?: Date;
     totalRows: number;
     migratedRows: number;
     archivedRows: number;
     skippedRows: number;
     errors: MigrationError[];
   }

   export interface MigrationError {
     rowId: string;
     error: string;
     timestamp: Date;
   }

   export interface MigrationStatus {
     isRunning: boolean;
     lastBatch?: MigrationBatch;
     totalMigrated: number;
     totalArchived: number;
   }

   export function rt2WorkMigrationService(db: Db) {
     // migrateLegacyWorkProducts: main migration entry point
     // - Reads all issue_work_products records (or batch if specified)
     // - For each record, creates rt2V33WorkEntities with state='draft'
     // - Archives original record to rt2V33WorkEntitiesArchive
     // - Idempotent: checks if already migrated before processing
     async function migrateLegacyWorkProducts(options?: {
       batchSize?: number;  // default 1000
       companyId?: string;  // filter by company
       dryRun?: boolean;    // don't actually migrate
     }): Promise<MigrationBatch>

     // getMigrationStatus: returns current migration state
     async function getMigrationStatus(): Promise<MigrationStatus>

     // rollbackMigration: reverts migration (unarchive, delete RT2 records)
     async function rollbackMigration(batchId: string): Promise<void>

     // verifyMigration: validates row counts match between source and target
     async function verifyMigration(batchId: string): Promise<{
       sourceCount: number;
       targetCount: number;
       archivedCount: number;
       matches: boolean;
     }>
   }
   ```

3. **Idempotency logic**:
   - Before migrating a row, check if `legacySourceId` already exists in `rt2V33WorkEntities`
   - If exists with same companyId+taskIssueId+deliverableWorkProductId: skip (already migrated)
   - If exists but different: error (data mismatch)

4. **Archive pattern** (per D-05: no data deletion):
   - When migrating, insert original record into `rt2V33WorkEntitiesArchive` with migrationBatchId
   - Original `issue_work_products` record is kept intact (not deleted)
   - Archive table preserves full original data + migration metadata

5. **Batch processing**:
   - Configurable batch size (default 1000)
   - Transaction per batch (atomic)
   - If batch fails, rollback that batch only
   - Continue to next batch

6. **Verification**:
   - After migration, count source rows and compare to target + archive
   - Log any discrepancies

7. **State mapping** from legacy issue_work_products.status to RT2 work entity state:
   - 'pending' | 'in_progress' → 'draft'
   - 'submitted' | 'approved' → 'active'
   - 'completed' → 'completed'
   - 'rejected' | 'cancelled' → 'cancelled'
  </action>
