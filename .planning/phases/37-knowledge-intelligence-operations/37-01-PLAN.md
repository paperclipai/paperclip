# Plan 37-01: Knowledge Intelligence Operations

<objective>
Add a deterministic company-scoped knowledge intelligence health gate and surface it in the existing RT2 Knowledge page, then record verification coverage for all v2.5 semantic knowledge requirements.
</objective>

<must_haves>
- Cover D-01 through D-15 from `37-CONTEXT.md`.
- Reuse existing semantic index, contradiction review, and Jarvis grounding data instead of adding a new storage model.
- Keep provider mode observational and local/CI behavior deterministic.
- Keep routes company-scoped and protected by `assertCompanyAccess`.
- Verification artifact must map all 19 v2.5 requirements to evidence.
</must_haves>

<tasks>

## Task 1: Shared Operations Contract
- **Type:** code
- **Files:** `packages/shared/src/types/rt2-knowledge.ts`, `packages/shared/src/index.ts`
- **Action:** Add response types for semantic knowledge operations health, including overall status, semantic index metrics, contradiction metrics, Jarvis grounding metrics, reason codes, and flow links.
- **Verify:** Type exports are available to UI and server consumers.
- **Acceptance criteria:** Contract represents `healthy`, `degraded`, and `failed`; reason codes can explain missing index, stale chunks, open contradictions, and Jarvis grounding gaps.

## Task 2: Health Aggregation Service and Route
- **Type:** code
- **Files:** `server/src/services/rt2-knowledge-operations.ts`, `server/src/routes/rt2-knowledge-operations.ts`, `server/src/routes/index.ts`, `server/src/app.ts`
- **Action:** Implement `rt2KnowledgeOperationsService.getHealth(companyId)` and expose `GET /companies/:companyId/rt2/knowledge/operations/health`.
- **Verify:** Route checks `assertCompanyAccess`; service uses existing semantic index runs/chunks, contradiction candidates/resolutions, and task/Jarvis-adjacent source rows.
- **Acceptance criteria:** API returns deterministic health with explicit reason codes and last successful semantic run separate from latest run.

## Task 3: Operations UI
- **Type:** code
- **Files:** `ui/src/api/rt2-knowledge.ts`, `ui/src/lib/queryKeys.ts`, `ui/src/pages/rt2/KnowledgePage.tsx`
- **Action:** Add an Operations tab to KnowledgePage that calls the health route and renders compact health cards, reason list, and links/actions into existing search, bridge, and Jarvis flows.
- **Verify:** Existing search/bridge tabs still render; operations query is only enabled on the operations tab.
- **Acceptance criteria:** UI shows index health, queue/run state, stale source count, provider/fallback mode, last successful run, open contradictions, Jarvis grounding status, and health reason codes.

## Task 4: Tests and Verification Artifact
- **Type:** test
- **Files:** `server/src/__tests__/rt2-knowledge-operations.test.ts`, `.planning/phases/37-knowledge-intelligence-operations/37-VERIFICATION.md`
- **Action:** Add route/service tests for healthy/degraded/failed reason codes and write milestone verification evidence.
- **Verify:** Run targeted tests and typecheck.
- **Acceptance criteria:** Tests prove deterministic fallback behavior, company boundary, route response shape, stale/open contradiction failure signals, and verification artifact covers SEM/SEARCH/CONTRA/JARVIS/OPS requirements.

</tasks>
