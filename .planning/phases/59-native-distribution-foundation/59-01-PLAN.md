# Phase 59 Plan 01: Native Distribution Foundation

<objective>
Create an operator-readable native distribution foundation contract for RealTycoon2 and a focused validation check that prevents Phase 59 from closing with vague or incomplete distribution decisions.
</objective>

<requirements>
- `DIST-01`: 운영자는 native shell packaging 후보와 platform capability 범위를 확인하고, macOS/Windows signing identity, certificate source, entitlement, updater key material을 release evidence로 관리할 수 있다.
</requirements>

<must_haves>
## Requirements
- DIST-01: Native shell baseline, platform capability boundary, signing identity/certificate/source inventory, entitlement owner, updater key material, and release evidence owner fields are documented.

## Decisions Covered
- D-01 through D-03: Tauri v2 shell baseline, Electron fallback only, no native dependency/lockfile churn in Phase 59.
- D-04 through D-08: `apps/desktop` layout, `ui/dist` reuse, runtime boundary, RealTycoon2 identity.
- D-09 through D-12: platform capability matrix and strict approval-first capture boundary.
- D-13 through D-17: macOS/Windows signing, updater keys, channels, and secret-reference inventory.
- D-18 through D-21: v2.9 regression gate bundle and no default Playwright E2E.
- D-22 through D-23: GSD tooling mismatch and dirty-worktree hygiene.
</must_haves>

<truths>
- The phase creates a foundation contract, not a native app implementation.
- Native shell choice is locked to Tauri v2 unless later official-doc research finds a blocker.
- `pnpm-lock.yaml` must not be changed by Phase 59.
- Phase 60-64 implementations remain deferred.
- Planning doc updates must be narrow because `gsd-tools phase complete 59` mis-parses the current v3.0 table shape.
</truths>
