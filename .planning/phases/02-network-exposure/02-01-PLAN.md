<objective>
Wire the Paperclip container into the Traefik network and correct the public URL so BetterAuth does not reject logins.

Purpose: Traefik routes traffic by container name inside traefik-net. Until the Paperclip server joins that network, Traefik cannot reach it even after the router config is added. The PAPERCLIP_PUBLIC_URL must be updated before the stack is restarted or BetterAuth will silently reject all login attempts from the new HTTPS hostname.

Output: Updated docker/docker-compose.yml (committed to repo) and updated .env on docker-001.
</objective>

  <action>
Edit docker/docker-compose.yml to add traefik-net. Two changes required — both are additive, nothing is removed:

1. Append a `networks:` block to the `server` service (after the `depends_on` block):
```yaml
    networks:
      - default
      - traefik-net
```

2. Add a top-level `networks:` section at the end of the file (per D-02 pattern):
```yaml
networks:
  traefik-net:
    external: true
```

The `default` network entry on the server service keeps db↔server communication working (db is on the default network). The `traefik-net` entry adds the second network attachment that lets Traefik reach the container by name `docker-server-1:3100`.

Do NOT add `traefik-net` to the `db` service — the database must not be reachable from Traefik's network.

Do NOT remove the `ports: "3100:3100"` mapping — removal is deferred.
  </action>
