<objective>
Create the internal DNS A record in Technitium so pc.thelaljis.com resolves on the LAN, then verify the complete HTTPS flow end-to-end.

Purpose: Without the Technitium A record, internal clients cannot resolve pc.thelaljis.com — they would hit Cloudflare's public DNS which may or may not point to the home IP. The internal A record ensures LAN clients go directly to docker-001 (192.168.50.117). After this task, a human verifies the full flow: DNS → Traefik → container → login page with valid cert.

Output: Technitium A record created. Human confirms dashboard is accessible over HTTPS with no cert warnings.
</objective>

Step 1 — Verify Technitium is reachable and find the correct zone name:
```
curl -s "http://192.168.50.112:5380/api/zones/list?token=1e5f4a1c1d548849d1e112283d1a2bd22d437d5dfed371e6405b42fb4ab1b532" | python3 -m json.tool | grep -i "thelaljis"
```
This confirms the zone exists. Note the exact zone name (likely `thelaljis.com`).

Step 2 — Check if a record already exists for pc.thelaljis.com (idempotency check):
```
curl -s "http://192.168.50.112:5380/api/zones/records/get?token=1e5f4a1c1d548849d1e112283d1a2bd22d437d5dfed371e6405b42fb4ab1b532&domain=pc.thelaljis.com&zone=thelaljis.com&type=A"
```
If the response contains `192.168.50.117` already, skip Step 3 and proceed to Step 4.

Step 3 — Add the A record (per D-06: pc.thelaljis.com → 192.168.50.117):
```
curl -s "http://192.168.50.112:5380/api/zones/records/add?token=1e5f4a1c1d548849d1e112283d1a2bd22d437d5dfed371e6405b42fb4ab1b532&domain=pc.thelaljis.com&zone=thelaljis.com&type=A&ttl=3600&ipAddress=192.168.50.117"
```
Expected response JSON: `{"status": "ok"}`. If the response contains `"status": "error"`, inspect the error message and resolve before continuing.

Step 4 — Verify DNS resolution from the local machine (or docker-001):
```
nslookup pc.thelaljis.com 192.168.50.112
```
Expected output includes `Address: 192.168.50.117`. If nslookup is not available:
```
ssh root@docker-001 "dig @192.168.50.112 pc.thelaljis.com A +short"
```
Expected: `192.168.50.117`

Step 5 — Verify Traefik has received a certificate for pc.thelaljis.com (may take up to 60s after Plan 02 deployed the router config):
```
ssh root@docker-001 "docker logs traefik 2>&1 | grep -i 'pc.thelaljis\|certificate\|acme' | tail -20"
```
Look for lines like `"Obtained certificate"` or `"Certificate renewed"` for `pc.thelaljis.com`. If no cert log appears yet, wait 30 seconds and retry — Traefik requests the cert when it first sees a matching request, not at config load time.

<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verify HTTPS access to Paperclip dashboard</name>
  <what-built>
    Plan 02-01: Paperclip container joined traefik-net, PAPERCLIP_PUBLIC_URL updated to https://pc.thelaljis.com
    Plan 02-02: Traefik router + service added for pc.thelaljis.com with Cloudflare certResolver
    Plan 02-03 Task 1: Technitium A record pc.thelaljis.com → 192.168.50.117 created

  <how-to-verify>
1. On a machine using Technitium as its DNS server (or that resolves via 192.168.50.112), open a browser and navigate to: **https://pc.thelaljis.com**

2. Confirm the Paperclip dashboard login page loads (you should see the login form — username/email and password fields).

3. Click the padlock icon in the browser address bar. Confirm:
   - Certificate is valid (no warnings, no "Not Secure" badge)
   - Certificate was issued by Let's Encrypt
   - Certificate is for `pc.thelaljis.com`

4. If the page does not load or shows a cert warning:

   **"ERR_NAME_NOT_RESOLVED"**: DNS record not propagated yet. Wait 30s and retry. Check `dig @192.168.50.112 pc.thelaljis.com A` returns 192.168.50.117.

   **"ERR_CONNECTION_REFUSED" or timeout**: Traefik not routing. Check `ssh root@docker-001 "curl -s http://localhost:8080/api/http/routers | python3 -c \"import json,sys; [print(r) for r in json.load(sys.stdin) if 'paperclip' in str(r)]\""` — should show the paperclip router.

   **Cert warning (invalid cert)**: Certificate not yet issued. Wait up to 90 seconds for Traefik to complete the Cloudflare DNS challenge. Check `ssh root@docker-001 "docker logs traefik 2>&1 | grep -i acme | tail -20"`.

   **Login page loads but login fails after entering credentials**: PAPERCLIP_PUBLIC_URL mismatch. Check `ssh root@docker-001 "grep PAPERCLIP_PUBLIC_URL /opt/paperclip/docker/.env"` — must be exactly `PAPERCLIP_PUBLIC_URL=https://pc.thelaljis.com` (no trailing slash).
  </how-to-verify>
