# Plan 29-01: Scheduled Wiki Consistency Lint

<objective>
Extend the existing RT2 wiki lint service so nightly batch execution can compare wiki pages for contradiction-like inconsistencies, return evidence-rich `embedding_consistency` findings, and stay completely read-only.
</objective>

<context>
- Read `.planning/phases/29-consistency-linting-batch/29-CONTEXT.md` before implementation.
- Existing service: `server/src/services/rt2-wiki-lint.ts`
- Existing route: `server/src/routes/rt2-daily-report.ts`
- Primary corpus: `packages/db/src/schema/rt2_v33_daily_wiki_pages.ts`
- Existing scheduler pattern: `server/src/services/plugin-job-scheduler.ts`
</context>

<tasks>
1. Extend `Rt2WikiLintIssue` / `Rt2WikiLintResult` to include `embedding_consistency`, evidence snippets, related page metadata, and summary counts.
2. Add semantic consistency comparison inside `rt2WikiLintService.lintWikiPages()`.
3. Keep the semantic comparison injectable/deterministic so tests do not depend on a live LLM provider.
4. Add a lightweight scheduled runner for nightly wiki linting with overlap prevention and visible run status/logging.
5. Wire the runner into server startup with an unref'd timer so it does not block tests or shutdown.
6. Add focused tests for evidence-only consistency findings and scheduled-run behavior.
</tasks>

<must_haves>
- D-01 through D-14 from CONTEXT.md must be represented.
- No code path may mutate wiki page content.
- The daily wiki write/materialization route must not trigger lint execution.
- Tests must be unit/integration-level Vitest, not Playwright.
</must_haves>

<verification>
- `pnpm --filter @paperclipai/server test -- rt2-wiki-lint`
- `pnpm typecheck`
- `pnpm test`
</verification>

<threat_model>
## Security and Safety Notes

- Lint evidence may include snippets from company wiki content. Keep all reads company/project-scoped and do not expose cross-company comparisons.
- Scheduled execution must avoid overlapping runs to prevent runaway provider usage.
- Provider failure must be visible as a skipped/failed semantic check rather than reported as a clean result.
</threat_model>
