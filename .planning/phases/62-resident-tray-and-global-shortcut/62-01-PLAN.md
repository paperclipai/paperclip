# Phase 62 Plan 01: Resident Tray And Global Shortcut Evidence Gate

<objective>
Add a deterministic RealTycoon2 resident surface evidence gate that validates tray/menubar status, OS-level global shortcut lifecycle state, privacy/focus behavior, and native quick-capture handoff into persistent draft review without adding native dependency or lockfile churn.
</objective>

<requirements>
- `RES-01`: Operators can see resident tray/menubar quick capture, sync/queue state, auth/company state, build identity, and release channel.
- `RES-02`: Operators can register, unregister, and change an OS-level global shortcut and see shortcut conflict, permission, focus, and privacy state.
- `RES-03`: Tray/global shortcut capture enters v2.9 persistent draft revision and board review inbox only; it never auto-applies work before approval.
</requirements>

<must_haves>
## Requirements
- RES-01: The manifest and report represent quick capture, queue/sync, auth/company, installed channel/build identity, update lifecycle state, and failure reason.
- RES-02: The manifest and report represent shortcut accelerator, registration, conflict, permission, focus, privacy, unregister, and change evidence.
- RES-03: The manifest and report validate `source: native` draft-review handoff and block any auto-promote/apply path.

## Decisions Covered
- D-01 through D-03: evidence-first resident surface gate, no native dependency churn, timestamped `.planning/native-resident-runs/` output.
- D-04 through D-06: tray status contract, Phase 61 installed/update state reuse, macOS/Windows required platform scope.
- D-07 through D-09: shortcut lifecycle, fail-closed states, and privacy-safe focus behavior.
- D-10 through D-13: existing inbound draft handoff, `native` source/channels, existing quick-capture queue reuse, no direct promotion.
- D-14 through D-19: focused package scripts, blocker categories, docs, and Phase 64-consumable summary.
- D-20 through D-21: focused verification and additive compatibility with v2.9 capture contracts.
</must_haves>

<truths>
- The current repo has no `apps/desktop` package and no Tauri dependency. Do not add them for this evidence gate.
- The Phase 61 update state vocabulary is the source of truth for tray-visible update lifecycle state.
- The resident gate must return non-zero when blockers exist and write durable evidence under a timestamped planning evidence directory.
- The shortcut privacy boundary is explicit typed input only; no foreground app, clipboard, selected text, screen, or window-title scraping.
- Existing dirty worktree changes must be preserved. Stage/commit only Phase 62 files.
</truths>

<threat_model>
| ID | Threat | Mitigation |
|----|--------|------------|
| T-62-01 | Tray status claims resident readiness while quick capture, queue, auth/company, build, channel, or update state evidence is missing. | Required manifest fields become blockers with stable codes and non-zero CLI exit. |
| T-62-02 | Shortcut registration conflicts or OS permission failures are hidden behind native logs. | Shortcut conflict and permission states require operator-visible reason and next action. |
| T-62-03 | Global shortcut captures private foreground context without explicit operator input. | Privacy validator blocks clipboard, selected text, screen/window title, active app, or foreground context scraping. |
| T-62-04 | Tray/shortcut capture bypasses persistent draft review and creates/promotes work directly. | Capture handoff validator requires `source: native`, inbound draft route, draft review status, and `autoApply`/`autoPromote` false. |
| T-62-05 | Raw native app tokens, keys, passwords, or credentials are committed in resident manifests or docs. | Secret-pattern rejection and sensitive-key secret-reference checks are covered by focused tests. |
</threat_model>
