## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dense badge gaps, drag handle gaps |
| sm | 8px | Compact chip padding, card internal control gaps, toolbar item gaps |
| md | 16px | Default lane/card spacing, board grid gap, side panel padding |
| lg | 24px | Board section padding and major toolbar-to-board separation |
| xl | 32px | Wide layout gaps where the board needs breathing room |
| 2xl | 48px | Rare page-level separation only; avoid inside cards |
| 3xl | 64px | Not used in this phase unless inherited page shell requires it |

Exceptions: icon-only buttons and drag handles must keep a minimum 32px visual hit area on desktop and 44px touch target where mobile/touch layouts are introduced. Lane widths must remain stable: desktop lanes use equal tracks or fixed `min-w-[21rem]`-style columns; toolbar height must not jump when filters are toggled.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 400 | 1.4 |
| Heading | 16px | 600 | 1.25 |
| Display | 20px | 600 | 1.2 |

Rules:
- Use only these four sizes for new Phase 50 UI. Existing inherited `text-[11px]` metadata chips may remain when already present on `Rt2DailyBoard` or `KanbanBoard`, but new controls should use 12px labels.
- Use only 400 and 600 weights. Do not introduce bold weights above 600.
- Board card titles use 14px/600 or 14px/400 depending on density; lane titles use 14px/600; toolbar labels use 12px/400.
- Letter spacing stays `0` except inherited uppercase design-guide labels; Phase 50 Korean labels should not use uppercase tracking.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` / dark `oklch(0.145 0 0)` | Page and board background, lane gutters, empty lane space |
| Secondary (30%) | `--card`, `--muted`, `--secondary` | Lanes, cards, toolbar shell, side metric panels, inactive chips |
| Accent (10%) | `--primary`, `--accent`, status tokens from `StatusBadge`/`status-colors.ts` | Active filter chips, focused edit field ring, selected lane/drop target, primary save affordance, status badges |
| Destructive | `--destructive` | Destructive actions only; Phase 50 has no default destructive action |

Accent reserved for: active filter chips, current sort selection, focused quick-edit field, drag/drop target highlight, pending/saved/error save indicators, quality issue and approval waiting badges. Do not use accent as general decoration or to color every clickable element.

Semantic status mapping:
- `저장중`: muted text plus subtle `--accent` background or spinner/dot.
- `저장됨`: green status token, compact and temporary.
- `저장 실패`: destructive token with retry affordance near the edited card.
- `산출물 누락`, `승인 대기`, `품질 이슈`: amber/red/violet status-token styling from existing status color conventions, not raw hex values.
- `OKR 직접 설정`: violet-tinted badge; `프로젝트 OKR 상속`: muted/violet outline badge; `OKR 없음`: muted or amber warning depending on missing-context severity.

## Phase 50 Interaction Contract

### Board Toolbar

- Place one fixed-height toolbar above the lanes, inside the `daily-work` board context. It must not be a hero section or marketing panel.
- Toolbar layout: left side filter chips, center/right search input, right sort select. On narrow screens, search wraps below chips but lane cards must not overlap controls.
- Required filter chips are visible and combinable: `오늘 업무`, `내 업무`, `산출물 누락`, `승인 대기`, `품질 이슈`.
- Active filter chips show selected state with `bg-primary text-primary-foreground` or existing active shadcn button variant. Inactive chips use outline/secondary styling.
- Filtering always preserves `할 일 / 진행 중 / 완료` lane grouping. Empty lanes remain visible with active-filter empty copy.
- Search matches visible card text and metadata: To-Do title, task title, assignee, deliverable title, OKR/goal title, status label, quality label.
- Sort is view-only. Changing sort must never mutate persisted lane/status or manual lane movement.
- Board-control state should survive board refreshes during the current UI session. Persisted preferences are out of scope.

### Quick Edit Surface

- Default card view remains scan-first. Title, owner, deliverable count/title, OKR, Gold/base price, quality, and lane/status remain readable without opening an editor.
- Quick edit appears only on intent: an icon button with tooltip `카드 편집`, an expanded card area, or a compact popover anchored to the card.
- Prefer the existing inline/edit-area pattern from `KanbanBoard` and `InlineEditor`. Do not navigate to a deep detail page for Phase 50 edits.
- Editing controls must not cause neighboring lanes to resize. Expanding one card may increase that card's height, but lane width and toolbar height stay stable.
- Editable fields: `제목`, `리스트`, `산출물`, `기준가`, `품질`, `OKR`.
- Field-level saves must be independent enough that one failed field does not visually imply all fields failed or succeeded.
- Use compact inputs: text input or `InlineEditor` for title, select for lane/status and quality, number input for base price, popover/select for OKR, small text/select pair for deliverable title/type when available.

### Feedback And Validation

- Save feedback appears on the edited card, not only in a global toast.
- Pending feedback: show `저장중` beside the changed field/card with subdued motion or a small status dot.
- Success feedback: show `저장됨` briefly and update the relevant badge/summary immediately.
- Failure feedback: show `저장 실패` and a `다시 시도` button near the failed field/card. Preserve the user's draft until retry or cancel.
- Required validation messages:
  - Empty title: `제목을 입력해 주세요.`
  - Invalid base price: `기준가는 0 이상의 숫자여야 합니다.`
  - Missing deliverable title when deliverable is required: `필수 산출물 제목을 입력해 주세요.`
  - Failed OKR update: `OKR 연결을 저장하지 못했습니다. 다시 시도해 주세요.`

### Accessibility And Keyboard

- Every icon-only edit, clear, retry, and filter control must have an accessible Korean label or tooltip.
- Filter chips are real buttons with `aria-pressed`.
- Sort uses a labeled select or shadcn select with label `정렬`.
- Search input label is `업무 카드 검색`.
- Card quick edit supports keyboard entry: Enter commits single-line title edits; Escape cancels local drafts where applicable; Tab order stays within the expanded card before returning to lane controls.
- Drag/drop remains available, but lane/status select remains the keyboard-accessible equivalent.

### Layout Boundaries

- Keep the daily board as the primary surface. Phase 50 must not introduce One-Liner inbox review, mobile/native draft review, or Jarvis/wiki/graph/economy evidence panels.
- Existing side context panels may remain, but board controls and quick edit must be visually subordinate to the three lanes.
- Avoid nested cards inside cards. The quick-edit area should be a bordered/background subsection inside a card only when needed for controls, using `rounded-md border border-border bg-background/70 p-2`-style density.
- Use lucide icons for edit/search/sort/filter/reset/retry controls where icons are present.
