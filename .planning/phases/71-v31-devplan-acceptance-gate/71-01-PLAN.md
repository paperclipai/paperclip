# Phase 71 Plan 01: v3.1 DevPlan Acceptance Gate

<objective>
Implement a deterministic v3.1 acceptance gate that runs focused checks, consumes the DevPlan alignment matrix, reports the 64% baseline delta, separates blockers/accepted debt/future scope, and prevents final milestone closure when prerequisite evidence is missing or dirty.
</objective>

## Requirements

- GATE-01: Run focused tests/scans for DevPlan alignment, identity, daily cockpit, runtime, wiki/graph, and economy loop.
- GATE-02: Report baseline delta, blockers, accepted debt, and future scope with concrete file evidence.

## Tasks

1. Add v3.1 acceptance gate script.
   - Create `scripts/rt2-v31-acceptance-gate.mjs`.
   - Reuse `runDevPlanAlignmentGate` from `rt2-devplan-alignment-gate.mjs`.
   - Add focused command metadata, command execution, score delta calculation, required evidence existence checks, dirty prior evidence anchor detection, future scope reporting, and summary/report writing.

2. Add acceptance gate tests.
   - Create `scripts/rt2-v31-acceptance-gate.test.mjs`.
   - Inject fake command results and alignment summaries so tests do not run `pnpm`.
   - Assert pass path, failed focused check blocker, non-positive score delta blocker, dirty prior evidence blocker, and artifact writing.

3. Expose package scripts.
   - Add `rt2:v31-acceptance-gate`.
   - Add `test:v31-acceptance-gate`.

4. Update DevPlan alignment row.
   - Change `v31-acceptance-gate` from `missing` to `complete` with Phase 71 evidence anchors.
   - Update test expectations to 100% score and all rows complete.

5. Write validation, verification, and summary artifacts.
   - Validation maps `GATE-01` and `GATE-02` to script behavior and tests.
   - Verification records focused checks and any blocking dirty evidence state.
   - Summary records implementation and residual blocker/debt status.

## Verification

Run:

```sh
node scripts/rt2-v31-acceptance-gate.test.mjs
node scripts/rt2-devplan-alignment-gate.test.mjs
pnpm run test:v31-acceptance-gate
pnpm run rt2:v31-acceptance-gate
```

If the acceptance gate passes without dirty prior evidence blockers, run:

```sh
pnpm typecheck
pnpm test
```

Do not run `pnpm test:e2e` as the default Phase 71 gate.

<threat_model>
## Threat Model

- The final gate can create false confidence if it only reports a score. Mitigation: require focused checks, score delta, and blocker buckets.
- The final gate can overclaim engine parity. Mitigation: continue using the alignment gate's engine reference evidence rule.
- The final gate can pass on missing files. Mitigation: validate evidence paths.
- The final gate can hide dirty prerequisite evidence. Mitigation: surface dirty/untracked prior evidence anchors as blockers.
- The final gate can absorb future scope into the score. Mitigation: keep public marketplace, autonomous Jarvis direct apply, federation, native credentials, payroll/payment export, and store operations in future scope.
</threat_model>
