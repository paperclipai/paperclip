# Phase 41 Plan 01: Native and Mobile Capture Hardening

<objective>
Harden existing RT2 inbound capture and mobile knowledge search without adding a separate native app or live Slack/Teams dependency.
</objective>

<must_haves>
## Requirements
- CAP-01: Operator-visible capture source installation state, signing status, and last inbound event.
- CAP-02: Inbound draft review queue includes semantic context, duplicate warning, and source evidence.
- CAP-03: Promotion preserves source evidence into RT2 work/audit path and mobile search preserves semantic/lexical/citation signals.

## Decisions Covered
- D-01 through D-04: extend existing inbound draft source model with deterministic signing evidence.
- D-05 through D-08: enrich existing capture draft queue.
- D-09 through D-12: explicit promotion with source evidence and activity log continuity.
- D-13 through D-15: mobile-safe existing `KnowledgePage` search surface with citation targets.
- D-16 through D-18: deterministic verification without live external APIs.
</must_haves>

<implementation_steps>
1. Extend shared capture contracts and validators with capture source records, signing status, inbound event metadata, semantic context, duplicate warnings, and citation targets.
2. Add migration/schema support for capture source installation records and additional draft evidence columns.
3. Extend `rt2WorkBoardService` with capture source listing/upsert, deterministic signature checks, enriched draft creation, queue enrichment, and promotion audit metadata.
4. Add capture source status routes and preserve existing inbound draft route compatibility.
5. Extend UI API and One-Liner capture section to show source installation/signing/last-event evidence and enriched queue status.
6. Harden `KnowledgePage` search results for small viewports with routeable citation actions and explicit semantic + lexical fallback evidence.
7. Add route/service fallback and embedded route tests for signed/unsigned/duplicate/promotion evidence paths.
</implementation_steps>

<verification_plan>
- Run `pnpm typecheck`.
- Run `pnpm test`.
- If embedded Postgres skips on this Windows host, rely on deterministic fallback route tests and document the environment constraint.
</verification_plan>
