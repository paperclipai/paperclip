# Phase 13 Plan: Enterprise Rollout and RT2 Terminology

## 목표

`ENT-01`, `ENT-02`, `ENT-03`을 한 번에 닫는다. Phase 13은 v2.1의 정식 마지막 Phase이며 wave로 쪼개지 않는다.

## 작업

1. Shared/server 계약 추가
   - `Rt2EnterpriseRolloutOverview`, `Rt2EnterpriseRolloutSettingsInput`, `Rt2TemplateApplicationPreview` 타입 추가.
   - `GET/POST /companies/:companyId/rt2/enterprise/rollout` 추가.
   - enterprise tables migration 추가.

2. Template preview/apply 보강
   - preview가 `create/skip/error` 객체 목록과 summary를 반환하게 변경.
   - apply가 같은 객체 구조와 `createdId`를 반환하게 변경.
   - duplicate budget/routine은 irreversible apply 전에 `skip`으로 표시.

3. RT2 UI 연결
   - `EnterpriseRolloutPage` 추가.
   - Sidebar와 route에 `Rollout` 추가.
   - Plan Map의 enterprise 상태를 shipped로 갱신.

4. 검증
   - DB migration check.
   - shared/server/ui typecheck.
   - Phase 13 server route test.

## 완료 기준

- 관리자가 RT2 화면에서 SSO, template, access mode, policy default를 설정할 수 있다.
- template preview가 irreversible apply 전 생성/스킵/오류 객체를 보여준다.
- 제품-facing RT2 surface에서 enterprise rollout이 Paperclip-first label 없이 노출된다.
