# Phase 5 Plan: Knowledge Core Projection

<objective>
Create a replay-safe RT2 knowledge projector that materializes cumulative wiki pages and persisted graph rows from RT2 domain events.
</objective>

<scope>
- Add cumulative wiki DB schema and migration.
- Add Drizzle schema exports for existing graph tables.
- Add shared RT2 knowledge contracts.
- Implement server-side knowledge projector service.
- Wire projector execution into RT2 domain event append flow.
- Add company-scoped knowledge inspection routes.
- Add focused tests and verification.
</scope>

<tasks>
1. Add `rt2_v33_wiki_pages` schema and migration for company-scoped cumulative markdown pages.
2. Add `rt2_v33_project_graph_projection` schema exports matching existing graph migrations.
3. Add shared `rt2-knowledge` types and validators.
4. Implement `rt2KnowledgeProjectorService` with:
   - `projectEvent(eventId)`,
   - `projectAll(companyId)`,
   - `listWikiPages(companyId, filters)`,
   - `getWikiPage(companyId, pageKey)`.
5. Generate wiki pages:
   - `index.md`,
   - `log.md`,
   - topic pages for project, task, todo, deliverable, actor, and event entity anchors.
6. Generate graph rows:
   - project/task/todo/deliverable/actor/event nodes,
   - extracted edges with event evidence,
   - project graph report summary.
7. Update `rt2DomainEventService.appendAndProject` to run `rt2.knowledge_core` after the activity/live bridge.
8. Add `rt2-knowledge` routes and register them in the app.
9. Add focused tests for projection, idempotency, provenance, and routes.
</tasks>

<verification>
- `pnpm exec vitest run packages/shared/src/rt2-knowledge.test.ts server/src/__tests__/rt2-knowledge-projector.test.ts server/src/__tests__/rt2-knowledge-routes.test.ts`
- `pnpm -r typecheck`
- If stable, `pnpm build`
</verification>

<threat_model>
- Company data leakage: all queries and projections must filter by company id.
- Inferred edge misuse: first pass should only emit `EXTRACTED` edges unless confidence/evidence is explicit.
- Duplicate projection side effects: all projection writes must be upserts or protected by unique keys and `processEvent`.
- High-cost rebuild path: rebuild route must be explicit and company-scoped.
</threat_model>
