# Phase 60 Plan 01: Signing And Notarization Evidence Gate

<objective>
Add a deterministic native signing/trust evidence gate for RealTycoon2 that validates macOS and Windows release artifact evidence, blocks incomplete signing/notarization/trust paths, and writes operator-readable evidence without committing secrets or introducing native dependency churn.
</objective>

<requirements>
- `DIST-02`: 운영자는 macOS release artifact가 Developer ID signing, hardened runtime, notarization submission, ticket stapling, Gatekeeper verification 상태를 통과했는지 pipeline에서 볼 수 있다.
- `DIST-03`: 운영자는 Windows MSIX/installer artifact가 Store re-signing, Azure Artifact Signing, OV/EV certificate, timestamping 중 선택된 trust path로 서명되고 install trust evidence를 남기는 것을 확인할 수 있다.
</requirements>

<must_haves>
## Requirements
- DIST-02: macOS Developer ID, hardened runtime, notarization, stapling, and Gatekeeper evidence is represented in the gate output and blocks on missing/failed status.
- DIST-03: Windows installer/MSIX trust path, signing, timestamping, signature verification, and install trust evidence is represented in the gate output and blocks on missing/failed status.

## Decisions Covered
- D-01 through D-03: deterministic evidence-first gate, local/CI fixture mode, fail closed on missing or failed evidence.
- D-04 through D-06: macOS evidence fields and independent trust checks.
- D-07 through D-09: Windows trust path, signer, timestamp, and install trust fields.
- D-10 through D-12: `.planning/native-signing-runs` output, stable blockers, secret redaction/rejection.
- D-13 through D-15: focused script/test first and narrow release docs integration.
- D-16 through D-17: v2.9 regression protection and no default Playwright E2E.
</must_haves>

<truths>
- Phase 60 is a release evidence gate. It does not need to add `apps/desktop`, Tauri dependencies, native build outputs, or `pnpm-lock.yaml` changes.
- The gate must be useful before real Apple/Windows credentials exist by validating manifests and referenced evidence files.
- Passing code signing alone is not sufficient on macOS; notarization, stapling, and Gatekeeper evidence remain required.
- Passing Windows signing alone is not sufficient; timestamping and install trust evidence remain required.
- Existing dirty worktree changes must be preserved. Stage/commit only Phase 60 files.
</truths>
