<threat_model>
- **T-57-01 Cross-company evidence leakage:** Filter/report endpoints expose source and draft evidence. Mitigation: keep authenticated company access checks and use company-scoped service queries.
- **T-57-02 Secret or raw payload exposure:** Reliability report and promoted evidence could leak signing secrets/provider payloads. Mitigation: report only redacted source evidence, labels, counts, event ids, and durable draft/revision ids.
- **T-57-03 Misleading operational metrics:** Retry/failure/latency counts could overstate reliability problems. Mitigation: separate duplicate from failure, count retry only from durable evidence, and use null latency when no promoted drafts exist.
- **T-57-04 Review state confusion:** New filters could hide actionable drafts. Mitigation: default unfiltered view remains current behavior, active filters show clear Korean empty text, and tests cover evidence filters.
- **T-57-05 Route shadowing:** Report route could be captured by `:draftId`. Mitigation: register static report route before draft detail route and test it.
</threat_model>
