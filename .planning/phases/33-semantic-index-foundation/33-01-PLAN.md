# Phase 33 Plan 01: Semantic Index Foundation

<objective>
Add a company-scoped semantic index foundation over existing daily wiki, graph, and work artifact evidence without replacing source storage.
</objective>

<must_haves>
- Cover SEM-01: semantic index is an additive layer; existing wiki/graph/projector/search tables remain source evidence.
- Cover SEM-02: chunks preserve source type, source ID, source key, source freshness, timestamps, and provenance.
- Cover SEM-03: deterministic local fallback embeddings pass tests without provider credentials or pgvector.
- Cover SEM-04: operator can trigger full or changed-source reindex and inspect run status/counts.
- Preserve company boundary on all reads/writes and route access checks.
</must_haves>

<tasks>

## 1. Add semantic index schema
- Create `rt2_v33_semantic_index_chunks` for source chunks and embeddings.
- Create `rt2_v33_semantic_index_runs` for inspectable reindex status.
- Export both tables from `packages/db/src/schema/index.ts`.
- Add migration `0081_rt2_v33_semantic_index.sql`.

## 2. Add indexing service
- Implement `rt2SemanticIndexService(db, options?)`.
- Collect sources from `rt2_v33_daily_wiki_pages`, `rt2_v33_graph_nodes`, `rt2_v33_graph_edges`, and `issue_work_products`.
- Chunk content deterministically and hash content.
- Use injected provider when supplied, otherwise deterministic fallback.
- Upsert unchanged chunks as skipped and changed/new chunks as refreshed.
- Record run status, provider mode, source count, refreshed/skipped counts, and errors.

## 3. Add operator API
- Add `GET /companies/:companyId/rt2/semantic-index/status`.
- Add `POST /companies/:companyId/rt2/semantic-index/reindex`.
- Accept `mode=full|changed`, defaulting to `changed`.
- Use `assertCompanyAccess`.

## 4. Add focused verification
- Pure test fallback determinism.
- Embedded Postgres test daily wiki/work/graph indexing and company scoping.
- Route/service-level behavior can be proven by service tests if route harness setup is too heavy.

</tasks>

<verification>
- `pnpm --filter @paperclipai/server test -- rt2-semantic-index`
- `pnpm typecheck`
- `pnpm test`
</verification>

<threat_model>
- **Company data leakage:** Blocked by `companyId` filters in source collection and `assertCompanyAccess` on routes.
- **Provider dependency outage:** Mitigated by deterministic fallback when no provider is configured.
- **Derived index treated as truth:** Mitigated by source/provenance metadata and additive schema; source tables remain authoritative.
- **Stale embeddings:** Mitigated by content hash, sourceUpdatedAt, freshness, and reindex run status.
</threat_model>
