# Phase 35 Plan 01: Contradiction Review Workflow

<objective>
Create reviewable contradiction candidates from deterministic knowledge conflict signals and let operators resolve them with audit evidence.
</objective>

<must_haves>
- Generate candidates from new wiki/graph facts and deterministic lint output.
- Store raw evidence and deterministic reason codes even when provider explanation is absent.
- Support operator decisions: false positive, accept newer evidence, keep older evidence, request follow-up work.
- Write resolution/audit evidence and reflect open/resolved state in semantic search freshness.
</must_haves>

<tasks>

## 1. Add contradiction persistence
- Add candidate and resolution tables.
- Export schema and add migration.

## 2. Add contradiction service
- Generate candidates from `rt2WikiLintService` `embedding_consistency`.
- List open/resolved candidates by company/project.
- Resolve candidates and write resolution records.
- Mark semantic chunks stale for open candidates and fresh after resolution.

## 3. Add API and shared contracts
- Add validators and shared types.
- Add routes for list, generate, and resolve.
- Register routes in server app.

## 4. Add operator UI
- Add API client methods and query keys.
- Add contradiction review block to Knowledge Bridge.
- Show raw evidence/reason code and four resolution actions.

## 5. Verify
- Run package typechecks.
- Run focused lint/semantic/search tests.
- Run full unit suite.

</tasks>

<verification>
- `pnpm --filter @paperclipai/db typecheck`

- `pnpm --filter @paperclipai/server typecheck`
- `pnpm --filter @paperclipai/ui typecheck`

</verification>

<threat_model>
- **Silent knowledge rewrite:** Blocked by review-only resolution; no source wiki/graph rows are rewritten.
- **Untraceable AI explanation:** Provider explanation is optional metadata; raw evidence and deterministic reason code remain mandatory.
- **Company leakage:** Blocked by company-scoped route access and company filters.
- **Search hiding unresolved conflicts:** Open candidates mark related semantic chunks stale.
</threat_model>
