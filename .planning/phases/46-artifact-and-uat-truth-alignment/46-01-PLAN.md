# Phase 46 Plan 01: Artifact and UAT Truth Alignment

## Objective

Make the repo-local milestone artifact gate reject stale validation metadata, report legacy UAT closure consistently, and enforce v2.7 requirement-to-verification traceability.

## Must Haves

- Extend `scripts/rt2-milestone-artifact-gate.mjs` for v2.7 active phases without breaking the existing v2.6 historical checks.
- Require current milestone completed phases to have `*-VALIDATION.md` YAML frontmatter with matching phase/status/requirements.
- Treat `.planning/phases/43-validation-debt-and-milestone-gate-closure/43-LEGACY-UAT-CLOSURE.md` as canonical legacy UAT truth and fail on unqualified `unknown`.
- Verify each completed v2.7 requirement has one traceability row and appears in exactly one matching phase verification artifact.
- Add fixture tests for stale validation metadata, legacy UAT conflict, duplicate traceability, and passing v2.7 artifacts.
- Write Phase 46 verification/validation artifacts and update `ART-01` through `ART-03` only after evidence exists.

## Tasks

1. Update the milestone gate.
   - Add v2.7 phase definitions for 44-47.
   - Parse validation frontmatter fields.
   - Check completed requirements only, allowing Phase 47 to remain pending during Phase 46.
   - Add legacy UAT closure checks against the Phase 43 artifact.
   - Add verification artifact requirement checks for completed requirements.

2. Extend fixture tests.
   - Keep tests pure and temp-dir based.
   - Cover passing v2.7 fixture, stale validation frontmatter, unqualified legacy UAT unknown, and duplicate requirement traceability.

3. Normalize v2.7 validation artifacts.
   - Add YAML frontmatter to Phase 44 and Phase 45 validation files.
   - Create Phase 46 verification and validation files after implementation evidence exists.

4. Update requirements truth.
   - Mark `ART-01`, `ART-02`, and `ART-03` complete after the gate and tests pass.
   - Keep `CONF-01` and `CONF-02` pending for Phase 47.

5. Verify.
   - Run `pnpm test:milestone-gate`.
   - Run `pnpm typecheck`.
   - Run `pnpm rt2:milestone-gate`.

## Acceptance Criteria

- `pnpm test:milestone-gate` passes.
- `pnpm rt2:milestone-gate` fails on stale validation frontmatter in fixtures and passes on the current v2.7 completed scope.
- Legacy UAT status is reported as classified closure, not active unknown.
- `ART-01`, `ART-02`, and `ART-03` are backed by Phase 46 verification evidence.

## Threat Model

- **Artifact false positive:** The gate could pass while metadata is stale. Mitigation: require machine-readable validation frontmatter and verification artifact references.
- **Historical artifact mutation:** Old UAT files could be edited to hide debt. Mitigation: keep Phase 43 closure as canonical and treat legacy files as snapshots.
- **Traceability drift:** Requirements could be duplicated across phases. Mitigation: fail duplicate traceability rows and multiple verification anchors.
