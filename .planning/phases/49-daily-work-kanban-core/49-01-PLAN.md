<objective>
Make the RealTycoon2 daily work kanban board the first operational work surface, with persisted To-Do/Doing/Done movement and card-front metadata required by BOARD-01 through BOARD-03.
</objective>

<context>
Read `49-CONTEXT.md` before implementation. The locked decisions are:

- The company default route moves from `one-liner` to a daily work board route.
- The canonical lanes are `todo`, `doing`, `done`, displayed as `할 일`, `진행 중`, `완료`.
- Existing `today/support_1/support_2` persisted values must be mapped safely.
- Card fronts must show Task/To-Do type, owner, due date if present, OKR/KPI, deliverable, price/gold, and quality state.
- Quick edit, filters/sort/search, capture draft review, and detailed evidence panels are out of scope.
</context>

<tasks>

## 1. Normalize Daily Lane Contract

- Update `Rt2DailyLane` in shared types to `todo | doing | done`.
- Update `rt2DailyLaneSchema`.
- Update shared tests to accept the new lane values and reject old/non-canonical values.
- Add server-side normalization for old persisted lane values so existing rows read safely during rollout.
- Update activity history parsing to recognize both old and new lane values, returning new canonical values.

## 2. Migrate Database Check Constraint

- Add migration `0103_rt2_daily_work_lanes.sql`.
- Update old rows:
  - `today` to `todo`
  - `support_1` to `doing`
  - `support_2` to `done`
- Drop/recreate `rt2_v33_daily_report_cards_lane_check` for `todo`, `doing`, `done`.
- Update Drizzle schema check to match.

## 3. Create Primary Daily Work Page

- Add `ui/src/pages/rt2/DailyWorkPage.tsx`.
- Use `authApi.getSession`, `projectsApi.list`, `rt2DailyReportApi.getBoard`, `rt2DailyReportApi.saveCard`, `queryKeys.rt2Daily`, and `calendarDateKey`.
- Pick the first active project by default and render project/date context.
- Render `Rt2DailyBoard`.
- Handle no company, no project, loading, and error states in Korean.

## 4. Update Routes And Navigation

- Add `daily-work` to `boardRoutes()` in `ui/src/App.tsx`.
- Change company root and route index redirects to `daily-work`.
- Add unprefixed `daily-work` redirect.
- Update sidebar to place `일일 업무` as the primary work item above dashboard/inbox.
- Add `daily-work` to company route roots.
- Update company page memory fallback from `/one-liner` to `/daily-work`.

## 5. Update Board UI

- Update `Rt2DailyBoard` lane labels to `할 일`, `진행 중`, `완료`.
- Show card-front metadata required by BOARD-03:
  - Task/To-Do type
  - owner/assignee
  - due date if available
  - OKR/KPI status
  - deliverable count
  - base price/gold impact
  - quality state
- Add Korean pending/saved/failed affordance based on `pendingTodoIssueId` and optional failed id.
- Keep save behavior through the existing `onSaveCard`.

## 6. Focused Tests

- Update `Rt2DailyBoard.test.tsx` to assert the new lanes and visible metadata.
- Update shared lane contract tests.
- Update daily report route test save payload expectations to use new lane values.
- Update company page memory tests for `/daily-work` fallback.

</tasks>
