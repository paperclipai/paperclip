# Phase 72 Plan: Public Marketplace Launch

## Goal
Transform the company-scoped marketplace into a public/open marketplace with approval workflow, public/private evidence contract separation, and evidence tier bucketing.

**Requirements:** MKT-01, MKT-02, MKT-03
**Success criteria:**
1. Public marketplace endpoints show only `approved` listings with public evidence contract (no raw gold amounts)
2. Listing creation defaults to `draft` status; approval workflow transitions to `pending_approval` ‚Üí `approved`/`rejected`
3. Public listing responses use evidence tier buckets (Bronze/Silver/Gold) instead of raw counts
4. Company-scoped routes expose full evidence including raw gold estimates
5. DevPlan alignment gate updated for public marketplace scope
6. Focused tests pass

## Task 4: Service ‚Äî Add approval workflow methods

**File:** `server/src/services/rt2-agent-marketplace.ts`

Add new methods:
- `submitForApproval(listingId: string, companyId: string): Promise<MarketplaceListing>` ‚Äî transitions `draft` ‚Üí `pending_approval`, sets `submittedAt`
- `approveListing(listingId: string): Promise<MarketplaceListing>` ‚Äî transitions `pending_approval` ‚Üí `approved`, sets `approvedAt`
- `rejectListing(listingId: string, reason: string): Promise<MarketplaceListing>` ‚Äî transitions `pending_approval` ‚Üí `rejected`, sets `rejectionReason`
- `getPendingApprovals(companyId: string): Promise<MarketplaceListing[]>` ‚Äî list pending_approval listings for a company (for review surface)
- `updateListingEvidenceTier` (private) ‚Äî derives evidence/reputation/quality tier buckets

**Evidence tier bucketing logic:**
```
EvidenceTier: Bronze (1-2 approved deliverables), Silver (3-5), Gold (6+)
ReputationTier: New (0 subscriptions), Established (1-10), Top Rated (11+)
QualityTier: Bronze (<50 avg), Silver (50-74), Gold (75+)
```
