# Phase 52 Plan 01: Supporting Surfaces and Identity Regression Gate

<objective>
Reposition Jarvis/wiki/graph/economy as compact support evidence on the daily work board and add a repeatable focused gate for RealTycoon2 Korean/product identity regressions.
</objective>

<requirements>
- SUPPORT-01: Operators can access Jarvis, wiki, graph, and economy as board-adjacent/card evidence.
- SUPPORT-02: Recommendations and citations appear as contextual support without obscuring card/deliverable work.
- SUPPORT-03: A focused verification path detects product-facing Paperclip naming, English defaults, and identity regressions.
</requirements>

<must_haves>
- Preserve `daily-work` as the primary surface. Do not create a new support dashboard.
- Keep support evidence compact and Korean-first.
- Use existing daily board/cockpit/card metadata before adding new backend contracts.
- Identity gate must avoid noisy whole-repo scans and target product-facing UI files.
- Do not modify `pnpm-lock.yaml`.
</must_haves>

<tasks>
1. Add board/card support evidence in `Rt2DailyBoard.tsx`.
   - Replace the existing plain right-side cockpit summary with a Korean `보조 근거` rail.
   - Show Jarvis recommendations from `board.cockpit.aiSummary`.
   - Show wiki/graph/economy evidence derived from trace rows, gap flags, deliverable/base price, quality, and OKR context.
   - Add compact per-card evidence rows for Jarvis/wiki/graph/economy without expanding card edit scope.

2. Extend `Rt2DailyBoard.test.tsx`.
   - Assert the board renders `보조 근거`, `Jarvis 추천`, `지식 근거`, `그래프 연결`, and `경제 근거`.
   - Assert card-level evidence remains contextual to the card.
   - Assert legacy product-facing terms are absent from the daily board render.

3. Add `scripts/rt2-identity-gate.mjs`.
   - Scan a focused default set of product-facing UI files.
   - Fail on product-facing `Paperclip`, `Paper Company`, `Multica`, and common English loading/empty support-surface defaults.
   - Print file, line, category, token, and guidance.
   - Support optional path args so tests and future maintainers can scan a temp fixture or narrowed path.

4. Add `scripts/rt2-identity-gate.test.mjs`.
   - Verify clean Korean RealTycoon2 copy passes.
   - Verify product-facing legacy naming fails.
   - Verify product-facing English default copy fails.

5. Add package script.
   - Add `rt2:identity-gate`.
   - Add `test:identity-gate`.
</tasks>

<verification>
- `pnpm exec vitest run ui/src/components/Rt2DailyBoard.test.tsx`
- `pnpm run test:identity-gate`
- `pnpm run rt2:identity-gate`
- `pnpm typecheck`
</verification>

<threat_model>
This phase is mostly read-only UI and local verification script work. Primary risk is false-positive or false-negative identity checks. Mitigation: default scan is focused to product-facing UI paths, script supports explicit path args, and tests cover allowed/pass and fail cases. No new auth, network, payment, or write-side mutation path is introduced.
</threat_model>
