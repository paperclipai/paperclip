# Phase 44 Plan 01: Release Host Verification Harness

<objective>
Build a deterministic release-host verification harness that runs the project typecheck and stable Vitest slices with durable timeout/failure evidence, owner classification, and failed-slice rerun support.
</objective>

<requirements>
- `REL-01`: Operator can run a documented release-host verification command that executes `pnpm typecheck` and `pnpm test` with stable timeout handling.
- `REL-02`: Operator can inspect failed or timed-out release-host verification runs with phase, suite, duration, and owner classification.
- `REL-03`: Operator can rerun only the failed release verification slice without losing the full-suite audit trail.
</requirements>

<must_haves>
## Requirements
- REL-01: A documented package script runs release-host verification through a stable command path.
- REL-02: Failure/timeout reports include suite, duration, owner, and retry recommendation.
- REL-03: Failed/timed-out slices can be rerun from prior `summary.json` while preserving prior attempts.

## Decisions Covered
- D-01 through D-03: release-host wrapper and durable evidence output.
- D-04 through D-06: timeout/failure classification fields and configurable timeout.
- D-07 through D-09: failed-slice rerun with stable slice identity.
- D-10 through D-14: reuse existing stable runner concepts, keep embedded Postgres to Phase 45, and verify with focused tests plus typecheck/test.
</must_haves>

<truths>
- The release-host command must be deterministic and local.
- Browser E2E and release-smoke suites remain separate commands.
- Embedded Postgres Windows host-readiness is recorded as evidence but not fixed in Phase 44.
- Rerun mode must append attempts to the existing audit trail instead of replacing the original full-suite evidence.
</truths>

<threat_model>
| ID | Threat | Mitigation |
|----|--------|------------|
| T-44-01 | Harness hides failing output behind a generic timeout. | Per-slice stdout/stderr logs, timeout status, owner, and retry recommendation are recorded in `summary.json` and `report.md`. |
| T-44-02 | Rerun destroys the original full-suite audit trail. | Rerun mode appends attempts to the existing summary/report and keeps prior attempts intact. |
| T-44-03 | Release gate accidentally expands into browser or provider-dependent checks. | Default slices remain typecheck plus stable Vitest; browser E2E and live providers stay separate. |
| T-44-04 | Phase 44 claims embedded Postgres coverage that still skips on Windows. | Embedded Postgres skip evidence is noted as Phase 45 scope, not marked fixed here. |
</threat_model>
