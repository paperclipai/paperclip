# Phase 45 Plan 01: Embedded Postgres Runtime Coverage

<objective>
Make embedded Postgres route/persistence coverage visible on Windows release hosts by adding structured skip evidence, a focused host-ready command, and release-host accepted-debt classification.
</objective>

<scope>
- Extend embedded Postgres test support evidence without changing the default Windows skip policy.
- Add a focused host-ready command that opts into embedded Postgres DB and RT2 route persistence suites.
- Extend release-host verification output so default Windows embedded Postgres skips are classified as accepted debt, while focused failures remain blockers.
- Add deterministic script/helper tests that do not require starting embedded Postgres.
</scope>

<tasks>

1. Add structured embedded Postgres host evidence.
   - Update `packages/db/src/test-embedded-postgres.ts` with reason codes for supported, explicit opt-out, Windows default disabled, and startup failure.
   - Export a helper that converts support probe state into machine-readable evidence including platform, env controls, reason code, and affected suite ids.
   - Preserve existing `getEmbeddedPostgresTestSupport()` behavior for current tests.

2. Add focused Windows host-ready command.
   - Add `scripts/rt2-embedded-postgres-host-ready.mjs`.
   - Run a narrow suite set with `PAPERCLIP_ENABLE_EMBEDDED_POSTGRES_TESTS=true`: DB migration/client coverage, RT2 task persistence, RT2 daily report persistence, RT2 task routes, and RT2 daily report routes.
   - Add a dry-run/exported command builder for deterministic tests.
   - Expose the command through `package.json`.

3. Extend release-host confidence classification.
   - Update `scripts/rt2-release-host-verify.mjs` to classify default Windows embedded Postgres skip as an `accepted_debt` synthetic attempt when the focused host-ready slice is not requested.
   - Add an opt-in flag to include the focused host-ready slice in release-host verification.
   - Ensure focused failure remains `failed`/`timeout`/`error`, not accepted debt.
   - Update report/status handling and exit semantics so `accepted_debt` is visible but not treated as a hidden pass.

4. Add focused tests.
   - Extend `scripts/rt2-release-host-verify.test.mjs` for accepted-debt classification and focused slice selection.
   - Add or extend script tests for the host-ready command builder without starting embedded Postgres.

5. Verify.
   - Run `pnpm test:release-host-verify`.
   - Run the new host-ready script in dry-run mode.
   - Run `pnpm typecheck`.
   - Run `pnpm test` if feasible; otherwise preserve release-host evidence and explain the skipped or timed-out full-suite status.

</tasks>

<must_haves>
- D-01 through D-04: skip evidence must be structured while preserving Windows default skip.
- D-05 through D-08: host-ready path must be repo-owned, opt-in, isolated, and blocker-producing on focused failure.
- D-09 through D-12: route-level persistence coverage must include a real embedded Postgres RT2 route flow.
- D-13 through D-16: release-host output must distinguish accepted debt from focused blockers.
- D-17 through D-19: deterministic tests must cover classification without requiring embedded Postgres startup.
</must_haves>

<verification>
- `pnpm test:release-host-verify`
- `pnpm rt2:embedded-postgres-host-ready --dry-run`
- `pnpm typecheck`
- `pnpm test` when host/time budget permits
</verification>
