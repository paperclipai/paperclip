# Phase 47 Plan 01: Runtime Confidence Operations Surface

## Objective

Create a repo-owned generated report that lets an operator inspect release confidence, accepted debt, blockers, deferred scope, and latest v2.7 verification evidence in one place.

## Must Haves

- Add `scripts/rt2-runtime-confidence.mjs` with JSON and Markdown output.
- Aggregate the latest release-host summary and current milestone artifact gate result.
- Normalize statuses into `blocker`, `accepted_debt`, `deferred_scope`, `passed`, and `pending`.
- Include v2.7 requirement evidence for REL, PG, ART, and CONF requirements.
- Surface exact next commands for blockers and accepted debt.
- Add focused fixture tests without running the full workspace suite.
- Mark `CONF-01` and `CONF-02` complete only after report generation and verification evidence exist.

## Tasks

1. Implement the runtime confidence script.
   - Discover latest `.planning/release-host-runs/*/summary.json`.
   - Accept `--release-host-summary`, `--output-dir`, `--root`, and `--json`.
   - Call `checkPlanningArtifacts(root)` for artifact gate truth.
   - Build normalized blocker, accepted-debt, pending, passed, and deferred-scope entries.
   - Write `summary.json` and `report.md`.

2. Add focused tests.
   - Build temp fixture roots with requirements, phase artifacts, and release-host summaries.
   - Cover all-passed, accepted-debt, missing release-host evidence, artifact-gate blocker, and pending requirement states.

3. Expose and document the command.
   - Add `rt2:runtime-confidence` and `test:runtime-confidence` scripts.
   - Document command usage and report interpretation in `doc/RELEASE-HOST-VERIFICATION.md`.

4. Write Phase 47 close artifacts.
   - Add summary, verification, and validation files with frontmatter.
   - Update `CONF-01` and `CONF-02` in `.planning/REQUIREMENTS.md` after checks pass.

5. Verify.
   - Run `pnpm test:runtime-confidence`.
   - Run `pnpm typecheck`.
   - Run `pnpm rt2:milestone-gate -- --json`.
   - Run `pnpm rt2:runtime-confidence -- --release-host-summary <fixture-or-latest>`.

## Acceptance Criteria

- Runtime confidence report shows release-host status, accepted debt, blockers, deferred scope, and latest verification evidence.
- Milestone-gate blockers and runtime confidence blockers use the same normalized taxonomy.
- Operators see exact commands for accepted debt closure and blocker recovery.
- `CONF-01` and `CONF-02` are verified by Phase 47 artifacts.

## Threat Model

- **False confidence:** A missing release-host run could be mistaken for pass. Mitigation: missing release-host summary is a blocker.
- **Taxonomy drift:** Artifact gate and confidence report could disagree. Mitigation: consume `checkPlanningArtifacts(root)` directly and normalize unknown gate issues as blockers.
- **Hidden debt:** Accepted debt could lack closure action. Mitigation: accepted debt entries require owner, reason, and next command; otherwise report as blocker.
