# Phase 69 Plan 01: Graphify v3 Corpus Graph Sidecar

<objective>
Implement a company-scoped Graphify v3-style corpus graph sidecar with source SHA cache, deterministic extraction/provenance, graph query APIs, clustering fallback, corpus-vs-product report, focused tests, and DevPlan alignment closure.
</objective>

## Requirements

- GRAPH-01: SHA256 file cache and source location metadata for repo/docs/wiki source files.
- GRAPH-02: Code/docs extraction interface, confidence score, relation provenance.
- GRAPH-03: Node, neighbors, community, shortest path, god nodes, graph stats APIs with explicit clustering algorithm/fallback.
- GRAPH-04: Graph report distinguishes product graph and corpus graph and exposes gaps, surprising connections, suggested questions.

## Tasks

1. Add corpus graph persistence.
   - Create Drizzle schema for corpus graph sources, nodes, edges, communities, and reports.
   - Add migration `0106_rt2_corpus_graph_sidecar.sql`.
   - Export schema from DB package.

2. Add shared contracts and validators.
   - Extend `rt2-graph` shared types with corpus graph source/node/edge/community/report/query shapes.
   - Add ingest/query validators.
   - Add focused shared tests.

3. Add server service.
   - Implement SHA256 ingest with skip-on-unchanged behavior.
   - Implement deterministic markdown/code/term extraction.
   - Persist provenance-backed nodes and edges.
   - Implement connected-components fallback clustering, centrality/god nodes, stats, node, neighbors, community, shortest path, and report methods.

4. Add routes.
   - Mount `/companies/:companyId/rt2/corpus-graph/ingest`.
   - Mount query endpoints for stats, report, node, neighbors, community, shortest path, and god nodes.
   - Enforce company access.

5. Add focused tests.
   - Service test proves ingest/cache/provenance/clustering/path/report.
   - Route test proves endpoint contract.
   - Alignment gate test proves Graphify row can become complete only with engine reference evidence.

6. Update DevPlan truth.
   - Mark `graphify-v3-sidecar` complete with concrete schema/shared/service/route/test/audit evidence after tests pass.
   - Update Phase 69 artifacts and planning state after verification.

<threat_model>
## Threat Model

- Source paths and URLs are untrusted input. Normalize source keys, store metadata as data, and never read arbitrary filesystem paths from route input.
- Corpus content may be large. Keep route payload bounded by existing Express body limit and make extraction deterministic.
- Graph provenance can expose source snippets. Tests use synthetic content; production callers should avoid sending secrets into corpus ingest.
- Company access is mandatory for every route.
</threat_model>
