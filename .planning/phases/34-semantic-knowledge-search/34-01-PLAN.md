# Phase 34 Plan 01: Semantic Knowledge Search

<objective>
Upgrade RT2 knowledge search so operators can search company-scoped wiki, graph, work artifact, and deliverable evidence with semantic ranking plus lexical fallback.
</objective>

<must_haves>
- Cover SEARCH-01 by searching daily wiki, graph evidence, work artifacts/deliverables, and legacy knowledge from a single company-scoped route.
- Cover SEARCH-02 by returning source type, source key, snippet, score evidence, freshness, confidence, provenance, and updated/source date.
- Cover SEARCH-03 by supporting project/work object, source type, date range, confidence, and contradiction-status filters in the API and UI.
- Cover SEARCH-04 by using deterministic JSON-vector similarity and lexical fallback without requiring pgvector or a provider.
- Preserve `assertCompanyAccess` and company filters on every search path.
</must_haves>

<tasks>

## 1. Upgrade backend search service
- Read Phase 33 semantic chunks for the current company.
- Embed the query with deterministic fallback.
- Score chunks with cosine similarity over stored JSON vectors.
- Merge semantic results with lexical fallback results from source tables.
- Deduplicate by source type/source ID and expose evidence labels honestly.

## 2. Extend search route contract
- Add query params for `sourceType`, `projectId`, `workObjectId`, `dateFrom`, `dateTo`, `confidence`, and `contradictionStatus`.
- Preserve existing `q`, `limit`, `offset`, and legacy `type` behavior.
- Keep missing query validation as a 400.

## 3. Add operator UI search surface
- Add an API client for RT2 search and semantic index status/reindex.
- Add query keys for semantic search and index status.
- Add a `Search` tab to `KnowledgePage`.
- Render filters, index status, reindex action, and evidence-forward result cards.

## 4. Verification
- Update legacy Phase 6 search expectation from fake `semantic-rerank` to honest `lexical-fallback`.
- Run focused semantic/search tests.
- Run repository typecheck and unit test suite.

</tasks>

<verification>
- `pnpm typecheck`
- `pnpm test -- rt2-semantic-index rt2-phase6-intelligence`
- `pnpm test`
</verification>

<threat_model>
- **Company data leakage:** Blocked by existing route access checks plus company ID filters on semantic chunks and fallback source queries.
- **Provider or pgvector dependency:** Mitigated by deterministic local embedding and JSON-vector similarity.
- **False semantic claims:** Mitigated by evidence labels that distinguish `semantic-index` from `lexical-fallback`.
- **Phase scope creep into contradiction review:** Mitigated by exposing placeholder/filter contract only; candidate generation remains Phase 35.
</threat_model>
