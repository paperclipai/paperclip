**Status:** Complete

2. **서버 API 확장**
   - quality review queue와 approve/reject action 추가
   - Auto policy preview와 evaluation policy decision 응답 추가
   - expected deliverable 기반 reverse-design task proposal 추가
   - governed skill capability list/create 추가

3. **DB migration 보강**
   - 기존 Drizzle schema에 있던 advanced AI runtime tables를 migration으로 동기화
   - migration journal numbering 확인

4. **UI 운영 표면 연결**
   - `Rt2QualityPanel`에 Jarvis manager review queue와 action 추가
   - `Rt2GovernancePanel`에 Jarvis runtime skill capability tab 추가

5. **검증**
   - shared/server/ui typecheck
   - DB migration numbering check
   - 서버 route regression test

## 완료 기준

- 관리자는 품질평가 evidence, delta, approval action을 볼 수 있다.
- Auto action은 policy band를 설명하고 band 밖이면 Co-Pilot으로 라우팅된다.
- Jarvis는 expected deliverable에서 task proposal과 rationale을 반환한다.
- runtime skill injection은 approval-linked capability로 노출된다.
