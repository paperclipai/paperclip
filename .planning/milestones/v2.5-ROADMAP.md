# Milestone v2.5: Semantic Knowledge Intelligence

**Status:** SHIPPED 2026-04-29
**Phases:** 33-38
**Total Plans:** 6
**Requirements:** 19/19 complete
**Audit:** initial `gaps_found`, closure verification passed via Phase 38

## Overview

Daily wiki/graph knowledge was upgraded into an RT2 semantic knowledge loop: semantic indexing, semantic + lexical knowledge search, contradiction review, Jarvis grounded answers, and operator-facing knowledge health. Phase 38 closed the milestone audit artifact gaps before archive.

## Phases

### Phase 33: Semantic Index Foundation

**Goal:** Existing wiki/graph/work evidence를 company-scoped semantic index에 안전하게 적재한다.
**Requirements:** SEM-01, SEM-02, SEM-03, SEM-04
**Plans:** 1

Plans:

- [x] 33-01: Add company-scoped semantic index storage, deterministic fallback embedding, reindex service, status/reindex API, and tests.

**Delivered:**

- Additive semantic index storage in `rt2_v33_semantic_index_chunks`.
- Inspectable run tracking in `rt2_v33_semantic_index_runs`.
- Deterministic local fallback embedding.
- Company-scoped indexing for daily wiki pages, graph nodes, graph edges, and work artifacts.
- Full/changed reindex behavior with refreshed/skipped counts and last-run status.
- `GET /companies/:companyId/rt2/semantic-index/status`
- `POST /companies/:companyId/rt2/semantic-index/reindex`

### Phase 34: Semantic Knowledge Search

**Goal:** Operator가 RT2 knowledge를 semantic + lexical fallback으로 탐색한다.
**Requirements:** SEARCH-01, SEARCH-02, SEARCH-03, SEARCH-04
**Plans:** 1

Plans:

- [x] 34-01: Combine semantic chunks with lexical fallback and expose searchable metadata/filter UI.

**Delivered:**

- RT2 search combines Phase 33 semantic chunks with lexical fallback.
- Deterministic JSON-vector query similarity without pgvector.
- Source metadata, freshness, confidence, contradiction status, provenance, and evidence labels.
- `/companies/:companyId/rt2/search` filters for project/work object, source type, date range, confidence, and contradiction status.
- `KnowledgePage` Search tab with query, filters, semantic index status, reindex action, and evidence-forward result cards.

### Phase 35: Contradiction Review Workflow

**Goal:** 새 evidence와 기존 knowledge 사이의 충돌 후보를 review/audit loop로 만든다.
**Requirements:** CONTRA-01, CONTRA-02, CONTRA-03, CONTRA-04
**Plans:** 1

Plans:

- [x] 35-01: Add contradiction candidate generation, resolution workflow, audit trail, and UI.

**Delivered:**

- Additive contradiction candidate and resolution storage.
- Candidate generation from deterministic `embedding_consistency` wiki lint output.
- List/generate/resolve API under `/companies/:companyId/rt2/contradictions`.
- Resolution decisions: false positive, accept newer, keep older, request follow-up.
- Activity-log audit writes on generation and resolution.
- Open/resolved contradiction state reflected in semantic chunk freshness.
- Knowledge Bridge contradiction review UI.

## Milestone Summary

**Key Accomplishments:**

- Added pgvector-ready semantic index storage with deterministic local fallback.
- Unified daily wiki, graph, work artifact, and deliverable evidence under company-scoped semantic search.
- Added contradiction candidate, resolution, and audit workflows without requiring live provider dependency.
- Grounded Jarvis advice with citations, stale evidence warnings, and unresolved contradiction warnings.
- Added operator-facing knowledge operations health gates.
- Closed initial audit gaps through Phase 38 artifact repair.

**Verification:**

- `pnpm typecheck` passed.
- `pnpm test` passed.
- Targeted RT2 semantic index, hybrid search, wiki lint, Jarvis grounding, and knowledge operations tests passed.
- Embedded Postgres route tests remain opt-in on Windows and were documented per phase.

**Issues Deferred:**

| Category | Item | Reason |
|----------|------|--------|
| federation | cross-company knowledge federation | trusted company ecosystem and sharing policy are not ready |
| autonomy | automatic knowledge rewrites without approval | contradiction review must remain approval-first first |
| provider | mandatory live provider dependency | local dev and CI must remain deterministic |
| mobile | native mobile semantic search UX | web operator loop remains the validated surface |

**Technical Debt:**

- Live provider-backed embedding and contradiction explanation are represented through injectable/optional interfaces but are not mandatory runtime dependencies.
- Windows embedded Postgres defaults still skip selected tests unless explicitly enabled.
