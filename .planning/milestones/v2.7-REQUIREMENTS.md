### Release Verification

- [x] **REL-01:** Operator can run a documented release-host verification command that executes `pnpm typecheck` and `pnpm test` with stable timeout handling.  
  **Outcome:** Validated in Phase 44. Release-host verification is documented and available through repo scripts, with stable timeout handling and generated evidence.

- [x] **REL-02:** Operator can inspect failed or timed-out release-host verification runs with phase, suite, duration, and owner classification.

- [x] **REL-03:** Operator can rerun only the failed release verification slice without losing the full-suite audit trail.

### Embedded Postgres

- [x] **PG-01:** Operator can run RT2 embedded Postgres persistence tests on Windows through a host-ready path instead of silent default skip.  
  **Outcome:** Validated in Phase 45. `pnpm rt2:embedded-postgres-host-ready` runs focused DB and RT2 route persistence suites with embedded Postgres enabled.

- [x] **PG-02:** Operator can see why any embedded Postgres suite was skipped, including host capability, env flag, and fallback evidence.  
  **Outcome:** Validated in Phase 45. Structured host evidence records support state, explicit opt-out, Windows default disabled status, startup failure, and env flags.

- [x] **PG-03:** Operator can verify route-level persistence flows that previously depended on embedded Postgres skip behavior.  
  **Outcome:** Validated in Phase 45. The host-ready path covers RT2 task and daily report route persistence suites.
