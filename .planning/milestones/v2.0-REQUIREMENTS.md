# Requirements Archive: RealTycoon2 v2.0 RT2 Refoundation

**Archived:** 2026-04-25  
**Milestone status:** Shipped  
**Source:** `.planning/REQUIREMENTS.md`

## 핵심 가치

회사 범위 work signal은 disconnected tool이나 Paperclip-shaped manual workflow를 강요하지 않고 logging → execution → knowledge accumulation → approval → economic feedback으로 이어져야 한다.

## 완료된 v1 요구사항

### Product Identity

- [x] **IDENT-01**: Operator는 Paperclip-first project tab이 아니라 One-Liner, Knowledge, Marketplace, P&L, Org, Governance가 main navigation인 RT2-branded primary shell로 앱에 진입할 수 있다.
- [x] **IDENT-02**: Operator는 company context를 잃지 않고 RT2 primary route에서 underlying Paperclip control-plane view로 이동할 수 있다.

### Triple Foundation

- [x] **FOUND-01**: Company-scoped work execution은 Paperclip issue route만 의존하지 않고 Multica-backed task lifecycle을 통해 실행될 수 있다.
- [x] **FOUND-02**: 모든 RT2 task와 deliverable은 orchestration context와 execution/runtime context를 함께 저장해 Paperclip과 Multica를 연결 상태로 유지한다.
- [x] **FOUND-03**: wikiLLM과 Graphify는 cumulative markdown page와 provenance-tagged graph entity를 위한 first-class persistence layer가 된다.

### Zero-Friction Logging

- [x] **LOG-01**: User는 multi-field form entry 없이 single freeform work input을 제출하고, 시스템은 이를 task, todo, deliverable, daily-log draft로 변환한다.
- [x] **LOG-02**: User는 web app 어디서든 global entry point를 통해 One-Liner를 실행할 수 있다.
- [x] **LOG-03**: 모든 One-Liner, task, deliverable write는 projection update 전에 append-only company-scoped event stream에 기록된다.

### CQRS and Knowledge

- [x] **CQRS-01**: Projection worker는 event에서 daily report, wiki page, graph view, search index, P&L read model을 idempotent하게 materialize한다.
- [x] **CQRS-02**: Projection worker는 중단 후 duplicate business effect나 data loss 없이 rebuild/resume할 수 있다.
- [x] **KNOW-01**: Daily work event는 task, deliverable, actor와 연결된 cumulative `index.md`, `log.md`, topic page로 compile된다.
- [x] **KNOW-02**: Graph view는 extracted relationship과 inferred relationship을 구분하고 new event에서 incrementally update된다.

### Jarvis, Quality, and Search

- [x] **JARV-01**: Jarvis는 empty stub이 아니라 live task, wiki, graph context에서 task advice, task breakdown, project insight를 반환한다.
- [x] **QUAL-01**: AI evaluation은 persisted evidence와 approval boundary를 가진 Shadow, Co-Pilot, Auto mode를 지원한다.
- [x] **QUAL-02**: Search는 hybrid keyword + semantic/reranked retrieval로 wiki, graph, task, deliverable을 검색한다.

### Economy, Collaboration, and Governance

- [x] **ECON-01**: 모든 task와 todo는 approval/reward logic에 들어가는 deliverable definition과 base price를 요구한다.
- [x] **ECON-02**: Personal/amoeba P&L dashboard는 approved deliverable event에서 ledger-backed revenue, cost, profit drilldown을 노출한다.
- [x] **MKT-01**: Marketplace listing은 live RT2 record의 agent skill, deliverable performance, reputation, pricing data를 재사용한다.
- [x] **COLLAB-01**: Collaboration reward와 reputation expansion은 UI-only stub data가 아니라 persisted cross-contribution evidence에서 계산된다.
- [x] **GOV-01**: 모든 RT2 mutation route는 company access, actor permission, activity/audit logging을 일관되게 강제한다.
