# Milestone v2.1: 개발기획서 반영 및 운영자 채택

**상태:** 완료, 2026-04-25  
**Phases:** 8-13  
**총 Plans:** 6  
**요구사항:** 24/24 완료

## 개요

v2.1은 업로드된 RealTycoon2 개발기획서를 기준으로 현재 앱의 gap을 보이는 checklist로 만들고, v2.0에서 만든 RT2 spine을 내부 운영자가 매일 사용할 수 있는 workflow로 끌어올린 마일스톤이다.

핵심 방향은 friction-zero capture, daily report cockpit, OKR/KPI traceability, Task Mesh, wiki/graph knowledge workspace, Jarvis rollout governance, enterprise rollout readiness였다.

## Phases

### Phase 8: 개발기획서 반영 기준선

**Goal:** 실행을 안내할 plan-to-product gap map과 adoption checklist를 명시적으로 유지한다.  
**Requirement:** ENT-04  
**Plans:** 1/1 완료

- [x] 08-01: 앱에서 보이는 개발기획서 adoption checklist

**완료 내용:**

- RT2 Plan Alignment page를 추가했다.
- capability를 shipped, partial, missing bucket으로 매핑했다.
- 각 gap을 v2.1 phase 또는 future scope와 연결했다.
- RT2 route, sidebar, command palette, company route normalization에 page를 등록했다.

### Phase 9: Friction-Zero Capture Surfaces

**Goal:** One-Liner 캡처를 web, shortcut, voice, messenger-style input에서 가능하게 하고 즉시 reward feedback을 보여준다.  
**Requirements:** CAP-01, CAP-02, CAP-03, CAP-04, CAP-05  
**Plans:** 1/1 완료

- [x] 09-01: friction-zero capture surfaces

**완료 내용:**

- global floating One-Liner widget을 추가했다.
- `c` shortcut과 shortcuts dialog 문서화를 연결했다.
- browser voice-to-draft와 unsupported/error fallback을 추가했다.
- Slack/Teams-style reviewed draft intake endpoint를 추가했다.
- commit 직후 created task, deliverable, proposed gold, XP, settlement state, reward rationale을 표시했다.

### Phase 10: Daily Report and OKR/KPI Cockpit

**Goal:** 일일보고와 6단계 OKR/KPI traceability를 주요 운영자 workflow로 만든다.  
**Requirements:** DAILY-01, DAILY-02, OKR-01, OKR-02, OKR-03  
**Plans:** 1/1 완료

- [x] 10-01: daily report and OKR/KPI cockpit

**완료 내용:**

- daily report API에 cockpit read model을 추가했다.
- daily card가 산출물, base price, 품질 상태, OKR 연결 상태, gap flags를 포함하게 했다.
- cockpit summary가 task, to-do, deliverable, notes, gold/XP, quality를 집계한다.
- task/to-do에서 Project와 goal parent path로 올라가는 trace row를 제공한다.
- `Rt2DailyBoard`를 왼쪽 context, 가운데 editor, 오른쪽 Jarvis/detail 패널로 재구성했다.

### Phase 11: Task Mesh and Knowledge Workspace

**Goal:** 개발기획서의 Task Mesh와 wiki/graph/Obsidian-ready 지식 workflow를 제공한다.  
**Requirements:** MESH-01, MESH-02, KNOW-01, KNOW-02, KNOW-03, KNOW-04  
**Plans:** 1/1 완료

- [x] 11-01: task mesh and knowledge workspace

- graph report가 God Node, surprising connection, stale warning을 구조적으로 반환한다.
- knowledge projector route에 Obsidian-compatible vault export preview를 추가했다.
- Knowledge page graph tab을 실제 `Rt2GraphPanel`로 연결했다.
