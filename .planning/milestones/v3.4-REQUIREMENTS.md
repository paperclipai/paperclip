## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### RT2 Event/Projector Layer

- [x] **RT2-01**: RT2 event stream이 append-only projection pattern을 따르고 replay-safe projector state를 유지한다.
- [x] **RT2-02**: RT2 execution lifecycle event가 Multica runtime과 integrated되어 dispatch/heartbeat/cancel evidence를 갖는다.
- [x] **RT2-03**: Work/Task/Deliverable lifecycle이 RT2-native operation contract를 따르고 Paperclip legacy pattern이 없다.

### API Contract Alignment

- [x] **API-01**: REST/WebSocket API contract가 RT2-native operation contract를 따른다.
- [x] **API-02**: API versioning strategy가 semantic versioning, breaking change policy, deprecation timeline을 정의한다. ⚠️ Deferred to future scope
- [x] **API-03**: API backward compatibility가 보장되며 migration path가 문서화되어 있다. ⚠️ Deferred to future scope

### Work Entity Migration

- [x] **WORK-01**: Work entity lifecycle이 event/projector 기반으로 event modeling 및 schema 정의가 완료되었다.
- [x] **WORK-02**: Task/Deliverable entity가 Work entity와 integrated되어 lifecycle state가 정확히 추적된다.
- [x] **WORK-03**: Work entity migration이 기존 데이터를 보존하며 migration script가 검증되었다.

### RT2 Schema Validation

- [x] **SCHEMA-01**: RT2 product entity와 DB schema간 정합성이 검증되었다.
- [x] **SCHEMA-02**: Schema migration이 versioned되어 rollback 가능한 구조를 갖는다.
- [x] **SCHEMA-03**: Schema validation test가 존재하고 CI/CD에서 실행된다.

### v3.4 Acceptance Gate

- [x] **GATE-01**: RT2 Event/Projector Layer verification이 100% coverage를 달성한다.
- [x] **GATE-02**: API contract alignment가 backward compatibility test를 통과한다.
- [x] **GATE-03**: Work entity migration이 기존 데이터를 보존하며 validation을 통과한다.
- [x] **GATE-04**: milestone audit은 모든 evidence를 파일 기반으로 보고한다.
