# 요구사항 Archive: v2.1 개발기획서 반영 및 운영자 채택

**상태:** 완료, 2026-04-25  
**완료:** 24/24

## 범위

이 마일스톤은 현재 RealTycoon2 앱이 업로드된 RealTycoon2 개발기획서를 얼마나 반영하고 있는지 기준으로 맞추고, v2.0에서 구현된 뼈대를 실제 사내 운영자가 매일 쓰기 쉬운 흐름으로 끌어올리는 데 집중했다.

## 완료된 요구사항

### 캡처와 보상

- [x] **CAP-01**: 사용자는 현재 작업 맥락을 잃지 않고 RT2 어느 페이지에서든 플로팅 One-Liner 캡처 위젯을 열 수 있다.
- [x] **CAP-02**: 사용자는 데스크톱 웹에서 문서화된 글로벌 키보드 단축키로 캡처 위젯에 진입할 수 있다.
- [x] **CAP-03**: 사용자는 짧은 음성 입력을 기록하고, 변환된 One-Liner 초안을 commit 전에 검토할 수 있다.
- [x] **CAP-04**: Slack/Teams 스타일 인바운드 명령 또는 webhook은 웹 UI와 같은 검토형 One-Liner draft 경로를 만들 수 있다.
- [x] **CAP-05**: 작업 신호를 commit한 직후 사용자는 생성된 task, deliverable, 획득 gold, 보상 근거를 즉시 볼 수 있다.

### 일일보고와 OKR/KPI

- [x] **DAILY-01**: 사용자는 navigation/context, report/task editor, detail/Jarvis 패널로 구성된 3패널 일일보고 cockpit에서 작업할 수 있다.
- [x] **DAILY-02**: 일일보고는 수행 task, 완료 to-do, 생성 deliverable, 시간/노력 메모, AI 요약, 품질 상태, gold/XP 영향을 보여줄 수 있다.
- [x] **OKR-01**: 사용자는 정보가 존재하는 경우 작업 항목을 Mission, Objective, Key Result, Project, Task, To-Do까지 추적할 수 있다.
- [x] **OKR-02**: 진행률과 KPI 신호는 To-Do/Task에서 Project/KR/Objective로 bottom-up 집계될 수 있다.
- [x] **OKR-03**: 시스템은 deliverable이 없거나 사용할 수 있는 상위 OKR/KPI 맥락이 없는 작업 항목을 표시할 수 있다.

### Task Mesh와 지식

- [x] **MESH-01**: 사용자는 project의 Task Mesh를 열고 hierarchy, dependency, timeline, collaborator, deliverable, knowledge, economy view를 전환할 수 있다.
- [x] **MESH-02**: Task Mesh node는 raw DB 확인 없이 deliverable, owner, execution, quality, gold, knowledge evidence를 노출한다.
- [x] **KNOW-01**: wiki/projector 출력은 provenance가 있는 운영자용 index, log, topic page 형태로 컴파일될 수 있다.
- [x] **KNOW-02**: 사용자는 markdown 파일을 primary write path로 만들지 않고 wiki page를 Obsidian-compatible vault 구조로 export/sync할 수 있다.
- [x] **KNOW-03**: graph output은 God Node, surprising connection, edge confidence, rationale, stale/ambiguous edge warning을 노출할 수 있다.
- [x] **KNOW-04**: background wiki/graph compiler는 중단 후 재개하고 pending event를 중복/유실 없이 처리할 수 있다.

### Jarvis와 변화관리

- [x] **JARVIS-01**: 관리자는 Shadow/Co-Pilot 품질 평가를 evidence, delta, 승인/거절 액션과 함께 검토할 수 있다.
- [x] **JARVIS-02**: Auto-mode action은 policy threshold로 제한되고, 범위를 벗어나면 Co-Pilot 승인으로 라우팅된다.
- [x] **JARVIS-03**: Jarvis는 예상 deliverable에서 task를 reverse-design evidence와 함께 제안하고 추적 가능한 rationale을 남길 수 있다.
- [x] **JARVIS-04**: runtime skill injection 또는 reusable skill attachment는 숨은 구현 세부사항이 아니라 governed Jarvis capability로 표현된다.

### Enterprise와 rollout

- [x] **ENT-01**: admin은 SSO, company template, binding/access mode, policy default를 RT2 용어의 화면에서 설정할 수 있다.
- [x] **ENT-02**: portable company template은 irreversible 변경 전에 생성/스킵/오류 객체를 preview하고 적용할 수 있다.
- [x] **ENT-03**: product-facing RT2 surface는 infrastructure/reference layer를 명시적으로 설명하는 경우를 제외하고 Paperclip-first label을 피한다.
- [x] **ENT-04**: 앱은 개발기획서 epic을 shipped/partial/missing app capability에 매핑하는 경량 내부 adoption checklist를 포함한다.
