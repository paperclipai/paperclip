# Milestone v2.2: 개발기획서 완전 정합성 고도화

**Status:** shipped 2026-04-25  
**Phases:** 14-18  
**Total Plans:** 5  
**Audit:** `tech_debt` (`.planning/milestones/v2.2-MILESTONE-AUDIT.md`)

## Overview

v2.2는 RealTycoon2 개발기획서 v3.2와의 정합성을 운영자 관점에서 다시 끌어올린 마일스톤이다. 사용자가 우려한 “Paperclip/Multica 본체 위에 RealTycoon2 장식만 얹힌 상태”를 줄이기 위해 product-facing surface를 RealTycoon2, Jarvis, 업무 보드, 일일업무일지, 지식 브릿지, P&L, rollout 검수 흐름 중심으로 고정했다.

마일스톤 완료 후 개발기획서 정합성은 약 94%로 평가한다. 남은 gap은 제품 정체성이나 핵심 workflow 누락이 아니라 실제 SSO/SCIM handshake, Obsidian bidirectional sync, pricing negotiation, settlement approval, mobile/native distribution, Trello advanced feature 같은 후속 운영 깊이다.

## Phases

### Phase 14: Daily Kanban Trello Parity

**Goal:** 일일업무일지 3칸 보드를 Trello처럼 직접 이동 가능한 운영 보드로 만든다.  
**Requirements:** `DAILY-BOARD-01`, `DAILY-BOARD-02`  
**Plans:** 1 plan

- [x] 14-01: 일일업무일지 3칸 칸반 drag/drop과 즉시 저장

**완료:** `Rt2DailyBoard`가 `today`, `support_1`, `support_2` 3칸을 유지하면서 카드 drag/drop 이동과 기존 daily report save API 기반 즉시 저장을 제공한다. 기존 lane select와 저장 버튼도 유지했다.

### Phase 15: Identity Shell Hardening

**Goal:** Paperclip/Multica를 엔진으로 숨기고 product-facing 표면을 RealTycoon2로 고정한다.  
**Requirements:** `ALIGN-01`, `IDENTITY-01`, `IDENTITY-02`  
**Plans:** 1 plan

- [x] 15-01: RealTycoon2 identity shell hardening

**완료:** shell, navigation, command palette, task dialog, Jarvis management, settings, runtime copy에서 Paperclip-shaped 제품명과 Agent/Issue 중심 노출을 줄이고 RealTycoon2/Jarvis/작업 용어를 사용했다. 내부 package/API/route compatibility 이름은 engine layer로 남겼다.

### Phase 16: Trello-Based RealTycoon Work Board

**Goal:** 프론트엔드를 Trello형 업무 보드 기반의 RealTycoon2 작업 경험으로 전환한다.  
**Requirements:** `TRELLO-01`, `TRELLO-02`, `CAPTURE-01`  
**Plans:** 1 plan

- [x] 16-01: RealTycoon2 업무 보드와 One-Liner capture channel contract

**완료:** `/issues` 메인 표면이 board-first RealTycoon2 업무 보드로 전환되었다. 카드에는 Task/To-Do, 담당자, 우선순위, 산출물, 가격, OKR badge가 표시되고 drag/drop 및 빠른 편집이 가능하다. One-Liner inbound draft source contract는 `slack`, `teams`, `webhook`, `mobile`, `native`까지 확장했다.

### Phase 17: Knowledge Bridge Completion

## Milestone Summary

### Key Accomplishments

- 일일업무일지 3칸 칸반이 Trello식 drag/drop 및 즉시 저장을 제공한다.
- product-facing UI에서 Paperclip/Multica 흔적을 줄이고 RealTycoon2/Jarvis/작업 중심 정체성을 강화했다.
- 메인 업무 표면을 Trello 기반 RealTycoon2 업무 보드로 바꾸고 Task/To-Do/산출물/가격/OKR badge를 카드에 연결했다.
- Knowledge Bridge가 projector, vault export, import preview, graph report, evidence status를 하나의 operator flow로 제공한다.
- P&L/marketplace/enterprise rollout 화면이 운영 검수 가능한 evidence와 저장값 hydrate를 제공한다.

### Issues Deferred

- Phase 14-18의 `VALIDATION.md`가 없어 Nyquist 검증 산출물은 `missing`이다.
- 일부 server route test는 embedded Postgres host init 제약으로 collected 후 skip되었다.
- 실제 SSO handshake, SCIM sync, provider metadata validation은 후속 범위다.
- Obsidian local writer와 bidirectional sync/conflict resolution은 후속 범위다.
- mobile/native app distribution은 route/contract 이후의 별도 배포 범위다.

### Technical Debt

- 내부 `@paperclipai/*`, `/issues`, `/agents`, `Issue` compatibility layer 이름은 아직 남아 있다. 제품 UI에서는 숨겼고, engine/internal migration은 별도 정당화가 필요하다.
- Trello의 checklist, due date, attachment preview, advanced sorting은 후속 작업이다.
- 가격 협상, settlement approval, anti-gaming reputation depth는 경제 고도화 후속 범위다.
