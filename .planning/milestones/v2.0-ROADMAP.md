# Milestone v2.0: RT2 Refoundation

**상태:** 2026-04-25 완료  
**Phases:** 1-7  
**Total Plans:** 13  
**Total Requirements:** v1 requirements 20개

## 개요

v2.0은 Paperclip-heavy brownfield였던 현재 앱을 RealTycoon2의 1급 operating loop로 전환했다. 이 마일스톤은 RT2를 primary shell로 만들고, low-friction work logging을 deliverable/base price와 연결했으며, execution lifecycle state, event-stream/projector infrastructure, cumulative wiki/graph knowledge, live evidence 기반 Jarvis/quality/search, ledger-backed P&L/marketplace/collaboration reward를 구현했다.

## Phase

### Phase 1: RT2 Shell and Product Truth

**목표:** 내부 Paperclip control plane 접근성을 유지하면서 RT2를 primary experience로 재구성.  
**Requirements:** IDENT-01, IDENT-02  
**Plans:** 2

- RT2-first company routing은 이제 One-Liner로 진입한다.
- primary navigation은 One-Liner, Knowledge, Marketplace, P&L, Org, Governance, Control Plane을 노출한다.
- legacy Paperclip surface는 company context를 잃지 않고 Control Plane 뒤에서 접근 가능하다.
- Windows runtime/worktree test blocker를 해결해 Phase 1 verification이 통과했다.

### Phase 2: One-Liner and Deliverable Capture

**목표:** low-friction logging을 primary input path로 만들고 deliverable-aware work capture에 연결.  
**Requirements:** LOG-01, LOG-02, ECON-01  
**Plans:** 6

- One-Liner는 freeform work input을 task, todo, deliverable, daily-log draft field로 변환한다.
- deliverable base price는 structured result의 일부이며 RT2 task engine을 통해 persist된다.
- global shell entry point는 company-scoped One-Liner flow로 연결된다.
- Windows materialization, adapter execution, diagnostics, runtime, full-suite verification gap을 닫았다.

### Phase 3: Multica Execution Backbone

**목표:** RT2 work를 실제 execution lifecycle에 연결하고 orchestration/runtime 사이의 명확한 contract 유지.  
**Requirements:** FOUND-01, FOUND-02  
**Plans:** 1

- enqueue, claim, start, complete, fail, retry를 위한 RT2 execution-attempt persistence를 추가했다.
- shared execution contract, scoped route handler, UI execution summary를 추가했다.
- execution/runtime state를 RT2 task와 todo surface에 다시 연결했다.

### Phase 4: CQRS Event Stream and Projections

**목표:** RT2 write를 append-only event path로 옮기고 resilient projector를 통해 read model을 재구축.  
**Requirements:** LOG-03, CQRS-01, CQRS-02, GOV-01  
**Plans:** 1

- durable RT2 domain event, projector state, projector event tracking을 추가했다.
- task, deliverable, participant, capacity, todo, execution mutation에 event append를 통합했다.
- activity/live update는 durable source of truth가 아니라 projection output이 되었다.

### Phase 5: wikiLLM and Graphify Knowledge Core

**목표:** RT2 event를 page와 graph structure로 바꾸는 cumulative company memory layer 구축.  
**Requirements:** FOUND-03, KNOW-01, KNOW-02  
**Plans:** 1

- `index.md`, `log.md`, topic page를 위한 cumulative wiki persistence를 추가했다.
- evidence와 provenance-aware `EXTRACTED` edge를 가진 graph node/edge persistence를 추가했다.
- company-scoped wiki page read와 projection API를 추가했다.

### Phase 6: Jarvis, Quality, and Hybrid Search

**목표:** empty assistant behavior를 live RT2 knowledge 기반 context-grounded AI operation으로 교체.  
**Requirements:** JARV-01, QUAL-01, QUAL-02  
**Plans:** 1

- Jarvis advice, breakdown, insight는 live task, deliverable, wiki, graph, quality evidence를 사용한다.
- quality evaluation은 persisted evidence와 approval boundary를 가진 Shadow, Co-Pilot, Auto mode를 지원한다.
- hybrid search는 wiki, graph, task, deliverable, document를 retrieve/rerank한다.

### Phase 7: Amoeba Economy, Collaboration, and Marketplace

**목표:** economic incentive와 market surface가 live RT2 work evidence에서 나오게 만들기.  
**Requirements:** ECON-02, MKT-01, COLLAB-01  
**Plans:** 1

- P&L은 approved deliverable evaluation을 actor ledger와 P&L row로 materialize한다.
- marketplace listing은 skill, pricing, deliverable count, quality, reputation, subscription evidence를 노출한다.
- collaboration reward는 persisted participant, work-product, quality evidence에서 idempotent하게 derive된다.
- P&L과 Marketplace page는 placeholder shell 대신 company-scoped API data를 render한다.

## 마일스톤 요약

### 주요 성과

- RT2가 nested Paperclip-shaped tab set이 아니라 primary company operator shell이 되었다.
- One-Liner work capture는 deliverable-aware, price-aware structured draft를 생성한다.
- task execution, append-only event, projector state가 durable RT2 backbone을 제공한다.
- company knowledge는 wiki page와 provenance-aware graph structure로 누적된다.
- Jarvis, quality evaluation, search는 live company-scoped evidence를 사용한다.
- Amoeba P&L, marketplace evidence, collaboration reward는 persisted work record와 reconcile된다.

### 주요 결정

- 안정적인 Paperclip control-plane asset은 제품 정체성이 아니라 infrastructure로 계속 사용한다.
- RealTycoon2 business truth는 RT2-controlled schema와 service에 둔다.
- high-contention RT2 work에는 event stream과 projector를 durable path로 취급한다.
- RT2 mutation 전반에서 company scope, permission check, approval boundary, audit/activity record를 보존한다.

### 미룬 이슈

- external embedding generation과 LLM reranking은 future work로 남겼다.
- Obsidian-compatible bidirectional wiki sync는 future `KNOW-03`으로 남겼다.
- native mobile, Slack, Teams logging surface는 다음 마일스톤 확장 후보로 남겼다.
- 두 historical UAT file은 close audit에서 `unknown`으로 보고되었지만 pending scenario는 0개였다.

### 기술 부채

- 일부 inherited runtime/orchestration test name과 string은 아직 Paperclip wording을 사용한다.
- execution-attempt lifecycle state를 넘어서는 실제 daemon/runtime loop는 이후 runtime phase로 남아 있다.
- 기존 Vite large chunk warning은 non-blocking build debt로 남아 있다.
