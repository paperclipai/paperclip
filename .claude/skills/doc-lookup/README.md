# doc-lookup

A **dynamic-database skill** generated by `lookup-skill-builder`. The model reads
a thin router, resolves what it needs, and opens a single file — instead of
loading the whole corpus into context.

## Layout

```
doc-lookup/
├── SKILL.md              # Tier 1 — description + navigation protocol
├── lookup.config.json    # what to index and how
├── index.json           # Tier 2 — generated router (aliasIndex + entries)
├── detail-index.json    # Tier 2 — generated section heading → file
└── scripts/build-index.cjs
```

The corpus lives at `/Users/user/sourceControl/paperclip/doc` and is **not**
duplicated here — `index.json` records its absolute path.

## Regenerating

Never hand-edit the `*.json` indexes. After any change to the corpus:

```bash
node doc-lookup/scripts/build-index.cjs
```

The index is always derived from the data, so it cannot drift.

> Note: the indexer is `.cjs` (CommonJS) because the Paperclip repo's
> `package.json` sets `"type": "module"`.
