# Daily Goal Tracking Loop

기준일: `2026-03-16`

## 목적

이 문서는 `Daily Focus`, `Cycle`, `Linear`를 어떻게 연결해야
팀이 하루 목표와 주간 목표를 잃지 않고 달릴 수 있는지 정리한 운영 문서다.

핵심은 이것이다.

`전일 약속이 실제 결과로 닫히고, 오늘 목표가 이번 주 Must-win goal과 연결되어야 한다.`

## 먼저 결론

지금 팀에는 아래 구조가 가장 맞다.

- `Cycle`: 이번 주에 반드시 바뀌어야 하는 것 1~3개를 고정
- `Daily Focus`: 오늘 누가 무엇을 끝내려는지 팀이 같이 보는 하루 스냅샷
- `Linear Issue`: 실제 작업 흔적과 결과를 남기는 실행 원본

한 줄로 말하면:

`주간 목표는 Cycle에서, 오늘 약속은 Daily Focus에서, 실제 결과는 Linear Issue에서 닫는다.`

## Daily Focus에 꼭 있어야 하는 것

### 1. 이번 주 Progress

Daily Focus 첫 화면에 이번 주 Must-win goal 진행도를 보여줘야 한다.

이유:

- 오늘 일이 이번 주 목표와 연결되는지 바로 보인다
- 우선순위가 흔들리는지 바로 보인다
- 대표가 하루만 봐도 주간 상태를 읽을 수 있다

### 2. 전일 목표 체크

Daily Focus는 오늘 목표만 적는 문서가 아니라
어제 약속이 실제로 끝났는지 먼저 닫는 문서여야 한다.

기록할 것:

- 전일 목표
- 관련 Linear Issue
- 실제 결과
- 완료 여부
- 이월 여부

### 3. 오늘 최우선 1개

사람당 `오늘 최우선 1개`만 적는다.

기록할 것:

- 관련 Linear Issue
- 완료 기준
- 첫 액션
- 요청 / 막힘

### 4. 저녁 마감 체크

하루는 반드시 닫혀야 한다.

기록할 것:

- 실제 결과
- 완료 / 미완료
- 미완료 이유
- 내일 첫 액션

## Daily Focus 자동 생성 기준

자동 생성 시 아래 정보를 먼저 채워주는 구조가 좋다.

### 1. 전일 문서에서 가져올 것

- 전일 사람별 최우선 목표
- 전일 미완료 / 이월 항목
- 전일 요청 / 막힘

### 2. Linear에서 가져올 것

- 전일 연결된 Issue 상태
- 최근 코멘트
- 완료 여부
- 현재 진행 중 Issue

### 3. 이번 주 Cycle에서 가져올 것

- Must-win goal 1~3개
- 현재 진행도
- 이번 주 핵심 막힘

## 실무 리듬

### 아침

1. Daily Focus 자동 생성
2. 전일 목표 체크 먼저 닫기
3. 사람별 오늘 최우선 1개 확정
4. 각자 관련 Linear Issue에 아침 코멘트 남기기

### 저녁

1. 각자 같은 Issue에 결과 코멘트 남기기
2. Daily Focus에 완료 / 미완료 / 이월 체크
3. 이월 항목은 다음날 전일 체크로 자동 승계

## Cycle도 같은 원리로 써야 한다

Cycle도 단순 회의록이 아니라 `주간 scoreboard`여야 한다.

Cycle에 꼭 있어야 하는 것:

- 지난주 carryover
- 이번 주 Must-win goal 1~3개
- 목표별 진행도
- 이번 주 핵심 이슈
- 필요한 결정
- 주말 마감 체크

즉 Cycle은 `이번 주 약속`, Daily Focus는 `오늘 약속`, Linear는 `실행 증거`다.

## 작은 팀용 규칙

1. 사람당 하루 최우선은 1개만 둔다
2. 최우선 목표에는 반드시 Linear Issue를 연결한다
3. 완료 기준이 없는 목표는 적지 않는다
4. 저녁 마감 없이 다음 날로 넘어가지 않는다
5. 같은 일이 2일 이상 이월되면 Cycle에서 다시 정렬한다

## 한 줄 정리

지금 팀에는 더 많은 문서가 필요한 게 아니다.
`전일 결과 -> 오늘 목표 -> 주간 Progress`가 끊기지 않는 운영 루프가 필요하다.
