# Google Drive 운영 방식

이 문서는 회사 자료를 Google Drive에서 읽고, 검색하고, 필요한 문서를 가져오는 기준을 정리한다.

핵심은 하나다.

`브라우저 로그인으로 빠르게 보고, 안정적인 자료 검색/다운로드는 OAuth API로 붙인다.`

## 결론

- 단순 API 키만으로는 private Google Drive 자료 운영에 부족하다
- 대표 개인 드라이브와 회사 자료를 안정적으로 읽으려면 `OAuth 2.0`이 기본이다
- 따라서 운영 기준은 아래 2단계로 간다
  - 1단계: 로그인된 브라우저 탭으로 빠른 확인
  - 2단계: OAuth 연결 후 Drive CLI로 검색/목록/다운로드

## 지금 가능한 것

현재 로그인된 크롬 탭 기준으로 아래는 바로 가능하다.

- 내 드라이브 최상위 폴더/파일 빠른 확인
- 최근 보이는 문서명과 구조 파악
- 어떤 자료가 드라이브에 있는지 빠르게 훑기

## 권장 운영 방식

### 1. Drive 역할 정의

- Drive:
  - 원본 자료
  - 제안서
  - 회계/재무 관련 스프레드시트
  - 고객사별 자료
  - 사업계획서/내부 참고 자료
- repo:
  - 판단
  - 요약
  - 운영 기준
  - 의사결정 문서

즉, Drive는 원본 저장소이고 repo는 해석 저장소다.

### 2. 대표 비서 동작 기준

- 자료 요청이 오면 먼저 Drive에서 원본을 찾는다
- 회의/전략/제안 판단은 repo에 요약한다
- 같은 자료를 반복해 찾는 경우 repo 쪽에 링크와 해석을 남긴다

### 3. 운영상 주의할 것

- Drive 원본을 repo에 무분별하게 복제하지 않는다
- 민감 문서는 다운로드보다 링크/위치/요약 중심으로 다룬다
- 같은 이름의 문서가 많아지면 폴더 기준으로 정리한다

## 권장 인증 방식

### 브라우저 로그인 기반

장점:

- 지금 바로 빠른 확인 가능
- 실제 보이는 폴더/파일 구조를 즉시 읽을 수 있다

단점:

- 화면에 보이는 범위만 읽기 쉽다
- 안정적인 검색/다운로드에는 약하다
- 브라우저 구조 변경에 취약하다

### Google Drive API + OAuth

장점:

- 검색, 목록, 파일 메타데이터 조회가 안정적이다
- 문서 자동 수집과 요약 워크플로우로 이어가기 좋다
- 나중에 대표 비서형 자료 조회에 적합하다

단점:

- OAuth 설정이 필요하다
- refresh token 관리가 필요하다

## 필요한 환경변수

- `GOOGLE_DRIVE_CLIENT_ID`
- `GOOGLE_DRIVE_CLIENT_SECRET`
- `GOOGLE_DRIVE_REFRESH_TOKEN`
- `GOOGLE_DRIVE_ROOT_ID`
