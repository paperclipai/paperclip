# Company Planning Stack

기준일: `2026-03-16`

## 목적

이 문서는 회사의 계획 계층을
`직관적이고 명료하게`, 그리고 `Linear 실행 흐름에 맞게`
다시 정리하기 위한 문서다.

핵심 질문은 이것이다.

`Chapter > Cycle > Daily Focus 구조를 어떻게 더 얇고 선명하게 만들 것인가?`

## 먼저 결론

지금 팀에는 아래 5단 구조가 가장 맞다.

1. `Initiative`
2. `Chapter`
3. `Project`
4. `Cycle`
5. `Daily Focus`

하지만 중요한 점은 하나다.

`Chapter는 계층 하나를 더 늘리는 관리 시스템이 아니라, 6~8주짜리 운영 브리프여야 한다.`

## 추천 계층

### 1. Initiative

도구:

- `Linear`

주기:

- 3~6개월

역할:

- 회사 차원의 큰 변화 방향
- 여러 Project를 묶는 상위 목표

규칙:

- 활성 Initiative는 `1~3개`만
- 대표나 리더가 owner를 가진다
- Initiative Update는 `격주 또는 Chapter 종료 시점`에 남긴다

### 2. Chapter

도구:

- `repo + Notion`

주기:

- 6~8주

역할:

- 이번 기간에 `무엇이 달라져야 하는가`를 한 장으로 고정
- Initiative를 현재 실행 기간으로 압축

규칙:

- `한 장`이면 충분하다
- 목표는 `2개`, 많아도 `3개`
- O/KR, Project 설명, 회의록을 다 넣지 않는다
- Chapter는 `요약과 정렬`, 실행은 Linear가 맡는다

### 3. Project

도구:

- `Linear`

주기:

- 2~8주

역할:

- Chapter 목표를 실제 deliverable로 쪼개는 실행 덩어리

규칙:

- Project는 `상태 변화`가 아니라 `끝낼 수 있는 묶음`이다
- 각 Project는 가능한 한 `하나의 결과`를 가져야 한다
- Chapter마다 핵심 Project는 `2~4개` 정도가 적당하다

### 4. Cycle

도구:

- `Linear Cycle + 회의록`

주기:

- 1주

역할:

- 이번 주에 어떤 Issue를 끝낼지 정렬
- Project를 주간 단위로 전개

규칙:

- Cycle 회의는 `이번 주 Must-win 1~3개`만 고정
- 회의 직후 관련 Project에 `Project Update`를 남긴다
- Cycle은 `주간 scoreboard`처럼 써야 한다

### 5. Daily Focus

도구:

- `repo Daily Focus + Linear Issue Comment`

주기:

- 1일

역할:

- 전일 결과 확인
- 오늘 최우선 1개 고정
- 저녁에 닫기

규칙:

- 사람당 하루 최우선 `1개`
- 관련 Linear Issue 필수
- 실제 결과는 이슈 코멘트와 Daily Focus에 같이 닫는다

## Linear 기능을 이렇게 쓴다

### Initiative

- 회사의 상위 방향
- 대표가 보는 큰 지도
- Update는 격주 또는 Chapter 종료 시점

### Project

- 실제 실행 덩어리
- Chapter 목표와 연결
- Update는 `주간`

### Project Update

- 매주 첫날 Cycle 회의 직후
- health / 이번 주 목표 / 막힘 / 다음 액션

### Cycle

- 이번 주 처리할 이슈를 timebox로 묶음
- 현재 작업 집중도를 보는 층

### Issue

- 가장 작은 실행 단위
- 상태 변경과 코멘트가 실제 흔적

## Chapter를 어떻게 더 얇게 만들까

Chapter에는 아래만 남기면 된다.

1. 기간
2. 이번 Chapter 한 줄
3. 이번 기간에 달라질 상태 2~3개
4. 하지 않을 것
5. 연결 Initiative
6. 연결 Project

즉 Chapter는 `전략 문서`가 아니라 `운영 브리프`다.

## 한 줄 정리

지금 팀에는 계층을 늘리는 것이 아니라,
`Initiative는 크게, Chapter는 얇게, 실행은 Linear에 깊게`
가는 방식이 가장 잘 맞는다.
