# 키움증권 투자자산 데이터

이 문서는 키움증권 웹에서 확인하는 법인 투자자산 현황을
회사 운영 논의에 연결하기 위한 기준 문서다.

## 목적

- 법인 투자자산 규모를 운영자금과 분리해서 본다
- 현금성 자산과 투자자산을 같이 보되, 섞어서 판단하지 않는다
- 필요할 때 현재 자산현황을 빠르게 기록하고 전략 논의에 반영한다

## 현재 제약

키움증권은 아래 이유로 Alfred처럼 바로 로그인 자동화하는 방식보다
`수동 로그인 후 세션 기반 조회`가 더 현실적이다.

- 보안 키패드 사용
- 보안 모듈(AhnLab, ASTX, Xecure) 로딩
- 일반 headless 브라우저 접근에 민감함

따라서 현재 권장 접근은 다음과 같다.

1. 사람이 직접 크롬에서 로그인
2. 자산현황 페이지를 연다
3. 열린 탭에서 숫자와 요청 경로를 읽는다
4. 가능한 범위부터 점진적으로 자동화한다

## 현재 확인된 자산 페이지

- 페이지: `https://www3.kiwoom.com/h/mykiwoom/asset/VWonAssetView?dummyVal=0`
- 화면 제목: `원화자산 | 투자현황 | 자산현황 | MY키움 | 키움증권`

## 현재 확인된 조회 경로

페이지 내부 스크립트에서 확인한 실제 조회 함수:

- 함수명: `fn_listAjax()`
- 호출 경로: `POST /h/mykiwoom/asset/SWonAssetAjax`
- 전달 파라미터: `pswd`

페이지 내부 스크립트상 반영되는 주요 필드:

- `tot_dep_amt`: 총 추정원화자산
- `tot_grnt`: 융자/대출총액
- `tot_est_amt`: 총 유가증권 잔고평가
- `tot_entr`: 예수금총액
- `wonAsset.g1`: 계좌별 현황 grid 데이터

## 운영 해석

이 숫자는 회사 운영 관점에서 아래처럼 봐야 한다.

- `운영자금`과 `투자자산`은 분리해서 본다
- 투자자산은 회사의 안전판일 수 있지만, 월 운영비와 같은 성격은 아니다
- 계좌 예수금은 매우 작고 대부분이 주식 평가금액이므로 유동성 판단에 그대로 쓰면 안 된다
