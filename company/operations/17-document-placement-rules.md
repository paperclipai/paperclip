# Document Placement Rules

기준일: 2026-03-16

이 문서는 새 자료가 들어왔을 때 `어느 폴더에 저장해야 하는지`를 정하는 운영 규칙이다.

핵심은 하나다.

`문서의 주제보다 역할을 기준으로 폴더를 정한다.`

## 1. 가장 먼저 보는 판단 기준

새 자료가 들어오면 아래 순서로 본다.

1. 이 문서는 `판단`인가, `원본`인가
2. 이 문서는 `회사 전체 기준`인가, `제품별 맥락`인가
3. 이 문서는 `반복 템플릿`인가, `실제 기록`인가
4. 이 문서는 `민감 원본 데이터`인가, `사람이 읽는 요약`인가

## 2. 폴더별 배치 기준

### strategy

넣는 것:

- 회사 방향
- 우선순위
- 시장/ICP/가격/중단 기준
- 자산 KPI

넣지 않는 것:

- 회의 메모
- 원본 사업계획서
- 개인 메모

### products

넣는 것:

- 제품별 사업계획서
- 제품 원본 Notion 아카이브
- 제품별 포지셔닝 원문
- 제품별 과거 가설

넣지 않는 것:

- 현재 회사 공식 운영 원칙
- 회사 전체 HR 정책

### roadmap

넣는 것:

- 연간 계획
- Chapter 운영
- 게임 투자 시간 모델
- 연매출 계획

넣지 않는 것:

- source of truth 문서
- 위키 원본 아카이브

### operations

넣는 것:

- 회사 운영 리듬
- 자금 보드
- 회의 운영
- 캘린더 / GitHub / Drive / 마케팅 운영 기준
- source of truth 지도

넣지 않는 것:

- 채용공고 원문
- 제품별 사업계획서 원문

### management

넣는 것:

- HR 기준
- 채용공고
- 회사 소개
- CRM 기준
- 회사 위키 source of truth
- 회사 법인/온보딩/정책 아카이브
- 수주 판단

넣지 않는 것:

- 장기 제품 전략 문서
- 일간 로그

### areas

넣는 것:

- 회사의 `3단계 실행축`별 운영 기준
- 외주개발 / 자체서비스 개발 / 게임개발사 영역 문서
- 특정 사업 영역에만 해당하는 워크플로우

넣지 않는 것:

- 회사 전체 공통 정책
- 원본 데이터
- 반복 포맷 템플릿

### rules

넣는 것:

- 운영 원칙
- 하지 말아야 할 것
- 예외 허용 기준

### checklists

넣는 것:

- 주간 점검표
- 월간 스코어카드
- 반복 운영 체크리스트

### daily

넣는 것:

- 오늘 한 일
- 오늘 판단
- 내일 단일 초점

### decisions

넣는 것:

- 큰 방향 전환
- 수주 / 거절 결정
- 조직 운영 방식 변경

### templates

넣는 것:

- 반복 포맷의 원본

### founder

넣는 것:

- 대표 개인 철학
- 창업 동기 초안
- 회사 공식 문서 전 단계의 개인 맥락

### tools

넣는 것:

- 수집 스크립트
- 조회 CLI
- OAuth 보조 도구

### private-data

넣는 것:

- 원본 스냅샷
- API 응답
- 민감 운영 데이터

## 3. 자주 헷갈리는 사례

### 사례 1. 노션에서 읽은 제품 사업계획서

- `products/`

### 사례 2. 노션 회사 위키 / 온보딩 / 정책 페이지

- `management/`

### 사례 3. 계약서, 세금계산서, 금융 데이터 원본

- 원본: `private-data/`
- 해석: `operations/`

### 사례 4. 대표 생각 정리

- 개인 철학/의미: `founder/`
- 회사 공식 방향: `strategy/`

### 사례 5. 반복 사용하는 회의 양식

- `templates/`

### 사례 6. 이번 주 실제 회의 내용

- Notion 회의록 또는 `daily/`, 필요하면 `decisions/`

### 사례 7. 외주개발 리드 대응 워크플로우

- `areas/outsourcing/`

## 4. 승격 규칙

- founder 메모가 회사 공식 방향이 되면 `strategy/`나 `management/`에 다시 쓴다
- 제품 원본이 현재 실행 기준이 되면 `strategy/`나 `roadmap/`에서 재해석한다
- 원본 수치가 운영 판단에 쓰이면 `operations/`에 요약을 만든다
- 반복되는 기록 형식은 `templates/`로 승격한다

## 한 줄 정리

새 자료는 `무엇에 관한 문서인가`보다 `회사 안에서 어떤 역할을 하는 문서인가`로 분류한다.
