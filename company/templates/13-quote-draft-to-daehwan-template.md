# Quote Draft To Daehwan Template

## 목적

고객에게 바로 보내는 문서가 아니라,
`대환이 확인하고 직접 발송할 수 있게 정리한 견적 초안 전달용 템플릿`

## 기본 원칙

- 짧고 분명하게 쓴다
- 고객이 요청한 범위와 우리가 이해한 범위를 같이 적는다
- 옵션별 차이가 있으면 나눠서 적는다
- 확정이 아닌 것은 `초안`이라고 분명히 적는다
- 마지막에 `대환 확인 필요 사항`을 남긴다

## 템플릿

제목: `[견적 초안] 고객명 / 프로젝트명`

대환님,

아래 건 견적 초안 정리했습니다.

- 고객: `고객명`
- 프로젝트: `프로젝트명`
- 현재 상태: `리드 / 검토 중 / 미팅 후`
- 요청 마감: `날짜`

### 1. 고객 요청 이해

- `요청 내용 1`
- `요청 내용 2`
- `요청 내용 3`

### 2. 우리가 잡은 범위

- 기본 범위:
  - `범위`
- 제외한 범위:
  - `범위`

### 3. 옵션별 차이

- 옵션 A:
  - `포함 내용`
  - `예상 금액`
  - `예상 일정`
- 옵션 B:
  - `포함 내용`
  - `예상 금액`
  - `예상 일정`

### 4. 전제 조건

- `전제 1`
- `전제 2`

### 5. 확인 필요한 점

- `질문 1`
- `질문 2`

### 6. 전달 제안

- 고객에게 이렇게 설명하면 좋음:
  - `설명 포인트 1`
  - `설명 포인트 2`

초안 기준이라 수정 필요한 부분 말씀 주시면 바로 반영하겠습니다.
