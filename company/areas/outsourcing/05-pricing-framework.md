# Outsourcing Pricing Framework

기준일: 2026-03-26

이 문서는 외주개발 / B2B2C 웹 구축 견적을 내부에서 일관되게 산정하기 위한 `단가 체계`다.

핵심은 하나다.

`MVP 1W로 진입 장벽을 낮추고, 확장 계약으로 전체 매출을 만든다.`

## 왜 이 문서가 필요한가

- 사람마다 견적이 들쭉날쭉해지는 문제를 줄인다
- superbuilder 기반 정형 작업은 빠르게, 낮은 단가로 처리한다
- 직접 설계와 안정화가 필요한 일에만 높은 가격을 붙인다
- 고객에게 설명 가능한 구조로 내부 판단을 먼저 고정한다
- AI 시대의 시장 가격 기대치를 반영한다

## 시장 환경 인식 (2026-03 기준)

AI 도구 보편화로 고객의 체감 가격 기준이 이전 대비 크게 낮아졌다.
superbuilder로 실제 원가도 줄었으므로, 이를 가격에 솔직하게 반영한다.

차별화 포인트는 `낮은 가격`이 아니라 `압도적 속도`다.

- 경쟁사: 2,500만 / 3개월
- 비브라이트코드: 2,000만 / 1개월

## 기본 원칙

### 원칙 1

정형화된 기능은 낮게 받는다.

### 원칙 2

다른 곳에서도 비슷하게 구현 가능한 흔한 기능은 높게 받지 않는다.

### 원칙 3

높은 가격은 `운영 플로우`, `예외 처리`, `안정성`, `직접 설계`에서 받는다.

### 원칙 4

QA 및 안정화는 별도 줄세우지 않고 기본 포함으로 본다.

### 원칙 5

feature 가격은 `이미 superbuilder에서 재사용 가능한 상태`일 때만 정가를 적용한다.

### 원칙 6

전체 견적을 한 번에 제시하지 않는다. MVP 진입 → 확장 계약의 2단계 구조로 제안한다.

### 원칙 7

MVP 단계에서 고객이 가장 궁금한 1가지를 실물로 보여준다. 나머지는 확장에서 얹는다.

## MVP 1W 상품 체계

### MVP 티어

고객 문의 시 전체 견적을 먼저 보여주지 않는다.
복잡도에 따라 MVP 티어를 먼저 제안하고, 확장은 MVP 이후에 논의한다.

| 티어 | 대상 | 기간 | 가격 | superbuilder 커버리지 |
| --- | --- | --- | ---: | --- |
| Lite | 랜딩+폼+어드민, 소개 사이트 | 1주 | 300만 원 | 90% 자동 |
| Standard | 예약/결제/회원 포함 서비스 | 2주 | 500~800만 원 | 70% 자동 + 커스텀 |
| Pro | 실시간 처리, AI 연동, 알고리즘 | 3~4주 | 1,000~1,500만 원 | 50% 자동 + 엔진 개발 |

### MVP 제안 원칙

- MVP에서 보여주는 것 = 고객이 가장 궁금한 1가지 (실제 동작하는 실물)
- 나머지 기능(관리자, 결제, 배포, 운영)은 확장 계약에서 얹는 구조
- "마음에 드시면 확장, 아니면 MVP까지만으로도 쓸 수 있는 상태" 전달

### MVP → 확장 수익 구조

```
MVP (300~1,500만) → 고객 확신 → 확장 개발 (1,500~3,000만) → 유지/운영 (월 30~50만)
```

### 고객 대면 메시지 기본형

> "전체 개발은 [X]만 원 규모입니다.
> 다만 저희는 먼저 핵심 기능이 동작하는 MVP를 [N]주 안에 [Y]만 원에 만들어 드립니다.
> 직접 써보시고, 방향이 맞다고 판단되시면 나머지를 이어서 진행합니다."

## 견적 계산 공식

`총 견적 = 기본 골격 + 선택 feature + 핵심요구 기능`

AI 시대 기준으로 superbuilder 공수 절감분을 반영한 단가다.

## 기본 골격 단가

| 구분 | 기준가 |
| --- | ---: |
| 랜딩 및 브랜드 웹 | 200만 원 |
| 관리자 기본 | 50만 원 |
| 블로그 / CMS | 150만 원 |
| 인프라 / 배포 환경 구성 | 100만 원 |
| QA 및 안정화 | 기본 포함 |

내부 관리는 `feature 개별`보다 `카테고리 > 세부 기능` 구조로 한다.

| 세부 기능 | 기준가 |

| 기본 회원 로그인 확장 | 70만 원 |

### 2. 콘텐츠 관리

| 세부 기능 | 기준가 |
| --- | ---: |

| 포트폴리오 관리 | 50만 원 |

### 3. 문의 / 리드 운영

| 세부 기능 | 기준가 |
| --- | ---: |
| 문의 저장 API + DB 연결 | 50만 원 |
| 문의 관리 확장 | 70만 원 |

### 4. 외부 연동

| 세부 기능 | 기준가 |
| --- | ---: |
| Slack 웹훅 연동 | 30만 원 |
| 연동 기능 묶음 패키지 | 100만 원 |

### 5. 맞춤형

| 세부 기능 | 기준가 |
| --- | ---: |

| 기존 feature로 흡수되지 않는 기능 | 별도 산정 |

| 세부 기능 | 기준가 |
| --- | ---: |
| 단순 AI 챗봇 / 요약 기능 | 200만 원 |

| AI 상담 / 워크플로우 연동 | 500만 원 |
| AI 에이전트 (멀티스텝, 도구 사용) | 800만 원~ |
| 복합 AI 파이프라인 (팩트체크 등) | 컨설팅 후 별도 산정 |

## 핵심요구 기능 단가 가이드

핵심요구 기능은 단순 feature가 아니라,
고객 운영 방식에 맞춰 직접 설계하거나 예외 처리가 필요한 기능이다.

| 구분 | 기준가 |
| --- | ---: |
| 견적 검토 플로우 | 80만~120만 원 |
| 견적 전달 / 발송 운영 플로우 | 80만~120만 원 |
| CRM 수집 / 분류 로직 | 80만~150만 원 |
| 고객별 별도 운영 로직 | 120만 원 이상 |
| 예약/스케줄링 시스템 | 200만~400만 원 |
| 결제 연동 (PG/인앱) | 150만~300만 원 |
| 실시간 채팅/메시징 | 300만~500만 원 |
| 네이티브 앱 배포 (iOS+Android) | 200만~400만 원 |

## B2B2C 견적 해석 규칙

### 1안

- 메인
- 소개
- 포트폴리오
- 문의 페이지

즉, 대외 표면 중심의 랜딩형 구축이다.

### 2안

- 1안 범위
- 관리자 로그인
- 문의 조회
- 상태 관리
- 운영 메모

즉, B2B 운영 표면이 붙는 구조다.

### 3안

- 2안 범위
- 블로그 / CMS
- FAQ
- 게시판
- 포트폴리오 관리

즉, 이미 정형화된 재사용 feature를 붙이는 구조다.

### 4안

- 3안 범위
- 견적 검토
- 견적 전달 / 발송 플로우
- CRM 수집 / 분류
- 고객별 특수 운영 로직
- 맞춤형 기능
- AI 연동 기능

즉, 운영 설계가 필요한 커스텀 구조다.

## 적용 규칙

### 정가 적용

아래 조건이면 카테고리 안의 세부 기능 정가를 적용한다.

- superbuilder에서 이미 재사용 가능하다
- 설정값 변경 위주다
- 예외 처리가 적다
- 새 도메인 설계가 필요 없다

### 가산 적용

아래 조건이면 정가의 `20%~30%`를 가산한다.

- 약간의 커스터마이징이 있다
- 같은 카테고리 안의 기능을 함께 붙이는 통합 공수가 있다
- 고객 정책 반영이 조금 복잡하다

### 별도 산정

아래 조건이면 정형화된 카테고리 기능이 아니라 핵심요구 기능으로 분리한다.

- 새 feature를 직접 만들어야 한다
- 고객별 운영 로직이 다르다
- 예외 처리가 많다
- 안정성과 실패 비용이 크다
- `맞춤형` 카테고리에 해당한다
- `AI 연동` 카테고리에 해당한다

## 빠르게 보는 예시

### 예시 1 — 소개 사이트 (MVP Lite)

랜딩 + 관리자 + 블로그 / CMS

- 200 + 50 + 150 = `400만 원` / 1주 납품

### 예시 2 — 서비스 사이트

랜딩 + 관리자 + 블로그 / CMS + 인증(카카오싱크) + 외부 연동(웹훅) + 문의 저장

- 200 + 50 + 150 + 50 + 30 + 50 = `530만 원` / 2주 납품

### 예시 3 — 운영형 사이트

예시 2 + 견적 검토 플로우

- 530 + 80~120 = `610만~650만 원` / 2~3주 납품

### 예시 4 — AI 연동 서비스

예시 2 + AI 챗봇

- 530 + 200 = `730만 원` / 2~3주 납품

### 예시 5 — 앱 + 실시간 기능

크로스플랫폼 앱 + 실시간 채팅 + 결제

- 200(랜딩) + 50(관리자) + 100(인프라) + 300(채팅) + 200(결제) + 200(앱배포) = `1,050만 원` (MVP)
- 전체 확장 시 2,000~2,500만 원

## 견적 문구 원칙

- 고객에게는 세부 항목보다 `카테고리별 금액`을 우선 보여준다
- 내부에서는 카테고리별 세부 단가를 조합해 빠르게 계산한다
- 연동 자체보다 `연동 이후 운영 플로우`에서 가격이 올라간다는 점을 명확히 본다
- `카카오싱크 자체`는 흔한 작업이므로 고가 feature로 보지 않는다
- `맞춤형` 카테고리는 정가를 두지 않고 항상 별도 산정으로 본다
- `AI 연동` 카테고리는 반드시 컨설팅 후 별도 산정으로 본다
