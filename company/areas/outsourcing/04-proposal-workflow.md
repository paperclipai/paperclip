# Outsourcing Proposal Workflow

기준일: 2026-03-16

이 문서는 `외주개발 제안이 필요한 경우 어떻게 정리하는지` 정리한 워크플로우다.

핵심은 하나다.

`제안은 길게 설명하는 문서가 아니라, 고객이 바로 이해하는 문서여야 한다.`

## 언제 제안을 쓴다

- 견적만으로 설명이 부족할 때
- 옵션 비교가 필요할 때
- 단계별 진행 구조를 설명해야 할 때
- 고객이 신뢰할 수 있는 정리 문서가 필요할 때

## 제안서에 꼭 들어갈 것

- 고객이 요청한 내용
- 우리가 이해한 범위
- 진행 방식
- 단계나 옵션
- 예상 일정
- 예상 비용
- 전제 조건

## 워크플로우

### 1. 고객 요청을 다시 정리한다

- 고객이 원하는 것
- 고객이 지금 급한 것
- 고객이 아직 확정하지 않은 것

### 2. 범위를 한 번 더 줄인다

- 지금 당장 하는 범위
- 다음 단계로 넘길 범위
- 이번 제안에서 제외할 범위

### 3. 제안 구조를 단순하게 쓴다

- 문제 이해
- 제안 범위
- 진행 방식
- 일정
- 비용

### 4. 대환이 보내기 좋은 형태로 정리한다

- 어려운 표현을 줄인다
- 기술 설명보다 고객이 이해할 말로 쓴다
- 길게 쓰지 않는다

## 한 줄 규칙

`제안은 많이 쓰는 문서가 아니라, 고객이 바로 이해하는 문서여야 한다.`
