# New Lead Intake Checklist

기준일: 2026-03-16

## 목적

새 리드가 들어왔을 때
`놓치지 않고, 무리한 약속 없이, 빠르게 신뢰를 만드는 것`

## Source Of Truth

- CRM 원본: `👟 영업 CRM`
- 첫 문의 원본: 메일 스레드 또는 문의 폼 원문
- 수주 판단 기준: [../management/07-project-intake-matrix.md](../management/07-project-intake-matrix.md)
- 현재 기회 해석: [../management/06-active-opportunities.md](../management/06-active-opportunities.md)

## D0 체크리스트

### 1. 리드 접수 직후

- [ ] 문의 원문을 끝까지 읽는다
- [ ] 아래 6가지를 바로 뽑는다
  - 이름
  - 회사/팀
  - 이메일 / 연락처
  - 무엇을 만들고 싶은지
  - 언제까지 필요한지
  - 지금 가장 급한 것이 무엇인지
- [ ] CRM에 리드를 만든다
- [ ] 상태를 `리드`로 둔다
- [ ] 마지막 연락일을 오늘로 넣는다

### 2. 첫 판단

- [ ] 이 리드가 아래 중 어디에 가까운지 본다
  - 빠른 POC형
  - 맞춤 구축형
  - 유지보수형
  - 장기 계약형
- [ ] 아래 5가지를 `높음 / 중간 / 낮음`으로 본다
  - 운영자금
  - 기술 자산
  - 비즈니스 사례
  - 고객 네트워크
  - 집중 손상 리스크
- [ ] 지금 당장 받으면 안 되는 신호가 있는지 본다
  - 범위가 너무 넓음
  - 일정이 비현실적임
  - 연구 리스크를 고정가로 떠안아야 함
  - 내부 핵심 목표를 바로 무너뜨림

### 3. 첫 회신

- [ ] 첫 회신은 `워크플로우 1단계`로 본다
- [ ] 순서는 아래 기준으로 고정한다
  - 문의를 받았다고 말한다
  - 정중하게 인사한다
  - 내가 누구인지 밝힌다
  - 고객 요청을 항목으로 다시 정리한다
  - 옵션에 따라 달라지는 범위를 짚는다
  - 추가로 필요한 정보 1~2개만 묻는다
  - 다음 회신 시점을 말한다
- [ ] 목적은 아래 4가지로 잡는다
  - 문의를 잘 받았다는 확인
  - 고객 말을 정확히 들었다는 확인
  - 우리가 이해한 범위를 다시 정리해 보여주기
  - 다음 회신 시점 안내
- [ ] 접수 당일 안에 첫 회신을 보낸다
- [ ] 첫 회신에는 가능하면 아래를 넣는다
  - 고객이 요청한 화면
  - 고객이 요청한 기능
  - 옵션에 따라 달라지는 범위
  - 추가로 받아야 하는 정보 1~2개
  - 언제 다시 회신할지
- [ ] 기본 서명은 아래로 통일한다
  - `이름` 드림
  - 비브라이트코드
- [ ] 첫 회신에서 하지 말아야 할 것
  - 바로 가능하다고 확답
  - 일정 확정
  - 금액 확정
  - 범위 확정

### 4. 내부 정리

- [ ] 리드 내용을 한 줄로 요약한다
- [ ] 받아야 하는 이유와 받지 말아야 하는 이유를 같이 적는다
- [ ] 필요한 경우 [../management/06-active-opportunities.md](../management/06-active-opportunities.md) 에 기회로 정리한다
- [ ] 필요한 경우 Cycle / Daily Focus에 액션으로 올린다

### 5. 다음 액션 결정

- [ ] 아래 셋 중 하나로 바로 정한다
  - 추가 질문 후 검토
  - 짧은 미팅 제안
  - 거절 또는 보류
- [ ] 추가 질문이 필요하면 아래를 먼저 묻는다
  - 현재 자료 또는 기존 서비스 링크
  - 원하는 일정
  - 꼭 필요한 범위
  - 예산 또는 발주 방식

### 6. 견적 초안 준비

- [ ] 견적이 필요하면 Google Drive의 `견적서 양식`으로 초안을 만든다
- [ ] 견적은 고객에게 바로 보내지 않는다
- [ ] 먼저 아래를 정리해서 대환에게 전달한다
  - 작업 범위
  - 옵션별 차이
  - 금액 초안
  - 일정 초안
  - 빠진 전제나 확인이 필요한 점
- [ ] 견적 전달 메일은 별도 `견적 초안 템플릿`을 사용한다
- [ ] 고객 발송본은 [../areas/outsourcing/06-customer-quote-delivery-workflow.md](../areas/outsourcing/06-customer-quote-delivery-workflow.md) 기준으로 만든다
- [ ] 최종 발송은 반드시 대환이 직접 한다

## 리드 대응 원칙

- 첫 회신은 `빠르게`
- 가능 여부 판단은 `천천히`
- 고객이 보기에는 `정리된 팀`처럼 보여야 한다
- 내부적으로는 `무리한 약속을 막는 팀`이어야 한다
- 견적은 먼저 만들고, 발송은 반드시 대환이 직접 한다

## 하지 말아야 할 것

- 느낌만으로 “가능합니다”라고 답하지 않는다
- 범위가 안 잡힌 상태에서 공수부터 말하지 않는다
- 전략과 안 맞는 리드를 습관적으로 끌고 가지 않는다
- 첫 회신이 늦어져 신뢰를 잃지 않는다

## 한 줄 규칙

`첫 회신은 빨리, 약속은 늦게, 판단은 문서로 남긴다.`
