# Customer Quote Edit Checklist

기준일: 2026-03-23

이 문서는 `견적서 원본 xlsx를 수정할 때 무엇을 반드시 바꿔야 하는지` 고정한 체크리스트다.

핵심은 하나다.

`견적서 수정은 감으로 하지 말고, 바뀌는 필드를 체크리스트대로 모두 확인한다.`

## 템플릿

- 원본: `company/templates/xlsx/견적서_템플릿.numbers` (Numbers 마스터)
- 작업용: `company/templates/xlsx/견적서_템플릿_placeholder.xlsx` (플레이스홀더 포함)
- 플레이스홀더 형식: `{{변수명}}` — openpyxl로 치환

## 언제 쓰는가

- 새 고객 견적서를 만들 때
- 기존 견적서를 다른 고객용으로 복제할 때
- 블로그 포함안 / 제외안 같이 복수 안을 만들 때
- PDF export 전에 빠진 값이 없는지 확인할 때

## 1. 고객 식별 정보

- [ ] 견적서 제목 고객명이 맞는지 확인
  - 셀: `A1` (머지 A1:O3) — `{{고객명}}`
  - 사람 이름이면 제목에 `님`을 붙임
  - 예: `개 발  견 적 서 - 엄희승님`
- [ ] 견적일자가 맞는지 확인
  - 셀: `A4` (머지 A4:D8) — `{{견적일자}}`
- [ ] 고객명이 맞는지 확인
  - 셀: `A4` — `{{고객명}}`
- [ ] 파일명이 포맷에 맞는지 확인
  - 포맷: `{고객명}{사람이면 "님", 회사면 생략}-개발견적서-{YYYYMMDD}.{확장자}`
  - 예: `이원재님-개발견적서-20260322.pdf`, `코메디닷컴-개발견적서-20260322.pdf`
  - `최종`, `수정본`, `new` 같은 표현은 쓰지 않음

## 2. 견적 기본 헤더

- [ ] 프로젝트명 또는 견적명 수정
  - 셀: `A11` (머지 A11:O11) — `{{프로젝트명}}`
- [ ] 공급자 정보가 의도치 않게 바뀌지 않았는지 확인
  - 구간: `E4:O8` (사업자번호, 소재지, 상호, 대표자 등)
- [ ] 템플릿 잔여 고객명이나 이전 프로젝트명이 남아 있지 않은지 확인

## 3. 본문 범위

- [ ] 1행 핵심 범위는 프로젝트명 한 줄로 작성
  - 셀: `B12`, `C12` (머지 C12:H12), `I12`, `K12`
- [ ] 항목(`B13:C23`)은 직관적이고 명료하게 작성
  - **항목별 단위금액은 넣지 않는다** — 총 공급가만 `K12`에 기재
  - **항목별 기간/개월차 설명은 넣지 않는다** — 일정 상세는 하단 `A24`에 기재
  - 항목명만 한 줄로 쓴다 (예: `시스템 설계`, `AI 에이전트 코어`, `모니터링 대시보드`)
  - 긴 부연 설명은 모두 하단 `일정 / 설명` 블록으로 내린다
- [ ] 포함 범위와 제외 범위 해석이 맞는지 확인
- [ ] 블로그 포함안 / 제외안 차이가 본문에 반영됐는지 확인
- [ ] 고객이 실제로 요청한 항목이 빠지지 않았는지 확인

## 4. 일정 / 설명

- [ ] `일정 / 설명`은 견적 항목 리스트에 넣지 않았는지 확인
- [ ] `일정 / 설명`은 푸터 위 전체 머지 메모 블록에 넣었는지 확인
  - 현재 기준 셀: `A24:O36` — `{{일정_설명}}`
- [ ] 항목별 상세 설명(기능 범위, 세부 내역)이 여기에 들어갔는지 확인
- [ ] 일정 흐름이 들어갔는지 확인 (예: `1개월차 설계 → 2~3개월차 코어 → 4개월차 통합`)
- [ ] 투입 인력 정보가 들어갔는지 확인
- [ ] 별도 청구 사항(LLM API 비용, 서버 운영비 등)이 들어갔는지 확인
- [ ] 외부 기관 심사 지연 가능성이 들어갔는지 확인
  - 예: 카카오, 결제사, 외부 연동 서비스, 도메인 연결
- [ ] 기술 용어가 과한지 확인
  - 고객이 IT를 모르면 `SSL`, `DNS` 같은 표현은 줄인다

## 5. 금액

- [ ] 공급가 금액이 맞는지 확인
  - 셀: `K12` (머지 K12:M23) — `{{공급가액}}`
- [ ] 하단 `합계 / 부가세 / 총계`가 비어 있지 않은지 확인
  - 셀: `F37`, `F38`, `F39`
- [ ] PDF export 전 아래 커맨드를 실행했는지 확인

```bash
./tools/ops/mycompany-quote-finalize /absolute/path/to/quote.xlsx
```

- [ ] PDF 변환은 Numbers 앱으로 했는지 확인 (Excel AppleScript는 매개변수 오류 발생)
- [ ] 숫자 기준이 맞는지 확인
  - `F37`: 공급가 합계
  - `F38`: 부가세
  - `F39`: 총계
- [ ] 블로그 포함안 / 제외안 등 안별 금액 차이가 의도와 맞는지 확인

## 6. 고객 발송물

- [ ] 유지보수 시트가 정말 필요한 경우가 아니면 고객 발송용 작업본에서 `유지보수견적서` 시트를 제거했는지 확인
- [ ] 기본 발송본은 실제로 `개발견적서` 1시트만 남아 있는지 확인
- [ ] xlsx 작업본 저장
- [ ] PDF export 완료
- [ ] PDF에서 하단 금액이 실제로 보이는지 확인
- [ ] 고객 발송 메일 문안이 최신 견적과 맞는지 확인
- [ ] 메일 상단은 짧은 결론, 하단은 쉬운 상세 설명 구조인지 확인
- [ ] `궁금하신 점은 얼마든지 문의 주세요` 같은 마감 문구가 들어갔는지 확인

## 7. 저장 위치

- [ ] 최종 xlsx와 PDF를 Google Drive `견적서` 폴더에 업로드
  - 폴더 ID: `1MU-4RHfxiLwcepAeAZas0hl4NvN86_Xa`
  - 새 폴더를 만들지 않는다. 이 폴더에만 업로드한다
  - gcloud ADC 인증 + google-api-python-client로 업로드한다
  - 인증이 안 되어 있으면 먼저 인증을 요청한다
- [ ] CRM 본문에는 로컬 경로 대신 Drive 링크와 파일명만 남김
- [ ] 로컬 생성 파일은 `generated/quotes/` 아래에 정리
- [ ] 다운로드 폴더 사본이 필요하면 깔끔한 파일명으로 별도 저장

## 8. CRM 업데이트

- [ ] 최종 제안 메시지를 CRM 항목 본문에 반영
- [ ] 포함안 / 제외안 금액을 CRM 본문에 반영
- [ ] Drive 폴더 링크를 CRM 본문에 반영
- [ ] 마지막 연락일 갱신
- [ ] 상태를 `제안 준비완료`로 변경

## 9. 마지막 1분 점검

- [ ] 이전 고객명 잔여 텍스트 없음
- [ ] 이전 날짜 잔여 텍스트 없음
- [ ] 금액 / 부가세 / 총계 일치
- [ ] 본문 설명과 메일 설명이 같은 안을 말하고 있음
- [ ] CRM, Drive, PDF, 메일 문안이 같은 최신 버전 기준임
