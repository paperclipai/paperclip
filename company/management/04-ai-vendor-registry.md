# AI 벤더 레지스트리

이 문서는 회사가 사용하는 AI 벤더와 시크릿 관리 기준을 정리하는 레지스트리다.

중요한 원칙은 하나다.

`실제 Key는 여기 쓰지 않는다.`

이 문서에는 `무엇을`, `어디에`, `누가`, `무슨 용도로` 쓰는지만 남긴다.

## 목적

- 어떤 AI 벤더를 실제로 쓰는지 한 번에 본다
- Key가 어디 저장되어 있는지 잃어버리지 않는다
- 개인 계정 의존, 대표 1인 의존을 줄인다
- 비용과 사용량 데이터를 어디서 가져올지 연결한다

## 기록 원칙

- 실제 Key 값은 쓰지 않는다
- 시크릿 이름과 저장 위치만 기록한다
- 용도와 연결 서비스까지 같이 적는다
- 퇴사/교체/만료 시 바로 갱신한다

## 벤더 등록 표

| 벤더 | 주 용도 | 연결 서비스/프로젝트 | 시크릿 이름 | 저장 위치 | 담당자 | 사용량 조회 경로 | 상태 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 예시: OpenAI | 내부 개발, 고객 납품 | Superbuilder, 내부 도구 | `OPENAI_API_KEY` | Vercel / 로컬 시크릿 저장소 | 대표 | 공급사 usage/billing 페이지 또는 API | 운영중 |
| 예시: Anthropic | 내부 개발, 분석 | 내부 도구 | `ANTHROPIC_API_KEY` | 로컬 시크릿 저장소 | 대표 | 공급사 usage 페이지 또는 API | 운영중 |

## 꼭 같이 적어야 할 것

### 용도

- 내부 개발
- 고객 납품
- 제품 운영
- 실험/연구
- 게임 제작

### 연결 위치

- 로컬 개발 환경
- Vercel
- GitHub Actions
- Neon 연동 서비스
- 외부 자동화 서버

### 위험 신호

- 대표 개인 계정만 알고 있는 Key
- 한 사람이 퇴사하면 끊기는 Key
- 비용은 크지만 사용처가 불명확한 Key
- 더 이상 안 쓰는데 살아 있는 Key

## 운영 규칙

- 월 1회 레지스트리를 검토한다
- 새 벤더를 쓰기 시작하면 이 문서부터 갱신한다
- 더 이상 안 쓰는 Key는 폐기 여부를 적는다
- 비용 급증 벤더는 사용량 문서와 같이 본다
