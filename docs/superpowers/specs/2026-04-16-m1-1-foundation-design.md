# RealTycoon2 M1.1 Foundation Design

> M1.1 브레인스토밍 결과를 구현 전 설계 문서로 고정한다.
> 범위: Phase 1 / M1.1 "기반 구축"
> 상태: Approved in brainstorming, implementation not started

## 1. 목표

M1.1의 목표는 기능 개발이 아니다.

`C:\Real-Tycoon 2`에서 `pnpm dev` 실행 후 브라우저에서 `http://localhost:3100` 에 접속했을 때, 경영진이 눈으로 확인 가능한 수준으로 **RealTycoon2의 독립 제품 뼈대가 시작되었다**는 점을 증명하는 것이다.

이 단계에서 증명해야 하는 것은 아래 세 가지다.

1. 로그인
2. DB 네임스페이스
3. 브랜딩

## 2. 사용자와 문제

### 직접 사용자

- 데모를 보는 경영진 1명

### 반드시 해소해야 하는 불안

- 로그인, DB, 브랜딩까지 최소 제품 뼈대가 연결됐나?

### 기존 상태

- Paperclip 원본은 뜨지만 RealTycoon2 정체성은 전혀 없다.

## 3. M1.1 완료 기준

### 완료 판정 체크 3개

1. `pnpm dev` 실행 시 서버와 클라이언트가 에러 없이 기동되고, `http://localhost:3100` 접속 시 로그인 화면이 200 OK로 표시된다.
2. 관리자 계정으로 로그인 후 빈 대시보드 진입까지 한 번의 흐름으로 성공하고, 로그인 화면과 대시보드 좌측 상단 두 군데 모두에 `리얼타이쿤2` 또는 `RealTycoon2` 브랜딩 텍스트/로고가 보인다.
3. PostgreSQL 에 `rt2_v33_` 접두사 테이블 최소 1개, 예시로 `rt2_v33_hello`, 가 마이그레이션되어 존재하고 `SELECT * FROM rt2_v33_hello;` 쿼리가 에러 없이 실행된다.

## 4. 명시적 제외 범위

M1.1에서는 아래 항목을 절대 넣지 않는다.

1. 자비스
2. One-Liner 입력
3. wikiLLM / Graphify 연동

이 세 항목은 M1.1의 성공과 무관하며, 범위 팽창만 만든다.

## 5. 설계 방향

### 선택된 접근

- 접근안: `B. 통제된 C-FULL`

### 의미

- 구조 중심은 기업 운영 플랫폼처럼 간다.
- 분위기는 AI 기반 운영체제처럼 가져간다.
- 리얼타이쿤 세계관 힌트는 얇게 넣는다.

즉, 첫인상은 강하게 RT2여야 하지만 정보 구조는 안정적으로 읽혀야 한다.

## 6. UI 설계

### 6-1. 로그인 화면

로그인 화면은 회원가입 없는 **관리자 전용 진입 화면**으로 설계한다.

필수 요소:

- `REALTYCOON2` 또는 `리얼타이쿤2` 브랜딩 타이틀
- 보조 카피: AI 운영체제 또는 경영 운영 플랫폼 느낌을 주는 한 줄 설명
- seed 관리자 계정 진입을 전제로 한 이메일/비밀번호 입력
- 회원가입 토글 제거

디자인 원칙:

- Paperclip 기본 구조를 일부 활용해도 되지만, 첫 화면에서 RT2 정체성이 분명히 보여야 한다.
- 강한 브랜드 톤은 허용하되, 이 단계에서 완성형 디자인 시스템을 만들지는 않는다.

### 6-2. 빈 대시보드

로그인 후 보여주는 첫 화면은 빈 대시보드다.

필수 요소:

- 좌측 상단 RT2 로고 또는 텍스트 로고
- “아직 기능이 비어 있는 것이 정상”이라는 문맥을 주는 empty state
- 현재 증명 항목이 부팅, 인증, RT2 DB 네임스페이스라는 점을 읽을 수 있는 최소 정보

디자인 원칙:

- 구조는 운영 플랫폼처럼 차분해야 한다.
- `Gold`, `Jarvis` 같은 세계관 요소는 직접 기능이 아니라 얇은 힌트로만 드러낸다.
- 칸반, One-Liner, OKR 위젯 등 실제 기능처럼 보이는 가짜 UI를 과하게 넣지 않는다.

## 7. 인증 설계

### 선택된 방식

- 개발 전용 seed 관리자 계정 자동 생성
- 로그인 전용 화면

### 이유

- 데모 재현성이 가장 높다.
- invite, signup, bootstrap flow까지 같이 손대면 M1.1 범위를 불필요하게 키운다.

### 요구사항

- 앱 실행 또는 초기 준비 과정에서 seed 관리자 계정이 일관되게 생성되어야 한다.
- 시드 계정 자격 증명은 RT2 개발 환경 기준으로 명확히 드러나야 한다.
- 로그인 성공 후 기본 랜딩은 빈 대시보드로 고정한다.

## 8. DB 설계

### 선택된 방식

- 외부 PostgreSQL 17 연결을 M1.1부터 고정

### 이유

- `rt2_v33_` 네임스페이스 증명이 가장 명확하다.
- 이후 Phase 1 데이터 모델 확장 시 경계가 깔끔하다.

### 요구사항

- Paperclip 기본 마이그레이션 이후 RT2 전용 마이그레이션을 독립 추가한다.
- M1.1에서는 upstream 기본 테이블 수정 없이 RT2 더미 테이블만 추가한다.
- 예시 테이블은 `rt2_v33_hello` 로 둔다.

### 검증 쿼리

```sql
SELECT * FROM rt2_v33_hello;
```

이 쿼리는 에러 없이 실행되어야 한다.

## 9. 환경변수와 실행 경계

M1.1에서 최소한 아래 설정 경계는 명시적으로 관리해야 한다.

- `DATABASE_URL`
- 인증 시크릿 계열 값
- seed 관리자 이메일/비밀번호
- RT2 브랜딩 문자열 또는 브랜딩 설정 값

원칙:

- “기본값이 우연히 맞아서 동작”하는 상태를 허용하지 않는다.
- RT2 실행 조건은 명시적으로 보여야 한다.

## 10. 브랜딩 적용 경계

M1.1에서 브랜딩을 강하게 보여줘야 하는 핵심 지점은 아래 두 군데다.

1. 로그인 화면
2. 로그인 후 대시보드 좌측 상단

M1.1에서 하지 않을 것:

- 전체 앱의 모든 문구 전면 치환
- 전체 메뉴 구조 RT2식 재명명
- 완성형 로고 시스템 구축
- 전체 디자인 시스템 재구축

즉, **핵심 접점 두 군데만 강하게** 바꾼다.

## 11. 4주 청크 분할

### 청크 1. 실행 기반 고정

목표:

- `pnpm dev` 실행 조건 고정
- 외부 PostgreSQL 17 연결 확인
- RT2 전용 환경변수 뼈대 확정

### 청크 2. RT2 네임스페이스 증명

목표:

- `rt2_v33_hello` 테이블 생성 및 조회 성공
- RT2 독립 네임스페이스 확인

### 청크 3. 관리자 진입 루프 고정

목표:

- seed 관리자 계정 자동 생성
- 로그인 전용 화면 구축
- 로그인 후 빈 대시보드 진입 확인

### 청크 4. C-FULL 브랜딩 적용

목표:

- 로그인 화면 RT2 브랜딩 적용
- 대시보드 좌측 상단 RT2 브랜딩 적용
- 빈 대시보드 톤을 선택된 방향으로 마감

## 12. 파일 경계 원칙

사용자 황금룰:

- 파일당 500라인 절대 초과 금지

따라서 M1.1 구현은 아래처럼 분리되어야 한다.

- 브랜딩 로직 분리
- seed 관리자 로직 분리
- RT2 마이그레이션 분리
- 환경설정 분리

큰 파일 하나에 덧붙이는 방식은 허용하지 않는다.

## 13. 리스크와 대응

### 리스크 1. seed 관리자 생성 시점 불명확

대응:

- 앱 시작 전 준비 단계인지, 서버 부팅 시점인지, 별도 스크립트인지 명시적으로 고정한다.

### 리스크 2. 마이그레이션 순서 혼선

대응:

- 기본 마이그레이션 뒤에 RT2 전용 마이그레이션이 적용되도록 순서를 명확히 설계한다.

### 리스크 3. 브랜딩 범위 팽창

대응:

- 로그인 화면과 좌측 상단 두 지점만 M1.1 필수 브랜딩 범위로 본다.

## 14. 구현 전 확인 질문

구현 계획 단계에서 반드시 잠가야 할 항목:

1. seed 관리자 계정 생성 방식과 실행 시점
2. 외부 PostgreSQL 17 연결 방법
3. `rt2_v33_hello` 마이그레이션 파일 위치와 적용 순서
4. 로그인 화면과 대시보드 브랜딩을 어느 파일까지 건드릴지

## 15. 최종 요약

M1.1은 “기능이 없는 것이 정상”인 단계다.

하지만 아래는 반드시 보여야 한다.

- RT2 이름으로 로그인할 수 있다
- RT2 이름으로 빈 대시보드에 들어간다
- RT2 이름으로 분리된 DB 네임스페이스가 실제 존재한다

이 세 가지가 연결되면, 경영진에게는 “이제 RealTycoon2가 시작됐다”는 메시지가 전달된다.
