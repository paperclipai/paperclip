# 054 · 运行详情叙事流 — 技术设计（2026-05-20）

**读者：** 编排者 / 董事会（根据 **一轮 run** 调衡唤醒与 prompt 叠层）。  
**入口：** 运行清单 **`/runs/:runId`**（主）；智能体 **`/agents/:agentId/runs/:runId`** 第二阶段对齐同一叙事（见 §8）。

**关联：** [050 · 唤起溯源 UI](../执行/050-运行时间线唤起溯源与档位-UI.md) · [060 §7.1–7.2](../执行/060-2026-05-20-验证闭环与编排首日执行.md) · [061 · 执行单](../执行/061-运行详情叙事流-UI.md)

---

## 1. 问题陈述（董事会口径）

现有运行详情 **信息齐全但交互难用**：

- **五 Tab**（唤起入队 / 输入编排 / 最终提示词 / 执行回传 / 运行记录）按「取证维度」切分，**拼不出一轮 run 的来龙去脉**。
- **唤起入队** 近乎占位，答不了「这轮怎么来的」。
- **输入编排** 与 **最终提示词** 重复展示同一段 prompt。
- **执行回传** 过薄（常仅一张适配器卡），**运行记录** 过厚（嵌整页 `HeartbeatRunDetailPanel`），且与智能体页 **双份 UI**。

**目标：** 默认一屏讲清 **来源 → 编 → 跑 → 果**；**原始 / 友好转写 / 最终提示词全文 / JSON** 收进 **可点开** 的深读层，服务「调编排」而非「翻抽屉」。

---

## 2. 设计原则

| # | 原则 |
| --- | --- |
| P1 | **叙事优先于 taxonomy** — 纵向四段顺序固定，不用平级 Tab 当主路径 |
| P2 | **调编排可读** — 每段回答「我要改什么」：唤醒策略、模板块、适配器、续跑省略 |
| P3 | **深读默认折叠** — 全文 prompt、原始日志、大块 JSON 不进首屏 |
| P4 | **单一真源** — 最终提示词全文 **只在一处** 维护（深读层）；概览只显示块清单与摘要 |
| P5 | **不新增 API** — 复用 `heartbeatsApi.get`、`heartbeatsApi.events`、现有 `contextSnapshot` / `adapter.invoke` 解析 |
| P6 | **中文产品说明** — 界面用正式、可扫的产品/运维文案（见 §2.1）；**不是**口语聊天，也**不是**裸英文术语 |

### 2.1 界面文案：「说人话」= 中文产品说明

董事会口径（2026-05-20）：

| 要 | 不要 |
| --- | --- |
| 中文产品说明、运维说明式短句 | 口语（「边跑边吐」「跟得上」「干等」） |
| 英文 **identifier** 保留时配中文：**流式输出**、**提示词段**、**首轮引导（bootstrap）** | 裸写 `adapterConfig`、`suppress`、`handoff`、`resultJson` 当主文案 |
| 参数名 `--print`、`stream-json` 可保留在「参数 / 取值」列 | 把整段说明写成英文或开发字段名 |
| 一行内说清「是什么 / 干什么」 | 小作文、外链文档才能懂 |

**三层：**

1. **概览（默认可见）** — 产品说明中文；技术 id 仅作括号或次要列。  
2. **深读** — 可含 JSON、原始命令行、英文日志；入口用中文（如「运行快照（JSON）」）。  
3. **开发 catalog** — 适配器包内登记参数说明；run 页只消费中文结果，不暴露内部文件名。

## 3. 页面结构（方案 A · 已定案）

### 3.1 主路径：单页四段（概览）

默认 URL：`/runs/:runId`（无 `tab` 或 `view=overview`）。  
页内 **锚点**：`#source` · `#input` · `#execution` · `#outcome`（侧栏或段首 sticky 小目录可选，窄屏可省略）。

```text
┌─────────────────────────────────────────────────────────┐
│ 顶栏：面包屑 · 终态/执行智能体/适配器（带标签）· [深读 ▾] │
├─────────────────────────────────────────────────────────┤
│ §1 来源 (#source)                                    │
│ §2 输入编排 (#input)                                     │
│ §3 执行与回传 (#execution)                               │
│ §4 结果 (#outcome)                                       │
└─────────────────────────────────────────────────────────┘
         │ 点击「深读」或段内「查看详情」
         ▼
┌─ Sheet / 全屏 Dialog / 页内 Collapsible ────────────────┐
│ 最终提示词 | 友好转写 | 原始 | contextSnapshot | …       │
└─────────────────────────────────────────────────────────┘
```

**移除：** 五个平级 `TabsTrigger` 作为主导航。  
**保留（可选）：** URL `?expert=prompt|transcript-friendly|transcript-raw|json` 便于分享深读态；默认仍打开概览。

#### 顶栏摘要（两层，勿混）

| 层 | 位置（代码） | 放什么 |
| --- | --- | --- |
| **① 全局面包屑** | `Layout` → `BreadcrumbBar` | 工作 / 运行清单 / 运行详情 · runId 前 8 位（**已有**，`setBreadcrumbs`） |
| **② 页面顶栏** | `OrchestrationInjectionRunDetail` 内 **`main` 最上、§1 之前** | 终态 · 执行智能体 · 适配器 · 深读 / 复制运行 ID |

**现网缺口：** 只有 ①，没有 ②；点进详情直接是 Tab，所以「成功 / T5 / CodeBuddy」**目前页面上没有固定位置**。061 要补的是 **②**。

**勿重复：** §1 来源写 **唤醒/派工叙事**；智能体名已在 ② 展示，§1 **不再**单列一行智能体 KV。

### 3.2 §1 来源（`#source`）

**回答：** 本轮 run **从何而来** — 谁、为何、链到哪。

| 块 | 内容 | 数据来源 |
| --- | --- | --- |
| **一句摘要** | 产品说明式主因（评论 / 分配 / 手动 / 定时 / 续跑…） | `effectiveTrigger`、`invocationSource`、`contextSnapshot.wake*`；对齐 [050 §产品陈述](../执行/050-运行时间线唤起溯源与档位-UI.md#产品陈述唤起归因应回答什么2026-05-20-董事会口径) |
| **派工细节** | 人类 / 智能体 / 系统；为何派（建单、改经办、接力…） | `triggerDetail` 汉化；`wakeTriggerDetail`；invoke 操作者（若有） |
| **合并** | 仅当 `absorbed.length > 0`：被挤掉的触发 + 时间 | `effectiveTrigger.absorbed` |
| **单源降级** | 无合并时 **不用**「合并后认账」标题 → 改为「本次叫醒」 | UI 文案规则（060 §7.1） |
| **关联链** | 事务 · 评论 · 智能体 · 重试/续跑自哪条 run | `contextSnapshot.issueId`、`wakeCommentId`；`describeRunRetryState` |
| **运行时长（首屏可见）** | 已开始 · 已结束 · **时长** · 运行中则实时计时 | `startedAt` / `finishedAt`；与 §4 不重复堆字段，此处只一行 |

**禁止：** 仅占位 + 链到编排闸门（闸门链收到折叠「未入队说明」或 footer 小字）。

### 3.3 §2 输入编排（`#input`）

**回答：** 控面 **怎么拼** 这轮输入 — **不是** 贴全文 prompt。

| 块 | 内容 | 数据来源 |
| --- | --- | --- |
| **cache / 续跑** | 冷启动 vs 续跑；`stabilityKey`；省略了哪些 section | `adapter.invoke` → `promptCacheCorrelation` |
| **块清单（核心）** | 表格式：**提示词段**（`promptSection.id`，见 §10）· 本 run **带入/省略/指针** · 体量 · 一行摘要 | `promptSections[]`；标签 **`promptSectionTitles`**（i18n，**中文（id）**） |
| **指标胶囊（可选）** | token 分块计量等 | `promptMetrics` |
| **入口** | 按钮 **「查看最终提示词全文」** → 深读层 `expert=prompt` | 现有 `fullPromptMarkdown` 拼装逻辑 |
| **入口** | **「查看唤醒载荷 / 上下文快照」** → 深读 JSON | `contextSnapshot`、`paperclipWake` |

**禁止：** 概览段内 `max-h` 滚动的大段 Markdown（迁入深读层）。

### 3.4 §3 执行与回传（`#execution`）

**回答：** 适配器 **怎么跑**、输出如何回来。

| 块 | 内容 | 来源 |
| --- | --- | --- |
| **命令拼装** | 基座 + **逐参数表**（§11）+ `commandNotes` | `adapter.invoke` → `command`、`commandArgs` |
| **运行中** | 已运行 · **距上次输出** 静默（059） | `lastOutputAt` / live poll |
| **转写摘要** | 友好模式 **最后 N 行** + 「查看完整转写」 | 从 `HeartbeatRunDetailPanel` **抽出** |
| **操作** | 取消（运行中）· 重试 / 续跑（失败） | 本节底部 |

深读：完整 argv · `transcript-friendly` · `transcript-raw` · 完整事件列表。

### 3.5 §4 结果（`#outcome`）

**回答：** 这轮 **交回什么**、行级事实。

| 块 | 内容 |
| --- | --- |
| **终态** | status · exitCode · 错误码/摘要 |
| **用量** | input/output/cached tokens · 费用（若有） |
| **结构化回传** | `resultJson` **摘要**（键列表或首段 preview）+ 「查看完整 JSON」 |
| **关联** | 触及的事务列表 · 链到智能体运行页（次要入口，非主路径） |

**禁止：** 再 mount 整页 `HeartbeatRunDetailPanel`。

## 7. 验收标准

1. 打开任意 succeeded run：**不滚动** 即可读到 — 怎么来的 · 跑了多久 · 带了哪些 **提示词段**（含省略）· 终态与回传摘要。
2. 最终提示词全文 **仅** 在深读层出现一次；概览 §2 无大段 Markdown。
3. 运行中 run：§3 显示静默旁路与取消；结束后 §4 显示用量与 result 摘要。
4. ROU-6 类单源 run：§1 **不出现**「合并后认账」误导文案（060 §7.1）。
5. `?tab=finalPrompt` 等旧链可用或平滑重定向。
6. 单测：`run-source-narrative.test.ts`、`prompt-blocks.test.ts` 覆盖至少 ROU-6 fixture。

## 10. 术语：「块」→ 提示词段（promptSection）

| 层级 | 用词 | 说明 |
| --- | --- | --- |
| **代码 / API** | `promptSections[]`、`AdapterPromptSection.id` | `packages/adapter-utils` · `adapter.invoke` 事件 payload |
| **协作文档** | **stdin 拼装段**、**D 层** | 见 [022 §3](../最佳实践/022-实践-无头编排提示词叠层与可观测性.md) |
| **界面列名** | **提示词段** | 不叫「块」——避免与 UI 组件块、JSON 块混淆 |
| **单元格展示** | **中文（英文 id）** | 复用 `orchestrationInjectionPage.promptSectionTitles`；命令名称如 **智能体设定**（`AGENTS.md` / `--system-prompt-file`），**不用**「默认指令」（易误解为出厂缺省） |

**状态列取值：** **带入** · **省略**（`suppressedSectionIds` / 0 字）· **指针**（如 skill minimize 单行）— 与 `promptCacheCorrelation` 并排说明。

## 11. 适配器 CLI 参数说明表（与运行详情联动）

**动机（董事会）：** 改 adapter 加 `--foo` 时，run 详情应 **自动解释** 该参数含义，不必翻腾讯/Qwen 外链文档；与 **编排闸门**、**心跳清单** 同逻辑——**控制面可见、可对照代码调参**。

### 11.1 真源位置（建议）

| 层 | 路径 | 职责 |
| --- | --- | --- |
| **Catalog（真源）** | `packages/adapters/<adapter>/src/cli-arg-catalog.ts`（或 `command-arg-catalog.ts`） | 该适配器 **会拼进 invoke 的 argv** 的说明 |
| **聚合（可选）** | `packages/adapters/*/src/index.ts` 导出 · 或 `server` 只读 API | UI / 智能体配置页共用 |
| **UI 消费** | run §3 **命令拼装表** · 未来 **适配器配置表单** | 读 catalog + 本 run 的 `commandArgs` 取值 |

**条目字段（最小）：**

```ts
{
  nameZh: "输出格式",                  // 命令名称列（表首列）
  flag: "--output-format",
  summaryZh: "流式输出：执行期间持续写入日志",
  valueEnum?: { "stream-json": "流式输出" },
  sourceZh: "适配器配置；未改时用 stream-json",  // 智能体设定行用「智能体 · 设定」
  docRef?: "docs/适配器/10 …"          // 可选，用户不必点
}
```

**界面文案纪律：** 见 **§2.1**；命令 catalog 的 `summaryZh` / `sourceZh` 同样遵守。

### 11.2 Run 详情 §3 怎么展示

1. **一行摘要：** 「CodeBuddy 无头单次执行 · 流式输出 · 模型 glm-5.1 · 工作目录为本仓库根」  
2. **参数表列顺序：** **命令名称** → **参数** → **本 run 取值** → **说明** → **来源**（**智能体 · 设定**、**智能体配置**、**适配器配置**、**代码固定**；禁止裸开发字段名）。

| 命令名称 | 参数 | 本 run 取值 | 说明 | 来源 |
| --- | --- | --- | --- | --- |
| 单次打印 | `--print` | 开 | 单次执行结束后进程退出，便于控制面回收 | 代码固定 |
| 输出格式 | `--output-format` | `stream-json` | **流式输出**：执行期间持续写入日志，运行详情可同步呈现 | 适配器配置；未改时用代码里的 stream-json |
| 选择模型 | `--model` | `glm-5.1` | 本次运行使用的模型 | 智能体配置 |
| 流式半包 | `--include-partial-messages` | 开 | 允许推送未完整的中间消息，降低流式延迟 | 选用流式输出时自动追加 |
| 智能体设定 | `--system-prompt-file` | `AGENTS.md` | 设定包（人设、规则等），冷启动注入入口文件 | **智能体 · 设定**（与 `/agents/:id/instructions` 同源） |

3. **`commandNotes`** 有则附在表下，一行一条。  
4. **未登记参数**（你在适配器新加的 `--xxx` 还没写说明）：表里仍显示 argv，来源列写 **「未登记说明」**（开发侧去适配器包补登记；用户不看到 `extraArgs` / `cli-arg-catalog` 字样）。  
5. **深读入口**文案：**「完整命令行与工作目录」** — 不是「完整 argv 与 cwd」。

### 11.3 与「适配器 UI 配置化」路线

与 run **原子叙事** 对称：

| 对象 | 可见性 | 配置化（后续） |
| --- | --- | --- |
| **Run** | 061 叙事流四段 | — |
| **适配器 invoke** | §3 命令表 + catalog | 智能体页 adapterConfig 表单 **字段 ↔ catalog 同源** |
| **解析器** | 047/027 方向 | stdout 解析规则可见（另单） |
| **编排闸门 / 心跳清单** | 已有 Board 叙事页 | 代码开关 ↔ 文案对照 |

**061 M2** 可先做 **CodeBuddy catalog + run 表**；全适配器与配置 UI **另开执行单（建议 062）**，catalog 设计一次定稿。

### 11.4 与智能体「设定」页同源

`--system-prompt-file` 注入的是 **instructions-bundle** 的**入口文件**（常为 `AGENTS.md`），内容域与 Board **`/agents/:id/instructions`**（原 Tab「指令」，产品文案 **「设定」**）一致，可含人设、规则、工具说明等多文件包。

**Run §3 要求：**

- 命令名称：**智能体设定**
- **来源**列链到该智能体设定页；取值列展示**本 run 实际注入的入口文件名**
- 不在 run 页重复编辑设定全文 — 编辑仍在设定页
