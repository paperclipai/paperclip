# 执行任务单 #029 — 设置页全量汉化批量执行

## 可复用打法

共性步骤见 **[011-实践-回形针UI页面汉化流程.md](../最佳实践/011-实践-回形针UI页面汉化流程.md)**（本单为 **设置页批量** 实例）。

## 范围

- **场景**：Paperclip UI 中公司设置（General / Environments / Access / Invites / Secrets / Skills）、实例设置（Profile / General / Access / Heartbeats / Experimental / Plugins / Adapters）共 **16 个页面组件 + 2 个侧边栏组件** 的用户可见英文文案。
- **排除**：技术占位符、API 返回动态值、AI 智能体生成文本、`data-testid`、类名、URL、包名。

## 前置条件

| 条件 | 状态 |
|------|------|
| 翻译字典文件已确认 | `ui/src/lib/i18n.ts`（~2157 行 const 对象，纯 zh-CN） |
| 目标文件清单明确 | 用户指定 16 个组件文件 |
| 无 i18next 框架 | 轻量方案：一个 i18n.ts 放所有 const 字典 |
| TypeScript 编译可运行 | ✅ |

## 未处理残留（有意保留英文）

| 类型 | 示例 | 原因 |
|------|------|------|
| 技术占位符 | `OPENAI_API_KEY`、`/absolute/path/to/folder`、`@paperclipai/plugin-example` | 配置示例值，非用户可见文案 |
| AI 智能体生成文本 | `buildAgentSnippet` 函数内容 | 给 OpenClaw 智能体用，非人类 UI |
| 运行时动态值 | API 返回的 provider 名称、健康状态详情、绑定目标类型 | 需后端或映射层支持 |
| 外部链接 | skills.sh、GitHub 搜索 URL | 链接地址 |
| 技术单位 | MiB、PID、RPC | 国际通用技术术语 |
| 代码标识符 | `data-testid`、CSS 类名、API key | 非用户可见 |

## 踩坑与经验

| 坑 | 经验 |
|---|------|
| 一开始搜 i18next/useTranslation 找不到，误判"没有 i18n" | 轻量方案可能只是一个 `i18n.ts` const 字典，搜中文字符 `[\u4e00-\u9fff]` 更可靠 |
| 文件太大（800+ 行），grep 搜索结果太多无法精确定位 | 用 `Explore` agent 做全文件扫描，返回结构化清单 |
| 16 个文件串行执行太慢 | 拆分成 5 个 agent 子进程并行处理，效率提升约 3 倍 |
| `companyAccessPage.cancel` 和 `unknownError` 初始遗漏 | 在 agent 执行时发现 key 缺失，动态补加到 i18n.ts |
