## 1. Jarvis（个人 AI 运维控制台）

**路径**：`C:\Users\wuhen\code\jarvis`
**类型**：应用（Application）— 本地运行的 Web 应用

### 1.1 定位

为 token-bridge-v2 项目提供：
- AI 聊天助手（CodePlan Qwen 主力 / CodeBuddy 降级）
- 记忆工程管线编排（提取 → 转录 → 生成 → 进化）
- 今日待办 / 项目状态一览
- 收件箱 / 报告查看

### 1.2 技术栈

| 层 | 技术 |
|---|------|
| 前端 | React 19 + Vite + TypeScript，纯 CSS（无 Tailwind/Shadcn/Redux），lucide-react, react-markdown |
| 后端 | Express.js 5 + tsx，端口 **8787** |
| 数据库 | SQLite（`%APPDATA%\jarvis\jarvis.db`） |
| AI 调用 | @tencent-ai/agent-sdk（CodePlan Qwen，端点 `coding.dashscope.aliyuncs.com`）+ CodeBuddy SDK 降级 |
| 语音 | Web Speech API（TTS 已升级为 Edge TTS） |
| 构建 | TypeScript + Vite build |

### 1.3 本地启动

```bash
npm run dev        # 前端(:2345) + 后端(:8787) 同时启动
npm run dev:api    # 仅后端
npm run dev:web    # 仅前端
npm run build      # 生产构建
```

- 前端 Vite dev server：`http://127.0.0.1:2345/`
- 后端 Express API：`http://127.0.0.1:8787/`
- 前端 `/api` 路径通过 Vite 代理转发到 8787

### 1.4 执行环境

|------|---|

| 部署 | 本地开发模式，无 Docker；`.vercel/project.json` 存在但未配置 Vercel |

| `DASHSCOPE_API_KEY` | `sk-sp-2b15...` | CodePlan Qwen 聊天 |

## 2. Token Bridge v2（LLM API 网关桥接平台）

**路径**：`C:\Users\wuhen\token-bridge-v2`
**类型**：平台（Platform）+ 业务产品 — 面向企业的多租户 SaaS

### 2.1 定位

供应商渠道管理、计费结算、用量统计、用户控制台、管理员后台。

### 2.2 技术栈

| 层 | 技术 |
|---|------|
| 后端 API | **Go 1.23** + Gin，Postgres（pgx v5）+ Redis + RabbitMQ，端口 **5400** |
| Console（用户侧） | **Next.js 16** + React 19 + TypeScript + Tailwind + shadcn/ui，端口 **5500** |
| Admin（管理后台） | React 18 + Vite + react-admin + MUI + Recharts，端口 **5300** |
| WWW（官网） | Next.js + React（静态导出），端口 **5200** |
| 网关 | LiteLLM（本地从源码启动，端口 **5600**） |
| 数据库 | Postgres 16（业务库 `token_bridge_v2` + LiteLLM 专用库 `litellm`） |
| 缓存/队列 | Redis 7 + RabbitMQ |
| E2E 测试 | Playwright |
| 包管理 | pnpm monorepo（`apps/*` + `packages/*`） |

### 2.3 本地启动

```bash
# 第一步：基础设施容器（Postgres + Redis + RabbitMQ + LiteLLM-DB）
pnpm mvp:env              # 或 pnpm env:full

# 第二步：应用进程（另开终端）
pnpm dev:www              # 官网 :5200
pnpm dev:admin            # 管理后台 :5300
pnpm dev:console          # 用户控制台 :5500
pnpm api:dev              # Go API :5400

# LiteLLM 网关（WSL 或本机另起）
pnpm litellm:local-dev    # :5600

# 健康检查
pnpm local:health
pnpm env:status
```

**关键架构**：local-dev 默认经 **Nginx 反代**（端口 **8080**），浏览器访问 `*.tokenbridge.local:8080`。

```bash
pnpm dev:stop    # 停止服务
```
