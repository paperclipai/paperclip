## 1. Contexto

### 1.1 Por que P2.5 existe

ClickUp separa **AI Hub** (jobs, avg cost, schedules dos agentes) de **Workload** (capacidade humana). Gap explícito do Ciclo 1B: **não unifica** capacity humano+IA.

Oportunidade Paperclip (D-09, D-13): um **Hybrid Team Panel** com:

1. Roster unificado (humanos + agentes como “colegas”)  
2. Capacity lanes (Workload-like) no **mesmo** produto  
3. Status operacional (idle / running / paused / offline)  
4. Membership (company + room/channel)

BoardChat continua sendo a **sala**; Team Panel é a **gestão do time** — não substituir o stream.

### 1.2 Problema

| Superfície hoje | Limitação |
|-----------------|-----------|
| `Agents.tsx` | Só IA |
| `CompanyAccess.tsx` / invites | Só humanos |
| `OrgChart.tsx` | Hierarquia agentes; humanos fracos |
| `ActiveAgentsPanel.tsx` | Runs ativas, não workforce |
| `UserProfile.tsx` | Perfil isolado |
| `Dashboard.tsx` | Métricas, não roster acionável |

### 1.3 Escopo

| Dentro P2.5 | Fora P2.5 |
|-------------|-----------|
| Página/aba **Team** com roster híbrido | Dual performance charts densos (P4.5) |
| Capacity lanes v1 (contagem / WIP) | Forecasting ML de carga |
| Status + filtros | Editar Org chart completo (reusa página Org) |
| Membership company + “na Room X” | HRIS sync / payroll |
| Deep-links Ask / Profile / AgentDetail / Routines | Ambient chat |

### 1.4 Cenário canônico

```
Board abre /team:
  Lanes:
    [Humans] Sofia (owner×3) · Carlos (owner×1)
    [Agents] CEO (runs:1, $12/d) · Dev (paused budget) · Researcher (idle)
  Clica CEO → drawer: status, avg cost 7d, routines, [Pedir ao agente], [Abrir Room]
```

### 1.5 Premissas

- Dados de agents, members, costs summaries, heartbeat runs já existem.  
- Capacity v1 = **WIP counts** (issues owned / runs active), não calendário full ClickUp.  
- Feature flag `enableHybridTeamPanelV1`.

### 1.6 Glossário

| Termo | Definição |
|-------|-----------|
| **Roster entry** | União discriminada `kind: "human" \| "agent"` |
| **Capacity lane** | Faixa visual por pessoa/agente com WIP |
| **Room membership** | Quem pode ver/postar/ser mencionado na Conference Room |
| **Status** | `offline \| idle \| busy \| running \| paused \| error` |

## 6. UX

### 6.1 Layout

```
┌ Team ──────────────────────────────┐
│ [All] [Humans] [Agents]  🔍 Search │
│                                    │
│ ▸ Lanes view (default)             │
│   Sofia ████░░ 3/5                 │
│   CEO   ██░░░░ 1/3  $4.20 today    │
│                                    │
│ ▸ Table view (toggle)              │

```

Uma composição: **não** dashboard de 12 widgets. Roster + lanes = um job.

### 6.2 Drawer de detalhe

- Humano: role, issues abertas, link profile.  
- Agente: adapter, status, cost7d, routines count, room membership toggle, Ask.

### 6.3 Sofia vs Board

| | Sofia | Board |
|-|-------|-------|
| Cost | “~$ hoje” opcional | breakdown + link Costs |
| IDs | ocultos | visíveis |
| Membership edit | read-only | edit allowlist |

### 6.4 A11y

- `aria-label` por lane (“CEO, 1 de 3 em progresso”).  
- Status não só por cor (ícone + texto).


### 7.4 Derivação de status (normativa)

```ts
function agentStatus(a): Status {
  if (a.paused || a.budgetBlocked) return "paused";
  if (a.lastAdapterError) return "error";
  if (a.activeRunCount > 0) return "running";
  return "idle";
}
```
