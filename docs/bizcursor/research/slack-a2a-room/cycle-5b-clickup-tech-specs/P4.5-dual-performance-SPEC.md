## 1. Contexto

### 1.1 Por que P4.5 existe

Ciclo 1B: métricas **dual** — Outcome + Collaboration + Reliance + Agent health + Cost + Human orchestration + Risk (Magentic / Co-Gym / McKinsey / Deloitte / Google Cloud).

Decisão **D-11**: painéis de performance **fora do stream** (aba Team / Insights), dual **Humano \| Agente \| Room**.

P4 já mostra custo na Room; P5-R conta eventos da sala. P4.5 **consolida** prova semanal de valor para:

| Persona | Precisa |
|---------|---------|
| **Sofia** | Digest curto: “esta semana a sala economizou X handoffs / custou $Y” |
| **Board** | Vista densa: breakdown human vs agent vs room, trends, risco |

### 1.2 Problema

| Hoje | Gap |
|------|-----|
| `Dashboard.tsx` / `DashboardLive.tsx` | Foco company/agents genérico — não dual hybrid |
| `Costs.tsx` | $ sem collaboration/reliance |
| P5 room metrics | Podem existir sem superfície “human orchestration” |
| Activity | Feed, não KPI |

### 1.3 Escopo

| Dentro P4.5 | Fora P4.5 |
|-------------|-----------|
| Página **Insights** / seção Dual Performance | Performance review RH / stack ranking punitivo |
| Dimensões human \| agent \| room | ROAS marketing claims |
| Sofia weekly digest (email/in-app) | ML forecasting |
| Board dense tables + export CSV | Substituir Costs page |
| Reuso costs + room metrics + activity | Novo billing |

### 1.4 Princípio anti-hype

> Medimos **orquestração e custo do ciclo**, não “produtividade moral” do humano.  
> Labels: “Human orchestration” — **nunca** “employee score”.

### 1.5 Glossário

| Dimensão | O que mede (v1) |
|----------|-----------------|
| **Room** | mentions, fan-outs, join success %, $/session, HITL wait time (se P3) |
| **Agent** | runs, success rate, $/agent, pause/budget incidents, avg latency |
| **Human orchestration** | work requests criados, approvals respondidos, time-to-approve, issues owned closed with agent delegate |
| **Collaboration** | % threads com ≥1 humano + ≥1 agente |
| **Reliance** | % issues com delegate agent vs só humano |
| **Risk** | budget 80/100 hits, failed joins, policy blocks |
