## 1. Purpose (por que P2.5 existe)

### 1.1 Problema de produto

ClickUp **confirma** (Cycle 2C) que:

1. **AI Hub** lista Super Agents com Schedules, Avg Cost (USD), # of Jobs, Jobs in Progress.  
2. **Workload** modela capacidade **humana** (assignee + limits, green/yellow/red) e **não** unifica capacity de agentes de IA na mesma vista.  
3. O gap AI Hub ≠ Workload é o diferencial Path B+ (D-09, D-13 LOCKED; R-03 PROMOTED).

Hoje no fork Paperclip as superfícies estão **separadas** como no ClickUp:

### 1.2 Job-to-be-done (Sofia)

> “Ver **quem** (humano ou agente) está no time, **o que** cada um carrega, **custo/jobs** dos agentes, **carga** dos humanos, e **agir** (convidar, adicionar, pausar, rebalancear) sem abrir o stream da Room.”

### 1.3 Decisões LOCKED consumidas

| ID | Texto | Implicação P2.5 |
|----|-------|-----------------|
| **D-09** | Path B+ (Room + Hybrid Team & Performance) | Team é first-class, não “nice-to-have” |

### 1.5 Cenário canônico (vertical Software House)

```
EM / Sofia abre /company/acme/team (flag on):

  Insights: Hybrid CT 2.1d · Co-touch 41% · Overload 1

  Roster (seleção sincroniza lane + drawer):
    Sofia · owner · busy          → lane 4/5 WIP · CT p50 1.8d
    João  · member · available    → lane 3/5
    ─────────────────────────────
    @CEO  · ceo · available       → 1/2 jobs · avg $12 · interv 0
    @Dev  · eng · overloaded      → 2/2 jobs · avg $41 · interv 2 ⚠
    @QA   · qa · available        → 0/2 jobs · avg $3

  Sofia seleciona @Dev → drawer:
    [Pedir ao agente] [Pausar] [Routines] [Custos] [Abrir Room @Dev]
    Copy: "Delegado a @Dev · Owner Sofia" (D-12)

  Rebalance (F-T4): seleciona @QA → Pedir ao agente (P1.5) com delegate=@QA
  Room stream permanece sem widgets de performance (D-11).
```

### 1.6 Premissas

- P0: agents, company members, room mention allowlist, feature-flag infra.  
- P2: runs `running|queued` e DelegationTrace correlacionável — lanes “busy” honestas.  
- P1.5: `WorkRequestDialog` / Ask button disponível para integração drawer (soft-gate: se P1.5 atrasar, deep-link `AgentDetail.tsx` + Room `@agent`).  
- Capacity v1 = **contagens WIP / jobs**, não calendário full.  
- Adapters beachhead: `cursor_cloud`, `opencode_local`.

### 1.7 Glossário

| Termo | Definição |
|-------|-----------|
| **Roster entry** | União discriminada `kind: "human" \| "agent"` com `principalId` estável |
| **Capacity lane** | Faixa visual por principal com progressbar + métricas P0 por kind |
| **Insights strip** | 2–4 KPIs Room (Hybrid Cycle Time, Co-touch, Overload count) — D-11 |
| **Detail drawer** | Painel lateral (desktop) / bottom sheet (mobile) do selecionado |
| **Owner** | User humano accountable (D-12) |
| **Delegate** | Agente executor (não assignee tradicional) |
| **Soft cap** | Limite visual (default humano 5, agente `maxConcurrentRuns` ou 2) — não hard-block |

### 8.3 Derivação de status (normativa)

```ts
function deriveHumanStatus(h: {
  invitePending?: boolean;
  capacityLoadPct: number;
  wipCount: number;
  softCap: number;
}): UnifiedStatus {
  if (h.invitePending) return "away";
  if (h.wipCount > h.softCap) return "overloaded";
  if (h.capacityLoadPct >= 70) return "busy";
  return "available";
}

function deriveAgentStatus(a: {
  paused?: boolean;
  budgetBlocked?: boolean;
  budgetUsagePct?: number | null;
  lastAdapterError?: string | null;
  jobsInProgress: number;
  softCap: number;
}): UnifiedStatus {
  if (a.paused || a.budgetBlocked) return "away";
  if (a.lastAdapterError) return "error";
  if (a.jobsInProgress > a.softCap || (a.budgetUsagePct ?? 0) >= 80) {
    return "overloaded";
  }
  if (a.jobsInProgress >= 1) return "busy";
  return "available";
}
```
