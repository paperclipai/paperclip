## 1. Contexto e visão ampla

### 1.1 Por que P4.5 existe (Path B+)

Cycle 2C **travou** o P0 metric set mínimo. Cycle 3C projetou layout, fontes Paperclip, refresh e anti-vanity. P4.5 **implementa** a superfície Insights com **exatamente 7 métricas** — nada mais no v1.

| Lane | Métricas P0 (CONFIRMED only) | Claim(s) |
|------|------------------------------|----------|
| **Human** | Cycle Time p50 · Capacity load | C2-D4-02, C2-D4-01 |
| **Agent** | Avg Cost/job · Jobs in Progress · Intervention count | C2-D4-01, C2-D4-05 |
| **Room** | Hybrid Cycle Time · Co-touch rate | C2-D4-02/03, C2-D4-06 |

**Job-to-be-done:** provar, **fora do chat**, se o time híbrido está fluindo (humano), custando e sendo intervindo (agente), e colaborando de verdade (room) — com números honestos e ações claras.

### 1.2 Problema

| Hoje | Gap |
|------|-----|
| `Costs.tsx` / `costs.ts` (P4) | Ledger $ por hop/session — **não** orquestração dual |
| `Dashboard.tsx` | Company/agents genérico — sem tabs Human \| Agent \| Room P0 |
| Team strip (P2.5) | WIP operacional — **não** prova semanal de valor |
| Room / BoardChat stream | Narrativa A2A — **proibido** embutir Dual (D-11) |

### 1.3 Escopo explícito

| Dentro P4.5 | Fora P4.5 (Won't / OUT 2C) |
|-------------|---------------------------|
| Página **Insights** com tabs **Human \| Agent \| Room** | ROAS, marketing lift, #agents vanity KPI |
| **Exatamente** as 7 métricas P0 acima | TTFT, latency p95, raw token count |
| Placement **fora do stream** (D-11 / R-09) | “80% autonomia” / Autonomy ratio / Collab Score UI |

| Reuso cost-events + issues + runs + memberships | Novo billing ledger / ML forecast |
| Feature flag `enableDualPerformanceV1` | Widgets Dual dentro do Room stream |

### 1.4 Princípio anti-hype

> Medimos **orquestração e custo do ciclo**, não “produtividade moral” do humano.  
> Labels: **“Orquestração humana”** — **nunca** “employee score” / “rating” / “PDI”.  
> Co-touch ≠ success (AAAI C2-D4-06). Intervention ≠ culpa do agente.

### 7.4 Mapa métrica → fonte Paperclip

| Métrica P0 | Fonte primária | Agregação | Notas |
|------------|----------------|-----------|-------|
| Cycle Time p50 | **issues** (start/complete ou status transitions) | p50 na janela; owner ∈ human memberships | Excluir never in-progress |
| Capacity load | **issues** WIP + **memberships** limits | WIP/limit; G/Y/R thresholds | Reusar P2.5 limits ou default company |
| Avg Cost/job | **cost-events** (P4) | sum($) / count(jobs done) por agentId | REUSE `costs.ts` |
| Jobs in Progress | **runs** running/queued | count **now** | Live; não histórico |
| Intervention count | **runs**/activity/HITL (`needs_you`, pause, override) | count na janela; rate = interv / jobs done | `null` se HITL ausente |
| Hybrid Cycle Time | **issues** + hops agentic (delegation/runs) | p50 subset ≥1 hop | Mesmo clock que Hu-1 |
| Co-touch rate | **issues** + hist. ações (comments/runs/work-requests) | % (human ∧ agent) | Proxy AAAI — **não** Int_cons |
