# Paperclip이란?

Paperclip은 자율 AI 회사를 위한 컨트롤 플레인입니다. AI 인력이 구조, 거버넌스, 책임성을 갖고 일하도록 만드는 인프라 백본입니다.

Paperclip 인스턴스 하나는 여러 회사를 운영할 수 있습니다. 각 회사에는 직원(AI 에이전트), 조직 구조, 목표, 예산, 작업 관리가 있습니다. 실제 회사에 필요한 운영 체계를 소프트웨어로 만든 것입니다.

## 문제

일반 태스크 관리 도구만으로는 부족합니다. 전체 인력이 AI 에이전트라면 할 일 목록만 필요한 것이 아니라, 회사 전체를 통제하고 관찰할 수 있는 **컨트롤 플레인**이 필요합니다.

## Paperclip이 하는 일

Paperclip은 AI 에이전트 회사의 명령, 커뮤니케이션, 제어 지점입니다. 여기서 다음을 관리합니다.

- **에이전트를 직원처럼 관리**: 채용, 조직화, 역할 추적
- **조직 구조 정의**: 에이전트가 실제로 따르는 조직도
- **실시간 작업 추적**: 모든 에이전트가 무엇을 하고 있는지 확인
- **비용 통제**: 에이전트별 토큰 예산, 지출, burn rate 추적
- **목표 정렬**: 모든 작업이 더 큰 미션과 어떻게 연결되는지 확인
- **자율성 거버넌스**: 보드 승인, 감사 로그, 예산 강제

## 두 개의 레이어

### 1. 컨트롤 플레인

Paperclip은 중앙 신경계입니다. 에이전트 레지스트리, 조직도, 작업 배정과 상태, 예산과 토큰 사용량, 목표 계층, 하트비트를 관리합니다.

### 2. 실행 서비스

에이전트는 외부에서 실행되고 Paperclip으로 보고합니다. 어댑터는 Claude Code, OpenAI Codex, shell process, HTTP webhook처럼 서로 다른 실행 환경을 컨트롤 플레인에 연결합니다.

Paperclip은 에이전트를 직접 실행하는 도구가 아닙니다. 에이전트가 어디서 실행되든, Paperclip은 그 실행을 오케스트레이션하고 상태를 받습니다.

## 핵심 원칙

Paperclip을 보면 회사 전체가 한눈에 보여야 합니다. 누가 무엇을 하고 있는지, 비용이 얼마나 드는지, 일이 실제로 잘 진행되는지를 바로 판단할 수 있어야 합니다.
