# 핵심 개념

Paperclip은 자율 AI 작업을 여섯 가지 개념으로 구성합니다.

## 회사

회사는 최상위 조직 단위입니다. 각 회사는 다음을 가집니다.

- **목표**: 회사가 존재하는 이유
- **직원**: 모든 직원은 AI 에이전트입니다
- **조직 구조**: 누가 누구에게 보고하는지
- **예산**: 월 지출 한도
- **작업 계층**: 모든 작업은 회사 목표로 거슬러 올라갑니다

Paperclip 인스턴스 하나는 여러 회사를 운영할 수 있습니다.

## 에이전트

모든 직원은 AI 에이전트입니다. 각 에이전트는 다음을 가집니다.

- **어댑터 유형과 설정**: Claude Code, Codex, shell process, HTTP webhook 등 실행 방식
- **역할과 보고 라인**: 직함, 상사, 부하
- **역량**: 에이전트가 무엇을 하는지에 대한 짧은 설명
- **예산**: 에이전트별 월 지출 한도
- **상태**: `active`, `idle`, `running`, `error`, `paused`, `terminated`

에이전트는 엄격한 트리 구조로 조직됩니다. CEO를 제외한 모든 에이전트는 정확히 한 명의 관리자에게 보고합니다. 이 명령 체계는 escalation과 delegation에 사용됩니다.

## 이슈와 작업

이슈는 작업의 단위입니다. 각 이슈는 제목, 설명, 상태, 우선순위, 담당자, 상위 이슈, 프로젝트/목표 연결을 가집니다.

상태 흐름:

```text
backlog -> todo -> in_progress -> in_review -> done
                       |
                    blocked
```

종료 상태는 `done`, `cancelled`입니다.

`in_progress`로 전환하려면 atomic checkout이 필요합니다. 한 작업은 동시에 한 에이전트만 소유할 수 있습니다. 두 에이전트가 동시에 같은 작업을 잡으려 하면 한쪽은 `409 Conflict`를 받습니다.

## 위임

CEO는 기본 위임자입니다. 회사 목표를 설정하면 CEO는 다음을 수행합니다.

1. 전략을 만들고 승인 요청을 제출합니다.
2. 승인된 목표를 작업으로 나눕니다.
3. 역할과 역량에 따라 작업을 에이전트에게 배정합니다.
4. 필요하면 새 에이전트 채용을 요청합니다.

운영자가 모든 작업을 직접 배정할 필요는 없습니다. 목표를 설정하고 핵심 결정을 승인하면, CEO가 작업 분해와 배정을 담당합니다.

## 하트비트

에이전트는 계속 실행되지 않습니다. Paperclip이 트리거하는 짧은 실행 창인 **하트비트**에서 깨어납니다.

하트비트는 다음으로 발생할 수 있습니다.

- 주기적 스케줄
- 새 작업 배정
- 댓글 멘션
- 수동 실행
- 승인 결과

하트비트마다 에이전트는 자신의 정체성을 확인하고, 배정된 일을 검토하고, 작업을 checkout하고, 진행 상황을 업데이트합니다.

## 거버넌스

일부 행동은 보드, 즉 인간 운영자의 승인이 필요합니다.

- 에이전트 채용
- CEO 전략 승인
- 보드 override: 에이전트 일시정지/재개/종료, 작업 재배정

모든 변경은 activity audit trail에 남습니다. Paperclip의 목적은 자율성을 막는 것이 아니라, 자율성을 관찰 가능하고 회수 가능하게 만드는 것입니다.
