# Internationalization (i18n)

This fork now uses a namespace-based i18n architecture with `i18next` + `react-i18next`.

## Folder structure

```txt
ui/src/i18n/
  index.ts
  provider.tsx
  locales.ts
  locale-validation.ts
  locales/
    en/
      common.json
      sidebar.json
      issues.json
      goals.json
      agents.json
      settings.json
      workflow.json
    pt-BR/
      common.json
      sidebar.json
      issues.json
      goals.json
      agents.json
      settings.json
      workflow.json
```

## Runtime behavior

- Default locale: `en`
- Fallback locale: `en`
- Supported locales are discovered automatically from `ui/src/i18n/locales/<locale>/<namespace>.json`
- Locale persistence key: `paperclip.ui.locale` (localStorage)
- `I18nProvider` restores locale on boot and updates `document.documentElement.lang`

## UI language selector

- Location: `Instance Settings > Profile`
- Options:
  - English
  - Português (Brasil)
- Changing locale updates UI immediately.

## Namespaces

- `common`: generic labels, loading states, global navigation, breadcrumbs
- `sidebar`: side navigation, workspace switcher, project/agent sidebar actions
- `settings`: profile and account-menu language
- `issues`: issues page and Brabrix issue sync labels/messages
- `goals`: goals page labels
- `agents`: agent-specific toast messages
- `workflow`: onboarding and workflow-related labels

## Validation and safety

`ui/src/i18n/locales.ts` validates each locale against English per namespace using:

- key parity (missing/extra keys)
- placeholder parity (`{{variable}}`)
- basic payload safety checks

## Adding a new language

1. Create `ui/src/i18n/locales/<new-locale>/`.
2. Copy all namespace JSON files from `en/`.
3. Translate values (preserve keys and placeholders).
4. Run:

```bash
pnpm --filter @paperclipai/ui typecheck
```

## Best practices

- Keep English as source-of-truth.
- Use `t("key", { defaultValue: "English text" })` for safe incremental adoption.
- Prefer namespace-appropriate keys (`sidebar.*`, `settings.*`, etc.) instead of a global flat file.
- Avoid hardcoded user-facing strings in new components.
