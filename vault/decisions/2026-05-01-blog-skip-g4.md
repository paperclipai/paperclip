# Policy: Blogs skip G4. CEO is the final blog approver.

## Decision

**Blogs no longer require human (Vardaan) approval. CEO G3 PASS = auto-publish within 5 minutes.** Only courses retain the G4 human gate, and only when `high_stakes: true`.

## Why

Vardaan is bottlenecking publish throughput by hand-approving every blog. The CEO is already trusted with G3 alignment (success-criteria, scope, budget, strategy) — that's enough quality control for daily news commentary. Blogs are time-sensitive ("Learn AI the day it ships"); a 24-hour human-approval lag erases the freshness advantage. Courses are different — they're brand-defining, multi-chapter, evergreen, and worth Vardaan's attention.

## Scope

| Content type | G0 | G_code | G2 | G3 | G4 |
|---|---|---|---|---|---|
| **Blog** (any blog ticket, `vault/blogs/...`) | ✓ | n/a | ✓ | ✓ (CEO PASS = publish) | **never** |
| **Course** — single-chapter rewrite | ✓ | n/a | ✓ | ✓ | **never** (auto-publish) |
| **Course** — full new course launch | ✓ | n/a | ✓ | ✓ | **yes** (high-stakes locked at ticket creation) |
| **Course** — module Vardaan flagged `high_stakes: true` | ✓ | n/a | ✓ | ✓ | **yes** |
| **Code / engineering** | n/a | ✓ | ✓ | ✓ | **never** (no publish_state) |

## Operational implications

- **CEO comment-trigger throttle stays in place** — the existing 5-min debounce keeps blog auto-publish from re-firing on stale issue_commented wakes.
- **EOD digest** still tells Vardaan what shipped — he reads what auto-published, doesn't approve it.
- **publish-action.sh cron** (every 5 min) is unchanged — it polls `status='done' AND metadata.publish_state IN ('ready','g4-approved')`. Blogs land in `ready`, courses can land in either.
- **Reject path for blogs** — if a published blog is wrong, the recovery is a follow-up ticket (correction post or take-down), not a pre-publish gate.

## Quality safeguard the CEO must maintain

Without G4 as a backstop, the CEO's G3 alignment check is now load-bearing. The CEO must:

1. **Read the work product, not just the brief** — every paragraph, no skim-and-stamp.
2. **Block on red flags QA missed** — factual errors, dead links, unsourced claims, "AI-tells" prose.
3. **Block on brand voice violations** — hype, hedging, generic press-release tone, or competitor framing that could backfire.
4. **Cost overrun threshold tightens** — was 1.5×; now 1.3× for blogs since there's no human catch-up gate.
5. **High-stakes flag check** — if the topic touches legal/regulatory/competitor-defamation risk, the CEO escalates the ticket to Vardaan as a one-off (not via G4 routing — just a direct ping) instead of auto-publishing.
