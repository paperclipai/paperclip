# Rule: Implementation Planning

Major changes must be documented in dated plan files within `doc/plans/` to maintain a history of architectural and feature decisions.

- **Activation**: `Always On` (whenever creating a new `implementation_plan.md` artifact)

## Guidelines

- **Location**: Permanent plans reside in `doc/plans/YYYY-MM-DD-plan-name.md`.
- **Content**: Include the goal, context, technical approach, open questions, and verification steps.
- **Linkage**: Reference the plan file in your Pull Request description.
- **Archiving**: Once a plan is fully implemented, do not delete it; mark it as `[COMPLETED]` in a header.
